### 1. SkillCoach: Self-Evolving Rubrics for Evaluating and Enhancing Agentic Skill-Use

- 分數：5
- arXiv: [2607.01874](https://arxiv.org/abs/2607.01874)
- 發表日期：2026-07-03
- 分類：cs.AI
- 入選理由：+2 提及頂級機構：mit; +3 收錄於 Hugging Face Daily Papers; LLM rank #1 / 9/10: Introduces SkillCoach, a novel self-evolving rubric framework that improves evaluation and training of agentic skill-use with clear experimental evidence and practical impact on LLM agents.

**Intro:** 本論文聚焦於大型語言模型（LLM）代理人在使用技能庫時的可靠性問題。隨著技能庫規模擴大，技能間重疊導致代理人難以正確選擇和執行技能，最終驗證器的成功判斷過於粗糙，無法揭示過程中的錯誤。作者提出SkillCoach，一種自我演化的評分標準框架，從實際執行軌跡中提取技能相關的過程指標，評估技能選擇、遵循、組合及反思四個維度，並將外部驗證器結果獨立保留，以區分過程質量與偶然成功。此方法旨在提升技能使用的評估與訓練效果。

**Motivation:** 在企業級應用中，代理人需反覆使用內部流程和工具，技能庫規模大且包含重疊技能，導致代理人可能選錯技能、跳過關鍵步驟或錯誤組合流程。僅依賴最終驗證器結果無法反映過程中這些錯誤，且可能導致訓練時學習到脆弱策略。現有研究多聚焦於技能生成或檢索，缺乏針對技能使用過程的細緻評估與訓練信號。SkillCoach透過自我演化的過程性評分標準，提供更精細的行為診斷與訓練指導，符合實際部署需求。

**Method:** SkillCoach定義代理人的技能使用為涵蓋技能選擇、技能遵循、技能組合及技能反思的軌跡級元能力。系統從任務指令、技能庫（包含金牌技能與干擾技能）、環境與驗證器出發，構建初始評分標準，並基於實際代理執行軌跡進行證據抽取與評分。透過多輪校準與驗證門控的局部修補，演化出更完善的評分標準。最終評分標準用於診斷技能使用失誤及篩選高質量軌跡，作為離線監督微調的訓練數據。評分標準明確分離過程評估與最終驗證結果，避免偶然成功干擾過程質量判斷。

**Result:** 實驗涵蓋18個訓練任務與10個測試任務，並引入干擾技能模擬企業環境。結果顯示，演化後的評分標準在覆蓋關鍵步驟、可用性、一致性及幻覺率等指標均優於初始標準。代理人在有金牌技能時任務成功率明顯提升，但加入干擾技能後技能選擇分數下降，反映技能選擇成為瓶頸。使用演化後的評分標準篩選軌跡進行監督微調，較僅依結果篩選提升顯著，且關鍵步驟遵循與組合順序對性能貢獻最大。進一步分析技能庫規模擴大與干擾技能類型發現，模型在技能選擇上存在明顯退化與崩潰邊界，且高相似度干擾技能對選擇影響最大。

**Conclusion:** SkillCoach提出一套自我演化的技能使用過程評分標準，能有效區分代理人技能使用的過程質量與最終任務成功，提供更細緻的失敗診斷與訓練信號。實驗證明該方法提升了評估的忠實度與訓練效果，並揭示了在大規模且重疊技能庫中技能選擇的挑戰。此框架對企業級代理人系統的技能管理與可靠性提升具有實際意義。限制在於實驗規模仍小於真實生產環境，且未涵蓋在線強化學習或長期部署反饋，未來可擴展至更大規模與多樣化場景。

**Research Gap:** 本研究雖在選定的技能依賴任務上驗證了方法有效性，但任務數量與多樣性仍有限，難以完全覆蓋大型生產環境中持續增長的技能與流程。演化過程依賴離線軌跡與人工設計的關鍵步驟，自動化程度與泛化能力尚待提升。訓練僅採用離線監督微調，缺乏在線強化學習或實際部署中長期反饋的驗證，無法評估方法在動態環境下的適應性與穩定性。此外，對於技能組合與反思維度的具體貢獻與失敗案例分析較少，未充分揭示各維度在不同任務類型中的重要性與限制。技能庫擴大時的計算成本與部署風險亦未明確討論。

### 2. Discrete Diffusion Language Models for Interactive Radiology Report Drafting

- 分數：5
- arXiv: [2607.01436](https://arxiv.org/abs/2607.01436)
- 發表日期：2026-07-03
- 分類：cs.AI
- 入選理由：+2 提及頂級機構：mit; +3 收錄於 Hugging Face Daily Papers; LLM rank #2 / 9/10: Presents a novel diffusion language model adapted for medical report drafting, demonstrating faster decoding and unique any-order infill capability with strong empirical results on medical datasets.

**Intro:** 本論文探討離散擴散語言模型（diffusion language models）在醫學影像報告撰寫中的應用，特別是與傳統自回歸（autoregressive, AR）模型的比較。現有醫學基礎模型多為自回歸架構，限制了互動式報告草稿的靈活性。作者以DiffusionGemma-26B為基礎，與同尺寸的Gemma-4-26B自回歸模型進行匹配實驗，評估兩者在醫學視覺問答（medical VQA）任務上的表現與推理速度。論文同時提出擴散模型固有的任意位置填充（any-order infill）功能，適合臨床報告中常見的片段修正需求。

**Motivation:** 醫學影像報告撰寫常需在草稿中局部修改並填補缺漏，傳統自回歸模型只能從左至右生成文本，無法有效利用後方上下文資訊。這限制了模型在臨床工作流程中作為互動輔助工具的實用性。離散擴散模型透過雙向去噪機制，能在任意位置進行文本填充，提供更靈活的草稿編輯能力。此外，擴散模型在生成速度上有潛力優於自回歸模型，對於需要即時反饋的臨床應用尤為重要。

**Method:** 作者採用DiffusionGemma-26B離散擴散語言模型及其自回歸同胞Gemma-4-26B，兩者均為約26B參數的混合專家模型，並使用相同的LoRA微調策略以確保比較公平。模型輸入為醫學影像及對應文本，擴散模型透過uniform-state dLLM目標進行訓練，隨機替換部分目標詞彙並學習還原；自回歸模型則採用標準的下一詞交叉熵損失。任意位置填充透過在每個去噪步驟中固定已知文本片段，讓模型雙向利用上下文填補缺口，這是自回歸模型無法實現的。

**Result:** 在三個醫學VQA資料集（VQA-RAD、SLAKE、VQA-Med-2019）上，擴散模型在經過LoRA微調後，LLM判分的語義正確率等同或優於自回歸模型，且在SLAKE資料集上差異達統計顯著。擴散模型的推理速度比自回歸快3.5至4.4倍，吞吐量提升5.7至7.1倍。任意位置填充實驗中，擴散模型利用左右兩側上下文填補被遮蔽句子，token-F1與語義判分均顯著優於自回歸模型，即使後者在提示中加入右側上下文也無顯著提升。

**Conclusion:** 本研究證實離散擴散語言模型在醫學影像報告生成任務中，不僅在準確度上可與自回歸模型匹敵，且在推理速度與互動式草稿編輯能力上具明顯優勢。任意位置填充功能特別適合臨床報告中多變且不一致的文本結構，提升了模型在實際醫療工作流程中的應用潛力。作者亦公開了程式碼與微調模型，促進後續研究與應用。

**Research Gap:** 論文中雖展示擴散模型在醫學VQA及報告填充上的優勢，但全文未明確說明模型在更大規模、多樣化臨床場景下的泛化能力與穩定性。實驗主要聚焦於LoRA微調，未探討全模型微調或其他適應策略對性能的影響。任意位置填充的臨床實際使用者體驗與錯誤案例分析缺乏，無法評估該功能在真實醫療環境中的風險與限制。此外，擴散模型的計算成本與部署複雜度相較自回歸模型是否存在潛在挑戰，全文亦未詳述。未來研究應補足這些方面以全面評估擴散模型的臨床可行性。

### 3. Separating Expert Retention from Autonomous Source Inference in Raw-ECG-Replay-Free Continual ECG Deployment

- 分數：5
- arXiv: [2607.01674](https://arxiv.org/abs/2607.01674)
- 發表日期：2026-07-03
- 分類：cs.AI
- 入選理由：+2 提及頂級機構：meta, mit; +3 提及頂級會議：acl; LLM rank #3 / 8/10: Addresses a practical deployment challenge in ECG multi-source models with a credible method and thorough evaluation, highlighting autonomous source inference as a key bottleneck.

**Intro:** 本論文針對多來源心電圖(ECG)部署中，當無法保留或重播早期原始ECG數據時，模型如何持續整合新數據來源的問題進行研究。傳統方法凍結預訓練骨幹網絡並為每個來源分配獨立分類器，可避免參數干擾，但在推理階段若缺乏來源元資料，仍需解決如何選擇合適專家的挑戰。作者提出IRFE-ECG方法，利用凍結的ECGFounder特徵建立增量專家庫，並設計輕量級路由器在保留的訓練特徵上進行來源推斷，進而融合多個專家的決策。研究聚焦於分離專家保留與自主來源推斷兩個問題，並在多個公開ECG資料集上驗證。

**Motivation:** 在實際ECG系統部署中，數據來源多樣且隨時間變化，且受到數據治理與共享限制，無法保留所有原始數據。傳統共享參數模型容易因新來源訓練而覆蓋舊來源知識，且來源未知時難以選擇合適模型。凍結骨幹並隔離專家參數能防止遺忘，但若無法準確推斷輸入來源，仍會造成性能下降。因而區分專家保留與來源推斷，並針對來源未知情況設計有效路由策略，對工程部署與臨床應用具有重要意義。

**Method:** IRFE-ECG方法基於凍結的ECGFounder骨幹，將每個新來源加入一個獨立的線性分類器專家，使用Balanced Softmax處理類別不平衡，並在驗證集上校準決策閾值。為解決來源未知問題，設計一個輕量級MLP路由器，僅使用保留的訓練特徵與來源標籤進行訓練，無需重播原始ECG。推理時，路由器輸出來源概率，並採用驗證校準的top-2邊際融合策略，結合兩個最可能專家的輸出以降低硬路由錯誤風險。實驗中亦比較多種路由器與基線方法，並評估不同特徵存儲預算對性能的影響。

**Result:** 在CPSC、PTB-XL、Georgia與Chapman-Shaoxing四個資料集上，來源已知的專家選擇達到約0.7915的Macro-F1，與離線同時訓練的獨立頭參考相近，顯示專家保留效果良好。無來源ID時，MLP路由器達0.7756 Macro-F1，top-2邊際融合略提升至0.7782，但提升不具統計顯著性。路由器在CPSC與Georgia來源辨識提升明顯，但仍存在Chapman來源混淆。不同資料順序下top-2與oracle間的性能差距穩定約0.0111至0.0133，表明自主來源推斷是主要瓶頸。特徵存儲量減少至10%時性能仍接近全量，顯示存儲效率良好。與共享參數基線比較，IRFE-ECG在避免參數干擾方面優勢明顯。

**Conclusion:** 本研究成功將專家保留與自主來源推斷兩個問題分離，證明凍結骨幹與獨立專家頭能有效防止遺忘，且來源已知時性能接近離線訓練參考。自主路由器雖提升來源推斷能力，但仍存在明顯性能差距，成為持續部署的主要限制。方法避免原始ECG重播，僅保留特徵記憶，提升了數據治理合規性。未來需針對患者級別分割驗證、特徵存儲優化及多標籤診斷擴展，以實現更完整的臨床部署方案。

**Research Gap:** 本論文未能驗證患者級別的數據分割，存在潛在的患者資料洩漏風險，限制臨床應用的可信度。僅使用四個二分類來源，未涵蓋多標籤診斷或更複雜的心電圖任務，泛化性有限。路由器依賴保留的訓練特徵，雖避免原始數據重播，但仍非完全無記憶，且未明確討論隱私風險與存儲成本。專家數量隨來源線性增長，可能影響長期部署的計算與存儲效率。top-2融合提升有限且不顯著，缺乏更強的路由策略或失敗案例分析。全文未明確說明在更大規模或異質來源下的適應性與穩定性。

### 4. Path-level Hindsight Instructions for Semantic Exploration in Vision-Language Navigation

- 分數：5
- arXiv: [2607.01754](https://arxiv.org/abs/2607.01754)
- 發表日期：2026-07-03
- 分類：cs.AI
- 入選理由：+2 提及頂級機構：mit; +3 提及頂級會議：acl; LLM rank #4 / 8/10: Proposes Phi-Nav, a novel hindsight instruction framework for vision-language navigation that effectively bridges semantic gaps in exploration, validated on strong benchmarks with data efficiency.

**Intro:** 本論文聚焦於視覺語言導航（Vision-Language Navigation, VLN）中，訓練代理人時常用的on-policy探索策略所帶來的語義監督不匹配問題。當代理人偏離專家示範路徑時，原始語言指令無法準確描述實際執行的視覺序列，導致訓練信號稀疏且不一致。為解決此問題，作者提出Φ-Nav框架，利用回顧式推理生成與實際路徑對應的語義指令，從而提升語義監督的密度與準確性。此方法在R2R-CE與RxR-CE兩大VLN基準數據集上展現出競爭力的導航性能，同時顯著降低對專家示範數據的依賴。

**Motivation:** 在VLN任務中，on-policy探索能擴展代理人所見狀態空間，提升模型魯棒性，但也造成語義監督的斷層，因為原始指令不再對應代理人實際行走的路徑。這種語義不匹配限制了探索軌跡的有效利用，降低訓練效率。現有方法多依賴靜態專家示範，無法充分利用探索過程中產生的多樣化路徑。Φ-Nav透過生成與探索路徑語義對齊的回顧指令，將偏離路徑的探索行為轉化為有意義的語言監督，從而提升訓練樣本的豐富度與質量，對工程上減少標註成本及提升模型泛化能力具有實際意義。

**Method:** Φ-Nav包含三個主要階段：第一階段，代理人在on-policy設定下進行軌跡採樣，並接受專家行動的即時糾正；第二階段，利用大型視覺語言模型（LVLM）作為回顧講者（Hindsight Speaker Agent），根據代理人實際觀察的視覺序列生成對應的回顧指令，並透過專家示範作為上下文示例以維持語言風格與結構一致性；第三階段，代理人以生成的回顧指令與軌跡對作為額外的專家示範，進行第二輪模仿學習。為避免生成指令中的幻覺或語義偏差，設計了軌跡-指令對齊權重機制，根據視覺與語言的一致性動態調整回顧監督的影響力，確保訓練信號的可靠性。

**Result:** 在R2R-CE數據集上，Φ-Nav在DAgger與Scheduled Sampling兩種on-policy訓練策略下均提升了成功率（SR）與路徑效率（SPL），例如在DAgger設定中，成功率提升約2個百分點；在Scheduled Sampling設定中，成功率提升超過5個百分點，並超越部分最新基線方法。於RxR-CE數據集，Φ-Nav同樣帶來穩定提升，尤其在長距離與語言描述更複雜的場景中，提升了導航精度與路徑相似度指標（NDTW）。此外，Φ-Nav顯著提升了樣本效率，能在減少約10%專家示範數據的情況下，仍維持甚至超越基線性能。消融實驗證實軌跡-指令對齊權重與專家上下文學習策略對性能提升均有關鍵作用，且較大規模的LVLM模型能產生更高質量的回顧指令。

**Conclusion:** 本文提出的Φ-Nav框架有效解決了VLN中on-policy探索導致的語義監督缺失問題，通過生成與實際路徑對應的回顧指令，將探索軌跡轉化為密集且語義一致的訓練信號。實驗結果顯示，Φ-Nav不僅提升導航性能，還大幅降低對昂貴專家示範的依賴，展現出良好的樣本效率與泛化潛力。該方法為實際部署中如何利用代理人自身經驗進行自我監督提供了可行方案，對推動具備語言理解能力的機器人導航系統具有重要價值。

**Research Gap:** 儘管Φ-Nav在多項指標上展現優勢，但仍存在若干限制。首先，回顧指令生成依賴大型LVLM，生成品質受限於模型容量與語義一致性，且尚未完全消除幻覺問題；其次，軌跡-指令對齊權重的計算較為輕量，缺乏更精細的語義驗證機制，可能導致部分噪聲監督影響訓練；第三，實驗主要集中於室內導航場景，對更複雜或動態環境的泛化能力未充分驗證；此外，對失敗案例、長距離導航中回顧指令生成的穩定性與效率缺乏深入分析；最後，全文未明確說明該方法在實際部署時的計算成本與延遲，對工程實用性評估仍有限。未來工作可聚焦於提升生成指令的準確度、擴展至多模態與多任務場景，以及優化在線推理效率。

### 5. Rethinking Complexity Metrics for LLM-Integrated Applications: Beyond Source Code

- 分數：4
- arXiv: [2607.01903](https://arxiv.org/abs/2607.01903)
- 發表日期：2026-07-03
- 分類：cs.AI
- 入選理由：+2 提及頂級機構：mila; +2 提及程式碼可用; LLM rank #5 / 8/10: Develops HECATE, the first complexity metric tool for LLM-integrated applications combining prompt and code layers, validated on large datasets with credible empirical evidence.

**Intro:** 本論文針對LLM整合應用中複雜度衡量的不足提出探討。傳統軟體複雜度指標僅針對程式碼層面，忽略了大量行為邏輯存在於自然語言提示層。LLM整合應用結合自然語言提示與程式碼，提示層的複雜度對維護成本影響顯著，但現有指標無法量化。作者提出HECATE工具，首次同時評估提示層與程式碼層的複雜度，並以維護活動作為實證依據驗證指標有效性。

**Motivation:** 隨著LLM整合應用從原型走向生產，複雜度成為工程挑戰。複雜度高的軟體更難維護、除錯且易出錯。現有複雜度指標無法捕捉提示層的行為邏輯，導致維護風險被低估。提示與程式碼層緊密耦合，且程式碼大小常主導指標效果，需設計能獨立於程式碼大小且涵蓋提示層的複雜度指標，提升工程與產品的維護可視化與管理。

**Method:** HECATE基於Prompt-as-Specification模型，將提示視為行為規範，包含條件規則、全域不變式與狀態謂詞，賦予提示結構化形式。從三大公開分類法萃取25個複雜度維度，設計52個候選指標涵蓋提示層、程式碼層及兩者介面。指標設計策略包括傳統指標遷移、分類法操作化及新指標創新。HECATE以靜態分析實現，無需執行程式或呼叫LLM，並以18個開源庫中118個元件的版本控制維護活動作為複雜度真實標準，採用控制程式碼大小的偏相關分析驗證指標有效性。

**Result:** 實驗結果顯示，52個候選指標中僅10個在控制程式碼大小後仍具顯著相關性，包括7個新指標與3個傳統指標。新指標多數衡量結構廣度，如LLM呼叫點數、記憶體屬性數、提示模板數，優於僅量化規模的指標。提示層指標在控制最強程式碼指標後仍顯著，證明提示複雜度為獨立維度。於6個未見庫中20個元件的驗證中，兩個最強指標維持預測能力，顯示良好泛化性。傳統指標如McCabe循環複雜度在控制大小後失效，RFC指標表現較佳但仍不及新指標。

**Conclusion:** 本論文提出HECATE，首個同時量化LLM整合應用提示與程式碼層複雜度的靜態分析工具。透過Prompt-as-Specification模型，設計出結構廣度為核心的新指標，超越傳統僅量化程式碼規模的指標。實驗證明新指標在多維維護複雜度指標上具顯著預測力且具泛化能力。HECATE可無執行成本整合至CI流程，為LLM應用維護提供新視角。限制在於僅量化靜態結構，未涵蓋動態行為，且目前實作以Python為主，未涵蓋多語言與多代理系統。

**Research Gap:** 本研究雖以118個元件與18個開源庫驗證指標有效性，但資料集仍以Python為主，缺乏跨語言與企業級應用驗證。指標依賴靜態分析與關鍵字匹配，可能漏判隱含控制流與動態生成提示。介面層指標在新興框架中泛化能力較弱，顯示對高階API封裝的適應性不足。維護複雜度以版本控制活動為代理，未直接測量實際維護成本或錯誤率。未明確探討指標在多代理系統或動態提示構造場景的適用性。未提供消融實驗分析各指標對預測力的獨立貢獻。
