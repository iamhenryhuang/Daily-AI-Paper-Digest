### 1. ATLAS: A Foundation Neural Sampler for Amorphous Materials

**Intro:** 非晶材料因其獨特的機械與功能性質，在材料科學與凝態物理中具有重要地位。然而，這類材料的能量景觀極為複雜，傳統分子動力學與蒙地卡羅方法在低溫下因需跨越高能障礙而效率低下。現有基於資料驅動的生成模型受限於稀缺且偏頗的參考結構數據，難以準確反映目標玻爾茲曼分布。本文提出ATLAS，一種基於擬合擴散過程的神經網絡採樣器，能直接從目標能量函數生成非晶結構，並可跨系統大小、溫度及組成泛化。此方法結合等變圖神經網絡與時間反轉擴散技術，實現高效熱力學量估計與結構導向。

**Motivation:** 非晶材料的結構多樣且能量景觀崎嶇，導致傳統模擬方法在低溫玻璃態下難以有效取樣，限制了對結構-性質關係的理解與材料設計。數據驅動生成模型依賴有限且偏頗的訓練集，無法保證生成樣本的熱力學正確性。開發一種能直接從能量函數取樣且不依賴預先計算結構的神經採樣器，對於工程與產品開發中快速獲得高品質非晶結構、估計熱力學性質及實現逆向設計具有實際意義。ATLAS旨在填補此空白，提升非晶材料模擬的效率與準確性。

**Method:** ATLAS利用一個基於擬合擴散過程的框架，將均勻先驗分布映射至目標玻爾茲曼分布。核心為一個E(3)-等變圖神經網絡，同時學習正向與反向隨機微分方程的漂移項，並以目標能量函數的力作為訓練信號，無需參考結構數據。訓練過程採用固定點迭代，自我引導生成樣本並更新模型。模型可條件化於溫度與化學組成，實現跨條件泛化。生成過程中利用路徑權重估計自由能、熵及期望值，並可透過推理時的序列蒙地卡羅方法對生成結構進行目標性質導向調控。

**Result:** 在二維Kob-Andersen系統中，ATLAS在溫度範圍0.2至5.0間訓練，成功重現平行溫度交換蒙地卡羅的結構分布與能量分布，低溫玻璃態下自由能誤差低於0.2%，且能量評估次數減少超過500倍。模型可無需重訓直接推廣至更大系統尺寸。對Cu-Zr及Cr-Co-Ni金屬玻璃，ATLAS能捕捉實驗觀察的短程有序趨勢，並有效調控Warren-Cowley參數及優化體積模量。組成攤銷預訓練顯著優於從零開始訓練，降低逆向設計成本數百倍。結合大型語言模型代理，ATLAS在八元素空間中尋找高熵金屬玻璃，於480次評估內收斂至Pareto前緣。

**Conclusion:** ATLAS建立了一個基礎神經採樣框架，統一了非晶材料的玻爾茲曼取樣、熱力學估計、結構導向生成與組成逆向設計。其跨溫度、組成及系統大小的泛化能力，尤其在結合高階機器學習勢能與語言模型代理後，展現出高效且靈活的材料設計潛力。儘管目前聚焦於固定體積與溫度的NVT系統，且系統尺寸有限，ATLAS為非晶材料模擬與設計提供了可擴展且可控的計算基礎。

**Research Gap:** 本文未明確展示ATLAS在更大尺寸系統（數千至數萬原子）上的效能與準確性，限制了其在界面、位錯等長程結構現象的應用。模型目前僅支持NVT系綜，缺乏NPT或半大宗系綜的擴展，無法自適應體積變化。推理時調控依賴近似梯度或代理模型，對非可微或昂貴性質的控制仍有挑戰。此外，對模型在極端條件下的泛化能力、失敗案例及計算成本的詳細分析不足。最後，逆向設計結果依賴於勢能模型的準確性，尚需實驗或更高階理論驗證。

### 2. Where Should Optimizer State Live? Tiered State Allocation for Memory-Efficient Mixture-of-Experts Training

- 分數：5
- arXiv: [2607.19058v1](http://arxiv.org/abs/2607.19058v1)
- 發表日期：2026-07-21
- 分類：cs.LG, cs.AI
- 入選理由：+3 收錄於 Hugging Face Daily Papers; +2 提及程式碼可用; LLM rank #2 / 8/10: SkewAdam presents a clear, practical memory optimization for MoE training with strong empirical evidence of improved perplexity and memory efficiency, addressing a concrete bottleneck in large-scale model training.

**Intro:** Mixture-of-experts (MoE) 模型在參數規模與計算效率上展現優勢，但其優化器狀態佔用大量記憶體，成為訓練瓶頸。以一個6.78B參數的MoE語言模型為例，AdamW優化器的狀態佔用超過50GB，遠超過模型權重本身。本文提出SkewAdam，針對MoE中不同參數群組（dense backbone、experts、router）特性差異，採用分層記憶體分配策略以降低優化器狀態佔用。此策略在保持模型效能的同時，大幅減少記憶體需求，適合有限記憶體資源的硬體環境。

**Motivation:** MoE模型的三個主要參數群組在大小與梯度統計上差異顯著，卻常被統一處理，導致優化器狀態浪費大量記憶體。現有記憶體節省優化器多半忽略這種結構差異，無法兼顧效能與資源限制。工程與產品層面，降低訓練記憶體需求可使MoE模型在更普遍的硬體上部署，降低成本並提升可擴展性。研究落地上，理解不同參數群組的優化需求，有助設計更有效率的優化器。

**Method:** SkewAdam根據參數角色分配優化器狀態：dense backbone保留float32動量與分解(second moment)估計，因其梯度密集且參數較少；experts僅保留分解的second moment，因其參數多且梯度稀疏；router保留完整的second moment以維持路由精度。分解second moment利用行列均值近似，顯著降低記憶體。訓練中採用更新裁剪與隨機抖動四捨五入以維持低精度下的數值穩定。實驗中保持相同初始化與數據順序，確保公平比較。

**Result:** 在6.78B參數MoE模型上，SkewAdam將優化器狀態從AdamW的50.6GB降至1.29GB，整體訓練峰值記憶體從81.4GB降至31.3GB，符合40GB加速器限制。SkewAdam在82M訓練tokens後達到驗證困惑度108.4，優於AdamW(126.8)、Muon(120.2)及Lion(393.7)。透過消融實驗證明，分層分配本身不提升準確度，準確度提升來自於在dense backbone保留動量。學習率調整後，SkewAdam仍領先最佳調參AdamW約10 perplexity。路由負載平衡方面，SkewAdam與AdamW接近理論均衡底線，顯示訓練穩定。

**Conclusion:** 本文提出的SkewAdam透過分層優化器狀態分配策略，顯著降低MoE模型訓練的記憶體需求，且不犧牲模型效能。此方法展現了針對MoE結構特性設計優化器狀態管理的重要性，為記憶體受限環境下的MoE訓練提供實用解決方案。限制包括模型深度較淺、訓練步數與上下文長度有限，以及部分基準未充分調參。未來需驗證多層MoE、長期訓練及多重隨機種子下的穩定性與泛化性。

**Research Gap:** 本研究使用兩層MoE模型，未測試多層MoE中分層策略的適用性與效益。訓練步數與上下文長度較短，無法反映長期訓練下優化器狀態分配的影響。多數實驗為單次執行，缺乏多隨機種子重複驗證，可能影響結果穩定性。部分基準優化器未充分調參，SkewAdam本身也未調參，可能影響公平性。權重衰減在實驗中未生效，未驗證其與分層策略的相容性。下游任務評估表現接近隨機，無法證明優化器改進對實際應用的提升。未探討分層策略在更大規模模型與多GPU環境的部署風險與成本。

### 3. ABot-World-0: Infinite Interactive World Rollout on a Single Desktop GPU

- 分數：5
- arXiv: [2607.19191v1](http://arxiv.org/abs/2607.19191v1)
- 發表日期：2026-07-21
- 分類：cs.CV, cs.AI, cs.LG
- 入選理由：+2 提及頂級機構：mit, nvidia; +3 收錄於 Hugging Face Daily Papers; LLM rank #3 / 7/10: ABot-World-0 offers a novel, real-time interactive video world model with extensive system design and evaluation, showing practical usefulness and credible evidence for long-horizon controllability on a single GPU.

**Intro:** 本論文提出ABot-World-0，一種基於動作條件的視頻世界模型，旨在實現實時、長時域的閉環互動。該模型利用多源數據基礎設施，涵蓋AAA遊戲、模擬引擎及網絡視頻，學習可控的世界動態。透過WorldExplorer系統進行智能數據收集，並結合多階段質量檢查與語義評估，確保數據質量。模型訓練採用由雙向教師模型到因果學生模型的漸進蒸餾策略，並引入LongForcing方法以減少長時域推理中的分布漂移。部署方面，設計了輕量級VAE解碼器、高效注意力機制及低位元推理，實現單GPU下720P、16FPS的流暢互動。實驗在WorldRoamBench及長時域互動中展示了良好的可控性與世界演化一致性。

**Motivation:** 隨著AI從生成靜態內容向生成可進入、可控且持續演化的世界轉變，建立一個持久且可互動的視覺世界模型成為關鍵挑戰。現有視頻生成技術雖提升視覺品質，但難以同時滿足可控性、狀態持續性、推理穩定性及實時部署需求。數據來源多樣且標註不一致，導致模型泛化受限。如何整合多源數據、實現統一控制介面並解決長時域累積誤差，成為推動實用互動世界模擬器落地的核心問題。

**Method:** ABot-World-0採用三階段訓練流程：首先，基於預訓練視頻生成模型，注入鍵盤動作控制信號與參考角色記憶，訓練雙向教師模型以捕捉全時域動態。其次，通過教師強制和ODE蒸餾，將雙向教師轉換為因果自回歸學生模型，實現低延遲推理。最後，LongForcing方法透過擴展教師監督至長時域學生自回放，減輕累積分布漂移。數據管線結合WorldExplorer智能代理驅動收集、多階段14項質量檢查及視覺語言模型輔助語義評估，統一多源動作標註與文本描述。部署方面，設計輕量VAE解碼器、低位元混合精度推理、快速RoPE位置編碼及記憶體感知調度，實現單GPU實時720P視頻生成。

**Result:** 在WorldRoamBench基準測試中，ABot-World-0在動作準確率、軌跡跟蹤、視覺美學、物理合理性及記憶保持等多項指標均達到競爭水準，優於多個先前模型。長時域（小時至天級）互動滾動實驗展示模型能維持場景結構、角色身份及動態連續性，避免快速退化。異域場景與角色控制測試證明統一鍵盤動作介面具良好泛化能力。物理交互案例中，模型能生成合理碰撞、水跡及足跡等細節。LongForcing相較於基線方法，有效減少60秒滾動中的視覺誤差累積，提升長期穩定性。部署測試顯示，優化後系統在RTX 5090 GPU上達成最高16 FPS、1.2秒動作到首幀延遲，峰值VRAM約19 GiB，符合實時互動需求。

**Conclusion:** ABot-World-0成功實現了基於動作條件的視頻世界模型，支持多源數據訓練、統一鍵盤控制及長時域穩定推理，並在單GPU上達成實時720P互動生成。其訓練流程結合雙向教師與因果學生蒸餾，並透過LongForcing減輕長期累積漂移，提升閉環互動的視覺與動態一致性。數據管線與部署架構的全棧設計確保了模型的實用性與可擴展性。該系統為本地視覺模擬器、互動內容創作及具身AI研究提供了實用基礎。限制方面，全文未明確說明在更複雜動作空間、極端場景或多角色交互下的表現，且對於更低端硬體的適配及量化訓練效果亦未深入探討。

**Research Gap:** 本論文雖在多源數據融合與長時域穩定性上有所突破，但仍存在數個不足。首先，對於動作空間僅限於鍵盤輸入，未涵蓋更豐富或連續控制信號，限制了泛化能力。其次，LongForcing雖減少累積漂移，但對於極端長時域（超過日級）或多角色複雜交互的穩定性未充分驗證。第三，雖有異域場景測試，但缺乏對真實世界動作多樣性及環境變化的系統性評估。第四，部署優化主要針對高端GPU，對中低端硬體的適配及量化訓練策略未明確說明，可能影響普及性。最後，消融實驗主要聚焦LongForcing，對其他組件如參考角色記憶、動作注入方式及數據收集策略的獨立貢獻缺乏詳細分析。

### 4. Mage-Flow: An Efficient Native-Resolution Foundation Model for Image Generation and Editing

- 分數：5
- arXiv: [2607.19064v1](http://arxiv.org/abs/2607.19064v1)
- 發表日期：2026-07-21
- 分類：cs.CV, cs.AI, cs.LG, cs.MM, eess.IV
- 入選理由：+2 提及頂級機構：nvidia; +3 收錄於 Hugging Face Daily Papers; LLM rank #4 / 7/10: Mage-Flow delivers a compact, efficient foundation model for high-resolution image generation and editing with strong system co-design and competitive benchmark results, enabling practical interactive use.

**Intro:** 本論文針對當前大型視覺生成模型在訓練、微調與部署上的高成本問題，提出一套名為Mage-Flow的緊湊型4B參數規模生成架構，支援高效的文字到圖像生成及指令式圖像編輯。Mage-Flow由兩個核心組件組成：Mage-VAE，一個輕量且高保真度的潛在空間編碼器，以及Native-Resolution Multimodal Diffusion Transformer，並採用修正流匹配訓練。該架構強調系統層級的效率優化，包含原生解析度的資料打包與CUDA核心融合，提升訓練吞吐量約2.5倍。論文同時發展出基礎版、強化版與Turbo快速推理版，兼顧生成與編輯任務。

**Motivation:** 現有高性能視覺生成模型多依賴數十億至數百億參數，導致訓練與推理成本極高，限制了研究與實務應用的普及。公開模型如FLUX.2、Qwen-Image等規模龐大，雖然品質優異，但難以在有限計算資源下進行調整與部署。此外，潛在空間編碼器（VAE）在高解析度下的編碼解碼成本亦成為瓶頸。Mage-Flow旨在透過模型與系統的協同設計，降低整體成本，提升高解析度生成與編輯的實用性，促進本地端部署與後續研究。

**Method:** Mage-Flow架構包含Mage-VAE與4B參數的Native-Resolution MMDiT。Mage-VAE採用一階擴散式編碼與解碼，並以anchor-latent KL正則化對齊強公眾VAE的潛在分布，達成高保真且大幅降低運算成本。Native-Resolution MMDiT則利用原生解析度打包技術，將不同解析度與長度的圖像與文字序列合併批次訓練，避免傳統固定解析度桶化限制，並支援變長FlashAttention與3D旋轉位置編碼。系統層面，作者融合Mage-VAE、Qwen3-VL文字編碼器與MMDiT的多個記憶體瓶頸操作為單一CUDA核心，顯著提升訓練效率。訓練流程包含分階段預訓練、監督微調、Diffusion-NFT後訓練以強化對齊，以及結合Decoupled DMD與對抗感知指導的4步驟蒸餾，產生低延遲Turbo版本。

**Result:** Mage-Flow在多項標準文字到圖像生成與指令式圖像編輯基準上展現競爭力表現。文字生成方面，於GenEval、DPG-Bench、TIIF-Bench、OneIG、CVTG-2K及LongText等八大評測中，Mage-Flow以4B參數規模取得最佳或接近最佳的成績，並在多語言文字渲染與複雜構圖理解上表現優異。編輯任務方面，Mage-Flow-Edit在ImgEdit-Bench、GEdit-Bench及TextEdit-Bench三大編輯基準中，與多數大型專家模型相當，且Turbo版本在4步驟推理下仍維持高品質。系統效能上，Mage-Flow在NVIDIA A100單卡上於1024×1024解析度生成時間約0.59秒，編輯時間約1.02秒，記憶體使用量低於多數對手，展現良好效率。

**Conclusion:** 本論文提出的Mage-Flow架構透過輕量化潛在空間編碼器、原生解析度多模態擴散Transformer及系統層級優化，成功在4B參數規模下實現高品質、高效率的圖像生成與編輯。該架構不僅在多項公開基準中與更大規模模型競爭，且顯著降低推理延遲與記憶體需求，使高解析度生成與編輯更適合互動式應用與本地部署。整體而言，Mage-Flow展示了模型與系統協同設計在視覺生成領域的實用價值，為後續研究與產品落地提供了可行且高效的基礎。

**Research Gap:** 儘管Mage-Flow在多項基準展現優異表現，全文未明確說明模型在更極端解析度（如超過2048×2048）或更複雜場景下的泛化能力與穩定性。此外，對於不同語言文字渲染的均衡性與細節控制仍有提升空間，尤其中文長文本渲染表現尚有待加強。蒸餾後的Turbo版本雖降低推理成本，但部分編輯指標表現不一，顯示在極端壓縮下的編輯精度仍有限。實驗中缺乏對失敗案例的深入分析，且成本與部署風險評估較為不足。未來可進一步探討模型在多模態理解、跨域適應及更細粒度控制上的能力與限制。

### 5. Beyond Score Prediction: LLM-Based Essay Scoring and Feedback Generation via Reinforcement Learning with Rubric Rewards

- 分數：4
- arXiv: [2607.19219v1](http://arxiv.org/abs/2607.19219v1)
- 發表日期：2026-07-21
- 分類：cs.CL, cs.AI
- 入選理由：+2 提及頂級機構：mit; +2 提及程式碼可用; LLM rank #5 / 7/10: RLAES advances LLM-based essay scoring and feedback generation with a novel RL framework and rubric-based evaluation, supported by strong experimental results and publicly available code and data.

**Intro:** 本論文聚焦於自動化作文評分（AES）與自動化回饋生成（AFG）兩大教育AI應用，指出現有方法多依賴提示工程或監督式微調，缺乏強化學習後訓練及回饋品質自動評估的系統性研究。作者提出一套統一的LLM強化學習框架RLAES，能同時優化作文分數預測與回饋生成。為解決回饋品質難以量化的問題，設計了基於細緻評分標準的回饋評估框架RFE，並引入Adaptive Gated Feedback Optimization（AGFO）以降低評估成本，同時提出Adjacent Contrastive Reasoning（ACR）改善分數的序數校準。實驗在ASAP資料集上展示了優越的分數預測與回饋品質。

**Motivation:** AES與AFG能減輕教師批改負擔並提供學生即時個性化診斷，但現有LLM方法多依賴提示設計或監督微調，缺乏強化學習後訓練的探索，且回饋品質評估多依賴人工，難以自動化且成本高昂。缺乏有效的回饋評估機制限制了回饋生成的優化。強化學習若能結合可驗證的回饋評估，將有助於提升模型在分數預測與回饋生成的整體表現，並促進教育AI系統的實際落地與應用。

**Method:** 核心方法包括三部分：首先，RFE框架設計了166項細粒度二元評分標準，涵蓋覆蓋度、證據、忠實度與安全性四個維度，並利用LLM作為評審者對回饋進行自動評分，將回饋品質轉化為可用於強化學習的獎勵信號。其次，AGFO透過門控機制，根據回饋獎勵是否達到門檻，動態決定是否啟用回饋評估，顯著降低評估成本並避免回饋品質下降。最後，ACR針對序數分類問題，要求模型在預測分數時對相鄰分數級別進行對比推理，強化模型對細微分數差異的辨識能力，減少相鄰分數混淆。整體訓練採用GRPO強化學習算法，並結合LoRA進行參數高效微調。

**Result:** 在ASAP資料集上，RLAES-AGFO在分數預測上達到QWK 0.803，超越所有LLM基礎方法，且接近單一人類評分者的表現（0.805）。ACR策略在多個基礎模型上均提升了分數預測準確度，減少了系統性低估分數的偏差。回饋品質方面，RLAES-AGFO的RFE評分達0.8399，與GPT-5.5（0.8334）相當，且避免了僅優化分數時回饋品質下降的問題。AGFO機制使訓練成本相比全面回饋優化降低約83%。RFE框架在區分真實與擾動回饋上達到100%準確率，且與專家偏好高度一致（93%），顯示其評估回饋品質的有效性與可靠性。

**Conclusion:** 本研究提出RLAES-AGFO，一套結合強化學習的LLM作文評分與回饋生成框架，透過細緻的RFE回饋評估與動態AGFO機制，有效提升了分數預測與回饋品質，並降低訓練成本。ACR策略則改善了序數分數的精細區分能力。實驗結果證明該方法在ASAP資料集上優於現有LLM方法，且回饋品質與專家評價高度一致。此框架為教育AI系統中同時優化評分與回饋提供了可行方案，促進了AES與AFG的實際應用。

**Research Gap:** 本論文依賴LLM作為評審者，然而不同LLM可能存在偏好差異，導致獎勵信號不穩定，全文未明確說明如何處理此類獎勵不匹配問題。實驗僅限於ASAP英文作文資料集，缺乏跨題目、跨語言及跨領域的泛化性驗證。回饋評估雖細緻，但未展示對不同回饋類型或長度的適應性，且成本仍較高，限制大規模部署。強化學習訓練過程中對失敗案例或模型偏差的分析不足，缺乏對模型在極端或低資源情境下表現的探討。此外，AGFO門控機制的超參數設定依賴離線估計，實際應用中可能需更靈活調整。未提供消融實驗以明確量化各組件對整體性能的貢獻。
