### 1. RuBench: A Repository-Level Agentic Coding Benchmark with Natively Authored Russian Task Specifications

- 分數：7
- arXiv: [2607.06411v1](http://arxiv.org/abs/2607.06411v1)
- 發表日期：2026-07-07
- 分類：cs.SE, cs.AI, cs.CL
- 入選理由：+2 提及頂級機構：meta, mit; +3 提及頂級會議：acl; +2 提及程式碼可用; LLM rank #1 / 9/10: Introduces RuBench, a novel benchmark with native Russian task specifications for agentic coding, providing contamination-controlled evaluation on real-world repository fixes with rigorous statistical analysis and full r

**Intro:** 本論文提出 RuBench 1.0，一個針對產品級代理程式碼維護任務的基準測試，特點是任務規格以原生俄文撰寫，非翻譯自英文。現有的庫級代理基準多以英文任務陳述為主，無法反映非英語使用者的真實需求。RuBench 採用來自五個活躍開源庫的25個真實修復任務，並以維護者的回歸測試作為評分標準。所有任務均在模型訓練截止日期之後，避免資料污染。此基準旨在衡量代理在非英語、真實庫環境中解決維護問題的能力。

**Motivation:** 隨著代理程式碼工具成為開發者日常，許多用戶以母語自然表達需求，這與英語中心的任務陳述截然不同。現有基準無法評估代理對非英語、客戶風格需求的理解與執行能力，且多數基準存在資料泄漏與測試弱化問題。RuBench 透過原生俄文任務和私有回歸測試，提供更真實且具挑戰性的評估環境，對工程師與產品團隊在多語言場景下部署代理工具具有實際參考價值。

**Method:** RuBench 採用真實修復提交挖掘25個任務，涵蓋Python、PHP、TypeScript、JavaScript四種語言及五個開源庫。每個任務以俄文撰寫，模擬客戶對承包商的請求，無翻譯痕跡。代理接收無歷史紀錄的庫快照與俄文規格，生成修補程式後由維護者的私有回歸測試判定通過與否。評估包含四種產品配置（Claude Code搭配不同模型及Codex CLI搭配GPT-5.5），每配置執行三次以捕捉變異，報告pass@1、成本、token使用量及置信區間。另對一配置進行軌跡審計以檢測模型替換行為。

**Result:** 最佳配置Claude Code + Opus 4.8達成78.7%任務通過率，最弱配置Haiku 4.5為53.3%。25個任務規模使得只有與最弱模型的差異在統計上顯著。成本與token效率呈現折衷，Sonnet 5在成本效益上較佳，GPT-5.5則在token使用上較節省。審計發現Fable 5配置在20%任務中因安全機制自動切換至Opus 4.8，顯示產品層面行為會影響評測結果。案例分析揭示部分模型未完全滿足俄文規格中行為契約，私有測試能有效區分修復品質。

**Conclusion:** RuBench 1.0展示了代理工具在非英語、真實庫維護任務中已有不錯表現，且不同產品配置間存在明顯性能差異。此基準強調以產品配置為評測單位，並提出軌跡審計以確保測量有效性。資料與軌跡公開，維護者回歸測試私有以防止資料污染。未來版本將擴充多語言對照組與更多難度層級，提升基準的廣泛適用性與細緻度。

**Research Gap:** RuBench 1.0任務數量有限（25個），導致僅能分辨較大性能差異，且所有規格由單一作者撰寫，可能帶有風格偏差，且未包含英語對照組，無法量化語言障礙影響。私有回歸測試限制第三方完全重現評分過程。產品配置自動更新與安全機制導致模型替換，增加評測複雜度且可能影響結果解讀。實驗中代理可存取網路，存在少量上游代碼檢索，雖影響有限但仍是潛在資料洩漏風險。未來需增加多作者、多語言規格，強化測試透明度與控制代理網路行為，以提升基準的全面性與公信力。

### 2. Danus: Orchestrating Mathematical Reasoning Agents with Fact-Graph Memory

- 分數：4
- arXiv: [2607.06447v1](http://arxiv.org/abs/2607.06447v1)
- 發表日期：2026-07-07
- 分類：cs.AI, cs.CL, cs.MA
- 入選理由：+2 提及頂級機構：mit; +2 提及程式碼可用; LLM rank #2 / 8/10: Presents Danus, a novel orchestration system for mathematical reasoning agents using a fact-graph memory enabling scalable, incremental proof construction on research-level problems with multiple case studies; strong lon

**Intro:** 本論文針對大型語言模型（LLM）在數學推理領域的應用，提出一套名為 Danus 的協調系統，旨在解決多代理並行證明搜尋時的記憶管理與協調困難。Danus 以共享的事實圖（fact graph）作為全局記憶機制，結合主代理負責規劃與協調、多個工作代理並行進行證明搜尋，以及無狀態的驗證器負責檢查數學命題的正確性。系統能累積長篇且細緻的數學證明，並支援與人類數學家的互動。論文透過六個研究級別的數學問題案例，驗證 Danus 在代數幾何、奇異理論與組合數學等領域的效能。

**Motivation:** 現有基於 LLM 的數學推理代理多以生成-驗證-修正循環為核心，但在擴展多代理並行證明搜尋時，缺乏有效的全局記憶管理與協調機制，導致中間命題混亂且錯誤傳播，影響整體推理品質。Danus 透過共享事實圖，將每個驗證過的命題及其邏輯依賴關係組織起來，讓多個工作代理能在不同證明路徑上並行探索，並由主代理統籌調度，提升推理效率與可靠性。此設計對工程與產品落地意義重大，因為它使得長期、多線程的數學推理任務可被有效管理與擴展。

**Method:** Danus 系統架構包含三類代理：主代理負責全局策略規劃與協調，定期總結證明狀態並重新分配工作；多個工作代理並行執行證明搜尋，專注於單一命題的提出與證明；無狀態驗證器獨立檢查工作代理提交的命題與證明，確保正確性。所有通過驗證的命題與其證明及依賴關係被存入事實圖，形成有向無環圖結構，作為系統唯一的真實來源。系統還包含雙層記憶：工作代理的本地記錄與全局記憶，幫助避免重複嘗試與記錄探索歷程。主代理可低頻率諮詢 GPT-5.5-pro 以獲取高階數學策略。完成證明後，系統能將事實圖轉換成可讀的論文稿件，並通過驗證器反覆校正稿件內容。

**Result:** 論文展示六個研究級別的數學問題案例，涵蓋代數幾何、奇異理論、組合數學等領域。實驗中，Danus 在多數案例中成功構建了長篇且細緻的數學證明，部分案例甚至完全由系統自動完成，無需人類數學指導。系統能自動分解問題、探索多條證明路徑、修正錯誤證明依賴，並生成符合數學嚴謹性的論文稿件。與先前系統 Rethlas 比較，Danus 在相同模型與問題下展現更強的推理深度與廣度，成功解決先前無法通過驗證的問題。實驗中亦記錄了人類介入的具體情境與效果，顯示系統與人類協作的可行性。

**Conclusion:** Danus 提供了一種基於事實圖的多代理協調架構，有效擴展了 LLM 在長期且複雜數學推理問題上的應用能力。系統通過嚴格分工與驗證機制，實現了多線程證明搜尋的組織與累積，並支援與人類數學家的互動與協作。實驗結果證明 Danus 能構建長篇、細緻且可靠的數學證明，並在多個研究領域取得突破。限制方面，系統仍依賴人類提供關鍵提示與最終審核，且論文撰寫階段仍存在格式與表述不完美，需人工調整。整體而言，Danus 是向人機協作數學研究系統邁進的重要一步。

**Research Gap:** 本論文雖展示了 Danus 在多個研究級問題上的成功應用，但仍存在若干不足。首先，系統在遇到無法自動突破的關鍵證明點時，仍需人類數學家提供提示，顯示自主創新能力有限。其次，論文撰寫階段的自動化仍不夠成熟，存在壓縮步驟導致錯誤的風險，且格式與符號表達需人工修正。第三，驗證器雖精確但依賴於引用文獻的正確性，若文獻本身有誤，系統難以自動糾正。第四，實驗範圍雖涵蓋多領域，但案例數量有限，泛化性與在更廣泛數學分支的適用性尚未充分驗證。最後，系統的計算成本與部署複雜度未明確評估，對實際工程落地的影響仍需進一步探討。

### 3. Canopy: A Heterograph Foundation Model for Metabolic Engineering

- 分數：5
- arXiv: [2607.06224v1](http://arxiv.org/abs/2607.06224v1)
- 發表日期：2026-07-07
- 分類：cs.LG
- 入選理由：+2 提及頂級機構：meta; +3 提及頂級會議：icml; LLM rank #3 / 7/10: Proposes Canopy, a large heterogeneous graph foundation model integrating multi-modal biological data for metabolic engineering with credible pretraining and downstream evaluation outperforming baselines; novel integrati

**Intro:** 代謝工程中設計能夠高效產生高價值化學品的微生物菌株是一項核心挑戰。現有計算方法多依賴於基於化學計量的約束模型，無法有效利用實驗數據，或是使用表格機器學習方法，忽略了生物知識的關聯結構。本文提出CANOPY，一個異構圖基礎模型，整合多種公開及專有數據源，構建包含690萬節點、13種節點類型及34種邊類型的知識圖譜，涵蓋基因、蛋白質、代謝物、反應、途徑、菌株及發酵實驗等多層次生物資訊。

**Motivation:** 代謝工程的設計-建造-測試-學習循環耗時且組合空間龐大，迫切需要計算工具來優先篩選基因改造方案以縮短實驗週期。傳統基於基因組規模代謝模型的約束方法無法融合實驗測量數據，且忽略調控及環境影響。現有機器學習方法多為表格形式，無法捕捉基因、蛋白質、反應等生物實體間的複雜關聯。異構圖神經網絡能夠跨物種整合多模態數據，提供更豐富的生物學上下文，有助於提升發酵產量預測及下游設計任務的表現。

**Method:** CANOPY利用BioCypher框架整合十個數據源，構建包含13種節點類型及34種邊類型的異構知識圖譜。節點特徵通過多模態預訓練基礎模型編碼，包括ESM-2處理蛋白序列、MoLFormer處理化學SMILES、PubMedBERT處理生物醫學文本。模型架構基於Heterogeneous Graph Transformer，結合SignNet位置編碼、Jumping Knowledge聚合及虛擬節點，採用四種自監督任務（連結預測、遮蔽節點建模、距離預測、實驗對比學習）進行預訓練，並利用同質不確定性加權自動平衡多任務損失。下游以凍結的節點嵌入進行發酵產量回歸，使用輕量探針評估表現。

**Result:** 在包含4791個發酵實驗的測試集上，CANOPY凍結嵌入配合輕量探針達到R2=0.41，明顯優於基於原始條件特徵的XGBoost（R2=0.24）及同質圖神經網絡變體。模型擴大至30億參數帶來約0.03的R2提升，顯示數據量限制了性能提升。消融實驗證實四種自監督任務均有貢獻，SignNet位置編碼、Jumping Knowledge及虛擬節點均顯著提升性能。多任務損失的同質不確定性加權有效避免了手動調整權重的困難。深度模型在訓練穩定性上略遜於淺層模型，但整體性能更佳。

**Conclusion:** CANOPY成功構建了一個跨物種、多模態的代謝工程異構知識圖譜基礎模型，通過自監督預訓練學習到的表示在發酵產量預測任務中超越傳統表格及同質圖模型。該模型提供了一個可重用的跨物種表徵基底，有望用於下游的菌株設計生成管線。限制包括實驗數據稀疏且分布不均、模型為預測性非機理性、缺乏圖外基線比較及部分數據和模型權重未公開。未來計劃擴充數據源、增加模態、拓展下游任務及結合生成模型實現閉環設計。

**Research Gap:** 本文尚未充分驗證模型在不同物種及化合物長尾分布上的泛化能力，且缺乏對單一數據源及模態貢獻的系統消融分析。模型為純預測性，缺少機理解釋，對失敗案例及模型不確定性未有深入探討。實驗數據主要來自文獻挖掘及專有庫，公開可用性有限，影響復現性。缺少圖外基線如純基礎模型嵌入拼接的比較，無法明確量化圖結構的增益。模型訓練及推理成本較高，尚未評估部署風險及實際應用中的效益轉化。未來需補充更多多樣化實驗數據及進行更全面的下游任務評估。

### 4. RSF-GLLM: Bridging the Semantic Gap in Multi-Hop Knowledge Graph QA via Recurrent Soft-Flow and Decoupled LLM Generation

- 分數：5
- arXiv: [2607.06527v1](http://arxiv.org/abs/2607.06527v1)
- 發表日期：2026-07-07
- 分類：cs.CL, cs.AI
- 入選理由：+2 提及頂級機構：mila; +3 提及頂級會議：icml; LLM rank #4 / 7/10: RSF-GLLM addresses a key limitation in multi-hop KG QA by enabling differentiable graph reasoning with theoretical convergence guarantees and competitive empirical results; methodologically sound and impactful for KG QA.

**Intro:** 多跳知識圖譜問答面臨一個核心挑戰：傳統的檢索後閱讀流程無法保持端到端可微分，導致檢索器無法學習跨越語義差距，尤其是當中間節點與查詢缺乏詞彙重疊時。為解決此問題，本文提出RSF-GLLM框架，將可微分的圖推理與答案生成解耦。該框架利用Recurrent Soft-Flow模組，通過GRU引導的查詢更新器傳播連續的相關性分數，並用動態門控機制依靠結構線索穿越語義不相似的橋接節點。最後，將推理路徑文本化並用於微調大型語言模型，確保生成結果基於真實拓撲結構。

**Motivation:** 多跳KGQA中，語義差距使得基於詞彙相似度的檢索無法有效找到中間橋接節點，導致推理失敗。現有方法要麼因離散檢索破壞可微性無法端到端學習，要麼直接用大型語言模型生成推理路徑，計算成本高且易出現幻覺。工程上，這限制了系統的效率和可擴展性。本文動機在於設計一種既可微分又能跨越語義差距的輕量級推理模組，並將其與LLM生成解耦，兼顧推理準確性與推理效率，利於實際部署。

**Method:** RSF-GLLM包含四階段：首先，利用密集檢索定位主題實體並構建子圖；其次，Recurrent Soft-Flow模組進行多步可微分圖推理，通過動態門控調節結構與語義信號的權重，允許在語義不匹配時依賴結構信息；該模組用熵正則化促使概率分布收斂至稀疏的離散推理路徑；第三，通過貪婪回溯從最終概率分布中提取推理路徑並將其文本化；最後，利用微調的LLM基於這些文本化路徑生成答案。此設計避免了LLM高方差梯度對圖推理的干擾，提升訓練穩定性與推理效率。

**Result:** 在WebQSP和CWQ兩個多跳KGQA基準上，RSF-GLLM在Hit@1和F1指標上均達到或超越現有最先進方法，WebQSP上Hit@1達90.45%，顯著優於RoG、GNN-RAG等基線。相比需要多次LLM調用的代理式方法，RSF-GLLM僅需一次LLM調用，推理速度提升約21倍。消融實驗證明內容偏置、動態門控、查詢更新和流稀疏正則化均對性能有顯著貢獻。子圖半徑擴大提升檢索性能，證實多跳推理能力。實驗同時展示了RSF模組在檢索效率和精度上的優勢，模型參數量遠小於大型LLM基線。

**Conclusion:** 本文提出的RSF-GLLM框架有效解決了多跳KGQA中的語義差距問題，通過動態結構-語義解耦和流稀疏正則化實現可微分且可解釋的推理路徑提取，並將推理與生成解耦以提升效率和穩定性。實驗結果證明該方法在準確率和推理速度上均具競爭力，適合實際工程部署。限制在於目前依賴固定K跳子圖，可能限制對大規模知識圖的覆蓋，且全文未明確說明在更複雜場景下的泛化能力和失敗案例。未來計劃通過動態擴展子圖和對比解碼等技術進一步提升性能和可靠性。

### 5. EntroPath: Maximum Entropy Path Ensemble Embedding for Manifold Learning

- 分數：4
- arXiv: [2607.06497v1](http://arxiv.org/abs/2607.06497v1)
- 發表日期：2026-07-07
- 分類：cs.LG, q-bio.QM, stat.ML
- 入選理由：+2 提及頂級機構：mila, mit; +2 提及程式碼可用; LLM rank #5 / 6/10: EntroPath introduces a theoretically grounded manifold learning method leveraging maximum entropy path ensembles, with strong empirical performance on synthetic and real benchmarks; clear novelty and practical value.

**Intro:** 本論文提出EntroPath，一種基於最大熵隨機漫步（MERW）的流形學習方法，旨在從資料圖中恢復測地幾何。現有方法多依賴局部正規化隨機漫步或最短路徑距離，前者易集中於高密度區域，後者對圖中虛假捷徑敏感。EntroPath透過聚合k步路徑的整體路徑集合，利用自由能不相似度來捕捉全局幾何結構，並證明其在短時間極限下收斂至平方測地距離。此方法同時提供可擴展的地標投影與擴散勢能偽時間推斷，並在合成流形及單細胞資料上展現優異表現。

**Motivation:** 高維資料通常集中於低維流形上，準確恢復其內在測地距離對降維與資料分析至關重要。傳統最短路徑方法易受圖中噪聲捷徑影響，且對非凸或分支流形恢復不佳。標準隨機漫步因局部度數正規化，導致擴散偏向高密度區域，忽略稀疏分支與瓶頸，影響單細胞發育軌跡推斷。MERW透過最大化整條路徑的熵，均勻分配路徑概率，有效保留瓶頸分離，提升對非均勻採樣與分支結構的魯棒性，對工程與產品中需精確流形結構的場景尤為重要。

**Method:** EntroPath核心在於利用MERW定義的k步轉移矩陣，計算路徑集合的自由能不相似度，該不相似度為k步路徑權重的負對數和，兼顧路徑長度與多樣性。此自由能不相似度在短時間極限下透過Varadhan熱核公式收斂至平方測地距離。方法中，MERW轉移矩陣由圖的最大特徵值與對應正特徵向量重整，形成類似薛丁格算子的拉普拉斯算子。EntroPath採用自適應帶寬高斯核構建圖，並以von Neumann熵的膝點自動選擇擴散深度k。為提升規模，提出地標Nyström投影與擴散勢能偽時間推斷，支持大規模資料的嵌入與軌跡分析。

**Result:** 在合成流形（Swiss roll、稀疏樹狀結構）上，EntroPath在距離層面（Spearman與Pearson相關）優於PHATE、HeatGeo、DTNE及Isomap，尤其在非均勻採樣與分支明顯的流形中表現突出。嵌入層面，EntroPath在保持局部結構（trustworthiness）與全局幾何（嵌入後測地相關）上均領先其他擴散方法。單細胞資料集（Paul15、Nestorowa、胰腺、淋巴細胞、胚胎體及Arabidopsis根系圖譜）中，EntroPath在DEMaP指標上多數情況下優於或接近最強基線，且保持良好局部鄰域保真度。地標擴展使其能在11萬細胞規模資料上快速運行，顯示良好可擴展性。消融實驗證實自動選擇的擴散深度接近最佳性能區間。

**Conclusion:** EntroPath結合最大熵隨機漫步與路徑集合自由能不相似度，提供一種兼具理論保證與實務效能的流形學習方法。其不相似度在短時間極限下收斂至平方測地距離，且具備精確的Gram矩陣分解，連結核方法視角。方法對非均勻採樣與分支結構具魯棒性，適合單細胞發育軌跡等應用。提出的地標與偽時間擴展提升了大規模資料處理能力。限制包括擴散深度選擇在短分支上較不穩定，且目前僅針對對稱親和矩陣，未充分驗證有向圖應用。未來可考慮連續時間擴散與有向數據擴展。

**Research Gap:** 全文未明確說明在極端高維或極大規模資料下的計算成本與記憶體需求，尤其在非地標模式下。擴散深度k的選擇雖有自動機制，但在短分支或複雜拓撲結構中表現不穩，缺乏更細緻的調節策略。對於非對稱親和矩陣（如RNA velocity等有向數據）僅提出理論可能性，缺乏實驗驗證。消融實驗主要集中於單一流形與參數，缺少跨多種資料類型與噪聲條件的廣泛測試。此外，方法對於極端稀疏或高度不均勻採樣的泛化能力仍需深入探討。最後，部署於實際產品或線上系統的延遲與資源消耗未被評估。
