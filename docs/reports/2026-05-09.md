### 1. PianoCoRe: Combined and Refined Piano MIDI Dataset

- 分數：7
- arXiv: [2605.06627v1](http://arxiv.org/abs/2605.06627v1)
- 發表日期：2026-05-07
- 分類：cs.SD, cs.LG
- 入選理由：+2 提及頂級機構：mit; +3 收錄於 Hugging Face Daily Papers; +2 提及程式碼可用

**Intro:** 本論文提出PianoCoRe，一個結合並精煉多個公開鋼琴MIDI資料庫的大規模鋼琴MIDI資料集。現有資料集多半涵蓋作曲家範圍有限、表演多樣性不足、缺少音符層級對齊或命名格式不一致，限制了音樂資訊檢索與表現建模的發展。PianoCoRe包含超過25萬次演奏，涵蓋超過5600首作品與483位作曲家，總時長超過2萬小時，並以分層子集形式發布以支援不同應用。此資料集同時提供音符對齊子集、MIDI品質分類器與對齊精煉管線，為後續鋼琴表現研究提供堅實基礎。

**Motivation:** 音樂資訊檢索與表現建模需要大量且高品質的符號音樂資料，尤其是匹配的樂譜與演奏資料。現有資料集多半規模有限或缺乏精確對齊，且資料來源多樣導致元資料不一致，難以整合使用。這些限制阻礙了更廣泛、多樣化且精確的表現分析與生成模型的訓練。PianoCoRe旨在彌補這些不足，透過整合多個資料庫並進行品質篩選與對齊精煉，提升資料的規模、品質與一致性，促進工程與研究的落地。

**Method:** PianoCoRe透過三階段流程整合多個公開鋼琴MIDI資料庫：首先合併ASAP與ATEPP兩大基礎資料集，建立核心結構；接著擴充樂譜來源，包括PDMX、MuseScore等公開樂譜庫；最後加入GiantMIDI-Piano、PERiScoPe與Aria-MIDI等演奏資料。為解決重複與品質問題，設計內容相似度基礎的去重方法與訓練基於Transformer的MIDI品質分類器，分類為高品質、低品質、損壞與樂譜型態。對齊方面，採用Parangonar進行初步音符對齊，並開發RAScoP管線進行對齊精煉，包含移除對齊空洞、時間清理、缺失音符插值與節拍同步，提升對齊的時間一致性與完整性。

**Result:** PianoCoRe最終包含250,046次演奏，5,625首作品，483位作曲家，總時長21,763小時。去重與品質篩選後的PianoCoRe-B子集包含214,092次高品質演奏。音符對齊子集PianoCoRe-A包含157,207次演奏與1,591首樂譜，PianoCoRe-A*為高品質子集。對齊精煉顯著降低了時間噪聲與極端節奏值，平均對齊召回率僅輕微下降1.5%。在表現渲染任務中，基於PianoCoRe-A訓練的模型在多來源測試集上展現更穩定且較低的驗證損失，並在無條件生成與續奏任務中優於僅用小型或未精煉資料訓練的模型，證明資料集的多樣性與精煉對模型泛化有正面影響。

**Conclusion:** PianoCoRe提供一個結合多個公開鋼琴MIDI資料庫、經過去重、品質評估與音符對齊精煉的大規模資料集，支援從大規模分析、預訓練到精細表現建模的多種應用。透過MIDI品質分類器與RAScoP對齊精煉管線，提升資料品質與對齊準確度，促進更穩健的表現渲染模型訓練。資料集遵循歐盟公有領域規範，並以CC-BY-NC-SA 4.0授權公開，促進可重現研究與跨資料集的非重疊分割。限制包括資料偏向西方古典音樂、部分樂譜與對齊可能含誤差，以及插值方法未涵蓋延音踏板效果。未來可擴展至多樂器資料與更細緻標註。

**Research Gap:** 本論文未明確說明對非西方音樂風格或當代作曲家的涵蓋度，資料偏向西方古典可能限制泛化。對齊精煉雖提升時間一致性，但未進行人類標註的精確評估，對齊錯誤與插值可能影響下游模型表現。MIDI品質分類器依賴自動標籤與合成數據，真實損壞樣本較少，可能影響分類器泛化。插值未處理延音踏板，對表現細節捕捉有限。實驗中缺乏主觀聽覺評測，無法驗證客觀指標提升是否轉化為感知改善。資料集規模龐大，部署與訓練成本未討論，可能對工程實務帶來挑戰。未提供對失敗案例或模型在極端表現風格下的穩健性分析。

### 2. BAMI: Training-Free Bias Mitigation in GUI Grounding

- 分數：7
- arXiv: [2605.06664v1](http://arxiv.org/abs/2605.06664v1)
- 發表日期：2026-05-07
- 分類：cs.CV, cs.AI
- 入選理由：+2 提及頂級機構：mit; +3 提及頂級會議：cvpr; +2 提及程式碼可用

**Intro:** GUI grounding 是實現 GUI 代理執行點擊、拖曳等操作的核心能力，但在複雜場景如 ScreenSpot-Pro 基準測試中，現有模型表現不佳。該任務需在高解析度且元素密集的介面中，根據自然語言指令準確定位目標元素座標，挑戰在於精度和語義辨識。本文提出的 Masked Prediction Distribution (MPD) 方法用於分析模型錯誤來源，發現主要問題為精度偏差與歧義偏差。基於此，作者設計了 Bias-Aware Manipulation Inference (BAMI) 方法，透過兩階段的結構化推理來減輕這些偏差，且無需額外訓練即可提升多種基礎模型的準確率。

**Motivation:** 現有 GUI grounding 模型在高解析度且複雜介面中容易出現定位誤差，主要因為模型在座標預測時存在精度限制與對相似元素的歧義判斷。這些偏差導致模型無法充分發揮潛力，且傳統透過語言擴充或上下文增強的推理方法效果有限。從工程與產品角度看，提升推理階段的定位精度且不需重新訓練模型，能大幅降低部署成本並快速提升現有系統性能，對實際應用價值極大。

**Method:** BAMI 方法包含兩大核心操作：首先是針對精度偏差的粗到細聚焦策略，透過多次迭代裁剪圖像並在縮小範圍內重新定位，逐步提升座標精度；其次是針對歧義偏差的候選框選擇策略，通過多輪遮罩生成多個候選框，並利用外部校正模型（如 GPT-5 或本地訓練的 Qwen3-VL-8B）根據預設的 GUI 功能性、互動性等規則選擇最合適的候選框。整體流程不需額外訓練，僅在推理階段結合結構化操作與外部知識模型完成。

**Result:** 在 ScreenSpot-Pro 複雜基準上，BAMI 對 TianXi-Action-7B 模型的準確率從 51.9% 提升至 57.8%，在多個開源基礎模型（如 OS-Atlas-7B、UI-TARS-7B）上均有穩定提升。消融實驗證實粗到細聚焦與候選框選擇兩個模組均獨立帶來顯著效果。參數分析顯示兩次迭代與約 50% 的裁剪比例為最佳平衡。校正模型選擇方面，GPT-5 與本地 Qwen3-VL-8B 均能有效提升性能，後者在隱私保護與獨立部署方面具優勢。實驗均在單張 RTX 4090 GPU 上完成，展示方法的實用性與效率。

**Conclusion:** 本文通過 MPD 方法系統診斷 GUI grounding 模型錯誤，發現大部分錯誤源於精度與歧義偏差，並提出 BAMI 結構化推理框架以無需訓練的方式有效緩解這些偏差。實驗結果表明 BAMI 在多模型、多數據集上均能穩定提升定位準確率，且可結合多種校正模型實現靈活部署。該方法為提升 GUI grounding 任務中模型推理能力提供了成本低且效果明顯的解決方案，對工程實踐具有重要參考價值。

**Research Gap:** 全文未明確說明 BAMI 在更大規模或多樣化 GUI 類型（如移動端、跨平台）上的泛化能力，且對於極端複雜介面或極小目標的表現仍缺乏深入分析。候選框選擇依賴外部校正模型，可能帶來額外計算成本與部署複雜度，且對校正模型的依賴程度及失效情況未充分探討。消融實驗主要聚焦於參數調整，對於不同類型偏差的交互影響及長迭代次數的潛在副作用缺乏詳細說明。此外，對於知識缺口類錯誤的處理未提出具體方案，限制了方法的全面性。未見對失敗案例的定量分析，無法全面評估方法的魯棒性與局限。

### 3. When No Benchmark Exists: Validating Comparative LLM Safety Scoring Without Ground-Truth Labels

- 分數：7
- arXiv: [2605.06652v1](http://arxiv.org/abs/2605.06652v1)
- 發表日期：2026-05-07
- 分類：cs.LG, cs.AI, cs.CL
- 入選理由：+2 提及頂級機構：mit; +3 收錄於 Hugging Face Daily Papers; +2 提及程式碼可用

**Intro:** 本論文針對在缺乏標註基準資料的情況下，如何比較大型語言模型（LLM）安全性評分的問題提出解決方案。許多實際部署場景中，特定語言、領域或法規尚無合適的安全基準，導致無法直接以標註資料驗證模型安全性。作者將此問題形式化為無基準比較安全評分，並提出一套工具與驗證鏈條以確保評分的有效性。此方法特別適用於低資源語言及受限環境，並以挪威公共部門的案例作為實證。論文同時提供開源工具SimpleAudit，支持本地化、可重複的安全評估。

**Motivation:** 現有的安全評估多依賴靜態基準資料，但這些基準通常昂貴且多為英語優先，且無法隨模型更新重複使用。自動化紅隊測試與審計工具雖能發現潛在問題，但缺乏可量化、可比較的分數，且無法提供明確的不確定性度量。實務中，尤其是公共部門需在無標註基準的條件下，做出可辯護的模型安全比較決策，且需本地執行以符合法規要求。這些需求促使作者提出一套可在無基準情況下，透過固定場景包、評分規則及審計流程，產生可重複且具比較意義的安全分數。

**Method:** 作者定義三個獨立角色：目標模型（Target）、審計員（Auditor）與評分員（Judge），並以固定的場景包與評分規則構成評分儀器。核心驗證鏈包含三個條件：對已知安全差異（安全模型與能力相當但拒絕行為被破壞的模型）有響應能力；分數變異主要由目標模型驅動而非審計員或評分員；分數在多次重複執行中穩定。SimpleAudit作為此方法的實作，支援本地執行、多角色替換、明確配置與不確定性報告，並提供完整的評分輸出與對應的統計分析。

**Result:** 在挪威語安全場景包上，SimpleAudit能以AUROC 0.89至1.00準確區分安全與破壞模型，目標模型為主要變異來源（η2約0.52），且分數在10次重複執行後穩定。審計員與評分員的選擇對分數有影響，審計員能力需與目標模型匹配以避免過度壓制安全模型分數。將相同驗證鏈應用於Petri工具，證明該鏈條具通用性。挪威公共部門採購案例中，Borealis與Gemma 3模型比較顯示安全性依場景類別與風險指標而異，強調必須報告完整分數組合而非單一排名。

**Conclusion:** 本論文提出無基準比較安全評分作為一種獨立的評估類別，並定義其聲明契約與驗證鏈。SimpleAudit作為該類別的參考實作，提供本地化、可重複且具不確定性報告的安全評分工具，適合低資源及受限部署環境。該方法強調評分工具必須對目標模型行為敏感、排除審計與評分員干擾，並在多次執行中穩定。此方法不取代領域專家判斷與標註基準，而是作為尚無基準時的過渡方案。驗證鏈亦可用於其他工具，確保評分的可比較性與可信度。

**Research Gap:** 論文未明確說明驗證鏈對於更廣泛安全行為（非僅拒絕行為差異）的適用性，且未涵蓋評估對抗性或eval-awareness問題的完整解決方案。實驗僅限於挪威語及特定模型族群，泛化性尚待驗證。審計員能力選擇對結果影響大，如何自動或系統化選擇合適審計員仍未解決。缺乏對不同場景包寬度與抽象層級對評分結果影響的深入分析。消融實驗與跨語言、跨領域的比較不足，且未充分探討部署成本與運算資源需求。最後，對失敗案例或評分不穩定情況的討論有限，實務應用中仍需領域專家介入判斷。

### 4. Revisiting Uncertainty: On Evidential Learning for Partially Relevant Video Retrieval

- 分數：7
- arXiv: [2605.06083v1](http://arxiv.org/abs/2605.06083v1)
- 發表日期：2026-05-07
- 分類：cs.CV, cs.IR, cs.LG, cs.MM
- 入選理由：+2 提及頂級機構：mila; +3 提及頂級會議：icml; +2 提及程式碼可用

**Intro:** 部分相關影片檢索（PRVR）旨在使用只描述部分內容的文字查詢來檢索未剪輯的影片。由於文字查詢通常簡短且模糊，與豐富的影片內容存在本質上的非對稱性，導致檢索過程中存在不確定性。這種不確定性在影片間語義歧義和影片內稀疏的時間監督兩個層面上尤為突出，前者使得查詢可能對多個影片產生模糊匹配，後者則限制了模型學習全面表示的能力。本文提出Holmes，一種分層的證據學習框架，通過多粒度跨模態證據聚合來明確量化和建模這些不確定性。

**Motivation:** 現有PRVR方法多忽略查詢的語義歧義，且多數依賴多實例學習（MIL）只監督最佳匹配片段，導致監督信號稀疏且模型易受局部噪聲影響。這使得模型在面對模糊或多義查詢時表現不佳，且無法有效利用影片中豐富的時間信息。為了提升檢索的準確性和可靠性，必須明確建模查詢的不確定性並提供更密集的時間監督，這對工程實踐中提升系統穩健性和用戶信任度具有重要意義。

**Method:** Holmes框架包含兩個層面的證據學習：在影片間層面，將跨模態相似度視為基於Dirichlet分布的證據，並根據三重原則（認知不確定性、標籤一致性和隨機不確定性）對查詢進行細粒度識別，進而實施查詢自適應的標籤校準；在影片內層面，通過引入帶有自適應dustbin桶的柔性最優傳輸（Flexible Optimal Transport）方法，建立軟性的查詢與多個片段對齊，從而緩解稀疏監督並抑制虛假局部響應。整體訓練結合基礎相似度損失、多樣性損失及兩層證據學習損失，採用兩階段優化策略。

**Result:** 在ActivityNet Captions、Charades-STA和TVR三個基準數據集上，Holmes在多項召回率指標（R@1、R@5、R@10、R@100）及召回率總和（SumR）上均超越多個先進PRVR方法，並且在效率上與同規模模型相當。消融實驗證明多尺度證據學習、查詢標籤校準、柔性最優傳輸及影片內證據學習均對性能提升有顯著貢獻。定性分析顯示Holmes能有效區分精確、模糊及多義查詢，並抑制MIL引起的虛假激活，提升檢索的可靠性。

**Conclusion:** 本文提出的Holmes框架通過分層證據學習明確建模PRVR中的不確定性問題，結合三重原則實現查詢細粒度識別與自適應標籤校準，並利用柔性最優傳輸累積密集時間監督，顯著提升了部分相關影片檢索的準確性和穩健性。該方法在多個公開數據集上取得了最先進的結果，並展示了良好的效率。限制方面，Holmes依賴預訓練特徵提取器且需完整影片輸入，對於線上流媒體檢索場景適用性有限。未來可考慮端到端訓練及更先進編碼器以進一步提升性能。

**Research Gap:** 本文雖然在多數公開數據集上驗證了Holmes的有效性，但全文未明確說明其在更大規模、更多樣化或實時檢索場景下的泛化能力和效率表現。此外，對於不同類型查詢（如極端模糊或多義查詢）的失敗案例分析較少，缺乏對模型在極端不確定性條件下的魯棒性評估。消融實驗中對柔性最優傳輸的參數敏感性及計算成本未充分探討，可能影響實際部署。最後，Holmes依賴預訓練特徵，限制了端到端優化的可能性，未來可進一步研究整合更強大特徵學習模組以提升整體性能和適應性。

### 5. Retrieval-Conditioned Topology Selection with Provable Budget Conservation for Multi-Agent Code Generation

- arXiv: [2605.05657v1](http://arxiv.org/abs/2605.05657v1)

**Intro:** 本論文聚焦於多代理大型語言模型（LLM）系統在程式碼生成任務中的拓撲路由問題。現有系統在選擇代理協調拓撲時，未充分利用程式碼庫的結構資訊，導致路由決策與實際程式碼複雜度不匹配。作者提出一種基於檢索的自適應協調架構（RGAO），透過從分層程式碼索引中提取結構複雜度向量，動態選擇最適合的協調拓撲。該方法整合了複雜度條件路由與形式化資源代數，實現了可證明的預算守恆性，提升了多代理系統的資源管理與效率。論文同時展示了系統架構、理論證明及實驗評估結果。

**Motivation:** 多代理LLM系統在處理跨模組或大型程式碼庫時，協調拓撲的選擇對性能與資源消耗影響甚大。然而現有方法多依賴查詢文本特徵或靜態規則，忽略了程式碼本身的結構複雜度，導致路由錯誤率高且資源使用無法有效控制。此問題在工程實踐中會造成效率低下與成本浪費，尤其在資源有限或需嚴格合規的環境下更為關鍵。作者透過引入程式碼結構檢索信號，實現動態且可驗證的拓撲選擇，為多代理系統的工程落地提供了更精確且安全的資源管理方案。

**Method:** 核心方法為Retrieval-Guided Adaptive Orchestration（RGAO），其流程包括：首先利用分層程式碼檢索引擎，結合多種檢索策略（如LATTICE校準、KohakuRAG多查詢重組、RepoGraph依賴擴展）獲取程式碼上下文及結構複雜度向量c，包含依賴深度、檔案數、符號數、樹深度及跨模組耦合度等五維指標。接著，基於這些指標與查詢，透過一組經過調參的閾值規則，決定四種協調拓撲之一（FASTPATH、SUBAGENT、MULTIAGENT、DEEPRESEARCH）。系統中每個子代理由形式化⟨I,C,T,M⟩合約管理，包含六維資源預算向量。提出一種靜態預執行的預算代數驗證方法，確保子代理預算總和不超過父代理，並在執行前以O(|V|+|E|)時間複雜度驗證整體預算守恆性，避免執行過度消耗資源的拓撲。

**Result:** 實驗使用250個標註路由任務集，RGAO將路由錯誤率從30.1%顯著降低至8.2%，統計檢定顯示p值小於10^-6，顯著優於基於正則表達式的基線方法。系統在200個檔案、約2000節點的樹狀索引上，完成索引構建時間約11.1毫秒，DAG構建時間低於0.01毫秒，預算檢查開銷約0.65微秒，均遠低於LLM推理延遲，顯示系統具備實際部署的低延遲特性。多步驟複合管線測試中，CODE-AGENT在包含介入恢復的五種典型多代理流程中達成100%成功率，且平均每管線約6000個tokens，較單一代理系統大幅節省資源。代理系統在代理間通信成本及檢索效率方面也展現優勢。

**Conclusion:** 本論文提出的RGAO架構成功結合了基於程式碼結構的動態拓撲選擇與靜態預算守恆驗證，實現了多代理程式碼生成系統中資源使用的可控與高效。實驗結果證明該方法在路由準確率、延遲及資源消耗方面均有顯著提升，且理論證明提供了嚴謹的資源保證。系統設計兼顧工程實踐需求，具備良好的擴展性與安全性。限制包括目前路由器為手動調參規則，預算守恆假設工具成本確定且檢索深度有限，且實驗尚未涵蓋所有程式語言及大規模真實場景。未來工作將聚焦於端到端真實評測、引入更多複雜度指標及學習式路由器。

**Research Gap:** 本論文雖提出靜態預算守恆驗證與基於程式碼結構的路由，但路由器仍為基於閾值的手工規則，缺乏學習能力，可能限制泛化與適應性。預算守恆理論依賴工具成本確定與有限檢索深度假設，對於實際中存在的非確定性與更複雜檢索場景，保證效果未明。實驗主要基於代理模擬與代理測試集，缺乏完整的SWE-bench Verified及Pro真實評測，且對多語言及異構代碼庫的適用性未充分驗證。系統對失敗案例、分布外語言及大規模部署的穩健性分析不足。此外，成本與部署風險評估未深入，未明確說明在資源受限環境下的性能表現與調優策略。未來需補充這些方面以提升系統完整性與實用價值。
