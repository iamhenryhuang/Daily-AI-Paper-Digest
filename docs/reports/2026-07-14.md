### 1. A Sovereign, Open-Source Foundation Model for German and English

- 分數：7
- arXiv: [2607.09424v1](http://arxiv.org/abs/2607.09424v1)
- 發表日期：2026-07-10
- 分類：cs.CL, cs.AI, cs.LG
- 入選理由：+2 提及頂級機構：mit; +3 收錄於 Hugging Face Daily Papers; +2 提及程式碼可用; LLM rank #1 / 9/10: Presents a novel, large-scale open-source Mixture-of-Experts foundation model specialized for German and English with strong benchmark performance and transparent release of data and code, demonstrating clear practical a

**Intro:** 本論文提出 Soofi S 30B-A3B，一款專為德語與英語設計的主權開源基礎模型。該模型採用混合 Mixture-of-Experts (MoE) 的 Mamba Transformer 架構，僅在推理時啟用約3B參數，顯著降低推理成本。訓練資料涵蓋約27兆個tokens，特別加重德語比例，目標是達成德英雙語的高效能與高透明度。模型在多項英德語基準測試中表現優異，並在歐洲主權模型中領先。此模型於德國工業AI雲端基礎設施上訓練，並將以高度開放的條款釋出完整訓練與評估資源。

**Motivation:** 現有開源模型多數僅釋出權重，缺乏完整訓練資料與流程透明度，難以複現與審計。多語言模型往往英語占比過重，德語資源不足，無法滿足歐洲語言主權需求。此外，密集模型在長上下文與高併發推理時，因注意力快取記憶體帶寬限制，推理效率大幅下降。Soofi S 針對這三大問題提出解決方案，兼顧開放性、德英雙語深度訓練與長上下文高效推理，對工程部署與產品落地具實質意義。

**Method:** Soofi S 採用 Nemotron 3 Nano 30B-A3B 混合架構，包含52層：23層 Mamba-2序列混合層、23層 MoE稀疏專家層與6層 Grouped-Query Attention層。每個token推理時僅啟用約3.2B參數，且只有6層維護KV快取，快取大小與上下文長度幾乎無關。訓練分三階段：第一階段多元預訓練，第二階段高品質資料退火，第三階段長上下文擴展至1M tokens。資料來源涵蓋高品質網頁、程式碼、數學、德語專屬資料及合成資料，並公開完整資料來源與token計數。訓練使用AdamW優化器與Warmup-Stable-Decay學習率調度，部署於德國工業AI雲端，確保主權與合規。

**Result:** Soofi S 在17個公開基礎模型中，英德語整體能力與程式碼生成表現最佳，尤其在德語基準上超越所有歐洲主權模型，且在英語與德語整體評測中達到14至27B密集模型水準。與同架構Nemotron 3 Nano相比，德語能力提升顯著，英語能力亦有所增強。推理效率方面，於40K上下文與批次32時，Soofi S的解碼吞吐率為4.82k TPS/GPU，約為同等密集模型的8至9倍，且吞吐率隨上下文長度增加幾乎不變，適合長上下文高併發生產環境。程式碼生成、數學、知識與推理等多項任務均展現競爭力，且公開完整訓練流程與資料透明度。

**Conclusion:** Soofi S 成功結合主權要求、開源透明、德英雙語深度訓練與長上下文高效推理，成為首款在活躍參數效率上與國際頂尖公開模型相當的歐洲主權基礎模型。其混合MoE架構與嚴謹資料策略，實現了推理成本大幅降低且能力不減。完整釋出訓練資料來源、超參數與程式碼，促進社群審計與後續擴展。限制在於德語競賽數學能力仍落後頂尖密集模型，開放域事實回憶能力受限於活躍參數規模，且全文未明確說明模型在更多下游任務與真實應用場景的泛化表現。

**Research Gap:** 全文未明確說明模型在多樣化下游任務與實際應用場景的泛化與穩定性，缺乏失敗案例與錯誤分析。德語競賽數學能力仍顯不足，顯示資料或模型容量分配仍有優化空間。開放域知識回憶受限於3B活躍參數，未探討結合檢索等增強策略。雖有長上下文擴展訓練，但對極長上下文的實際推理效果與記憶保持未深入評估。消融實驗主要聚焦資料配方，對架構細節與稀疏路由穩定性缺乏更細緻比較。部署成本與硬體需求雖有描述，但缺少對中小型機構可行性的分析。

### 2. Self-Guided Test-Time Training for Long-Context LLMs

- 分數：6
- arXiv: [2607.09415v1](http://arxiv.org/abs/2607.09415v1)
- 發表日期：2026-07-10
- 分類：cs.CL, cs.AI
- 入選理由：+3 收錄於 Hugging Face Daily Papers; +3 提及頂級會議：acl; LLM rank #2 / 8/10: Proposes a novel, effective method (Self-Guided Test-Time Training) to improve long-context LLM reasoning with credible evidence on challenging benchmarks, balancing novelty and practical usefulness.

**Intro:** 長上下文處理對大型語言模型（LLMs）越來越重要，但單純延長上下文視窗並不保證能有效利用長輸入。隨著輸入長度增加，模型準確度往往下降，顯示模型難以識別並使用與問題最相關的證據。測試時訓練（TTT）是一種有前景的解決方案，通過將測試上下文視為訓練樣本，進行實例特定的參數調整。然而，對整個長上下文應用TTT計算成本過高，隨機採樣片段則引入大量噪聲，因為大多數片段與問題無關，甚至可能降低模型性能。

**Motivation:** 作者的初步研究發現TTT對訓練片段質量高度敏感：在LongBench-v2上，隨機片段TTT會降低性能，而使用帶有oracle標註的片段則顯著提升。這說明選擇合適的訓練數據是長上下文TTT的核心瓶頸。基於此，提出Self-Guided TTT（S-TTT），讓模型在適應前自我識別應該學習的證據片段，僅對這些片段應用標準語言模型訓練目標。這種方法兼顧了計算效率與訓練信號質量，對工程和產品部署具有實用價值。

**Method:** S-TTT包含兩個階段：第一階段，模型基於問題和完整上下文，自行標註最相關的連續證據片段；第二階段，從基模型複製一份參數，僅對選中的片段進行多步次的下一詞預測訓練，更新模型參數。最終，使用更新後的模型在完整上下文和問題條件下生成答案。該方法不改變模型架構或解碼流程，僅優化測試時的訓練數據選擇，並在無oracle標註的情況下自動選擇高質量訓練片段。

**Result:** 在LongBench-v2和LongBench-Pro兩個長上下文推理基準上，使用Qwen3-4B-Thinking-2507和Llama-3.1-8B-Instruct兩款模型，S-TTT均穩定提升性能，尤其在長上下文（64k至128k tokens）區間，提升相對明顯。與隨機片段TTT、全上下文TTT、LongLLMLingua壓縮方法及QRHead Span TTT等多種基線比較，S-TTT在大多數情況下表現最佳或相當。分析顯示，模型自我標註的片段優於基於內在不確定性指標（如困惑度、熵）選擇的片段。效率方面，S-TTT在長上下文時的整體延遲低於其他非凍結KV緩存的TTT方法。

**Conclusion:** 本文提出的Self-Guided TTT通過模型自身選擇問題相關的證據片段進行測試時訓練，有效解決了長上下文TTT中訓練數據質量低下的問題。實驗證明該方法在多種模型和長上下文基準上均帶來穩定性能提升，且在長上下文場景下計算效率優於多數現有TTT方法。S-TTT保持了模型架構和解碼流程不變，便於實際部署，為長上下文推理提供了一種簡潔且有效的測試時適應策略。

**Research Gap:** 本文未明確說明模型自我標註片段的失敗案例及其對性能的影響，對於標註失敗時的fallback機制僅在附錄中提及，缺乏深入分析。實驗中僅使用兩款中等規模模型，尚未驗證方法在更大規模模型或不同語言、領域的泛化能力。消融實驗主要聚焦於片段選擇策略，對於不同訓練步數、學習率等超參數的敏感性分析不足。此外，S-TTT在極端長上下文（超過128k tokens）或多輪交互場景下的性能和效率表現未展開討論。最後，方法依賴模型自身判斷相關片段，若模型本身推理能力有限，可能導致選擇偏差，這部分風險和限制未充分揭示。

### 3. Terminal Dimension Reduction for Time Series with Applications

- 分數：5
- arXiv: [2607.09490v1](http://arxiv.org/abs/2607.09490v1)
- 發表日期：2026-07-10
- 分類：cs.DS, cs.CG, cs.LG, stat.ML
- 入選理由：+2 提及頂級機構：mila; +3 提及頂級會議：icml; LLM rank #3 / 7/10: Introduces a theoretically grounded generalization of terminal embeddings for time-series clustering with strong methodological novelty and practical coreset applications, supported by experiments.

**Intro:** 時間序列資料在感測網路、氣象系統、股市等領域廣泛存在，且通常為高維度且包含多重時間序列。傳統聚類演算法多針對向量資料設計，難以直接處理時間序列的非同步與缺失問題。理論計算機科學提出以多邊形曲線表示時間序列，並利用Fréchet距離進行聚類，但高維度時間序列的維度縮減仍是挑戰。終端嵌入（terminal embeddings）是近年來強大的維度縮減工具，能保證任意終端點與查詢點間距離近似保持，但尚未擴展到包含線性結構的時間序列聚類問題。

**Motivation:** 現有的終端嵌入無法線性表示，因而難以保留時間序列中以線段連接的線性結構，限制了其在時間序列聚類中的應用。維度縮減對於提升聚類演算法的運算效率、降低記憶體需求及樣本複雜度至關重要。若能設計保留Fréchet距離的終端嵌入，將可實現無維度依賴的coreset構造，進一步推動高維時間序列聚類的理論與實務發展。

**Method:** 本論文提出一種將終端嵌入擴展至仿射線段與多邊形曲線的框架，核心在於結合Johnson-Lindenstrauss隨機投影與資料依賴的正交投影，將輸入曲線與查詢曲線分別映射至低維空間。透過構造覆蓋集與分段線性映射，保證對輸入曲線中任意點與查詢曲線中任意線段上的點間距離的近似保持。此方法避免了過去需解半定規劃或最近鄰查詢的複雜步驟，並且可用於構造尺寸與維度無關的coreset，提升聚類演算法的可擴展性。

**Result:** 理論上證明存在目標維度為O(ℓε^{-4} log(nm))的終端嵌入，可保證Fréchet距離以(1±ε)誤差保持。基於此嵌入，構造出尺寸為˜O(kε^{-2-z}ℓ^{2}log^{2}m)的coreset，達到在k與ε上的最佳依賴，且不再依賴原始維度d。實驗部分以實際溫室氣體濃度時間序列與合成資料驗證，結果顯示終端嵌入在保持Fréchet距離近似上與JL投影表現相近，且優於PCA，且終端嵌入能擴展至任意查詢曲線，展現其理論優勢。

**Conclusion:** 本論文成功將終端嵌入技術推廣至包含線性結構的時間序列多邊形曲線，實現了Fréchet距離的維度縮減與coreset構造，為高維時間序列聚類提供了理論與實務上的新工具。方法避免了過去複雜的優化程序，且理論保證嚴謹，實驗結果支持其有效性。限制在於目標維度與coreset尺寸對精度ε的依賴仍較高，且實際部署時需考慮計算成本與查詢曲線預處理時間。

**Research Gap:** 論文未明確說明終端嵌入在極高維度或極大規模時間序列資料上的實際運算效率與記憶體消耗，尤其查詢曲線映射的預處理時間較長。實驗僅涵蓋部分資料集與較低維度，泛化性與在多樣化真實場景下的表現尚待驗證。方法對於非線性時間序列結構或非多邊形曲線的適用性未探討。此外，對於ε依賴的提升空間、對失敗案例或近似誤差累積的分析不足，未提供消融實驗比較不同構造細節對結果的影響。最後，部署於實際系統時的成本與風險評估尚缺乏。

### 4. Robustifying Vision-Language Models via Test-Time Prompt Adaptation

- 分數：5
- arXiv: [2607.09450v1](http://arxiv.org/abs/2607.09450v1)
- 發表日期：2026-07-10
- 分類：cs.CV, cs.LG
- 入選理由：+2 提及頂級機構：mit; +3 提及頂級會議：icml; LLM rank #4 / 7/10: Presents a robust test-time prompt adaptation method for vision-language models that improves adversarial robustness with a novel distribution-level alignment approach and strong empirical validation.

**Intro:** 本文針對預訓練的視覺語言模型（VLMs），如CLIP，在面對對抗性擾動時性能急劇下降的問題進行研究。現有的測試時適應方法多依賴於樣本層級的置信度啟發式，忽略了數據的內在分布結構，導致無法有效區分對抗性錯誤與真實語義一致性。作者觀察到對抗擾動在結構上較脆弱，雖然整體表示被破壞，但增強視圖的分布中仍保留語義完整性。基於此，提出一種基於分布層級對齊的測試時提示適應框架RITA。

**Motivation:** 對抗擾動會嚴重影響VLM的零樣本泛化能力，這對安全敏感的實際應用構成威脅。現有方法如對抗訓練成本高且需標註數據，測試時提示調整則多忽略視覺增強視圖的分布結構，限制了防禦效果。作者發現利用多視圖的分布結構能更好地保持語義一致性，並且通過分布層級的對齊能有效減輕對抗樣本的影響，提升模型的穩健性和實時適應能力。

**Method:** RITA核心在於將對抗圖像的多個增強視圖和多個文本提示原型分別建模為離散分布，並利用熵正則化的Optimal Transport（OT）方法進行分布層級的對齊。此方法能夠緩解對抗樣本中的異常點，修正跨模態語義錯配。此外，RITA引入動態緩存機制，根據置信度篩選低熵的增強視圖，將其累積於緩存中，並在線上不斷優化分布對齊，實現持續的測試時適應。整個過程僅更新提示參數，不改變模型權重。

**Result:** 在八個細粒度分類數據集及ImageNet及其變體上，RITA在多種對抗攻擊（PGD、CW、DI）下均顯著提升了模型的對抗魯棒性，平均提升幅度達約45%至50%，同時保持或略微提升乾淨數據的準確率。與多個先進的測試時適應方法（如TPT、R-TPT）比較，RITA在多數場景下均取得更優結果。結合對抗訓練的CLIP模型，RITA仍能進一步提升魯棒性。實驗還展示了動態緩存和增強視圖數量對性能的影響，並證明了方法在效率和準確率間取得良好平衡。

**Conclusion:** 本文提出的RITA框架通過分布層級的Optimal Transport對齊和動態緩存機制，有效提升了預訓練VLM在對抗擾動下的穩健性，且無需重新訓練或標註數據。實驗結果表明該方法在多種數據集、攻擊類型和模型架構上均具備良好的泛化能力和實時適應性。限制方面，本文主要聚焦於分類任務，對生成式任務如圖像描述的適用性尚未探討，且全文未明確說明在更大規模或更複雜場景下的部署成本與失敗案例。

**Research Gap:** 本文雖然在多個分類數據集上驗證了方法有效性，但缺乏對生成式任務的實驗驗證，限制了方法的應用範圍。動態緩存的更新策略和容量對長期推理場景的影響未充分探討，可能存在累積錯誤或記憶衰減問題。對抗攻擊類型雖多樣，但對更複雜或黑盒攻擊的防禦能力未明確評估。實驗中對計算資源消耗和延遲的分析有限，對實際產品部署的成本與風險評估不足。此外，方法對不同視覺語言模型架構的泛化性和潛在失效場景未充分說明，這些均是未來改進的方向。

### 5. SVF-CR: Synchronized Visual-Facial Cross-Refinement for Multimodal Ambivalence and Hesitancy Recognition

- 分數：5
- arXiv: [2607.09417v1](http://arxiv.org/abs/2607.09417v1)
- 發表日期：2026-07-10
