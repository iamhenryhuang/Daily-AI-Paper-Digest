### 1. How Good Can Linear Models Be for Time-Series Forecasting?

- 分數：0
- arXiv: [2606.27282v1](http://arxiv.org/abs/2606.27282v1)
- 發表日期：2026-06-25
- 分類：cs.LG
- 入選理由：LLM rank #1 / 9/10: Provides a strong empirical and theoretical study challenging current trends in time-series forecasting, showing that careful preprocessing and hyperparameter tuning with simple linear models can rival large architecture

**Intro:** 時間序列預測領域近年來多聚焦於大型模型架構，如專門設計的Transformer及通用基礎模型，普遍認為模型容量是提升準確度的關鍵。本文則提出相反觀點，認為透過調整前處理流程即可在低成本下大幅縮小性能差距。作者以Ridge回歸作為基礎模型，利用其封閉解與可解釋權重，系統性搜尋多項前處理超參數，並在八個標準資料集上進行評估。研究發現最佳的歷史窗口長度、局部正規化範圍及跨序列共享策略皆具資料集特異性，且優化後的線性模型在多數情況下超越先前線性方法及多種非線性基準。此研究強調前處理調整的重要性，並將超參數優化視為資料特性診斷工具。

**Motivation:** 現有時間序列預測研究多聚焦於模型架構設計，忽略了前處理設定對於低容量模型的影響。大型模型因具備強大表達能力，能部分彌補不佳的輸入表示，但線性模型無法，導致其表現被低估。若能針對前處理進行細緻調整，尤其是歷史窗口長度、正規化範圍及資料增強，則可在不增加模型複雜度的前提下提升預測準確度。這對工程與產品層面意義重大，因為線性模型訓練成本低、解釋性強，適合資源有限或需快速部署的場景。作者希望透過系統性搜尋前處理超參數，揭示資料內在結構並提升線性模型競爭力。

**Method:** 核心方法是以Ridge回歸作為預測模型，利用其封閉解快速訓練並取得可解釋權重。前處理流程包含四個主要軸向：歷史窗口長度（context length）、局部正規化範圍（local normalization，透過學習的尾部比例r決定）、正規化方法（標準化或魯棒正規化）、以及資料增強（時間域或頻率域的高斯噪聲）。作者提出一種分組超參數搜尋策略，允許在不同預測時段與序列間共享或獨立設定超參數，平衡搜尋成本與表現。搜尋過程採用Optuna的樹結構Parzen估計器，並以時間序列擴展窗口交叉驗證評估候選組合。最終選擇的超參數用於測試集評估。

**Result:** 在八個多變量時間序列資料集（如ETT系列、Weather、Electricity、Traffic、Exchange）上，作者的SearchCast方法在七個資料集優於所有線性基準，並在六個資料集超越多種非線性模型（Transformer、MLP、CNN）。實驗涵蓋多個預測時段（96、192、336、720步），結果顯示調整歷史窗口長度與局部正規化比例能帶來最高16%的MSE改善。分析發現不同資料集對歷史窗口長度與正規化範圍的需求差異大，部分資料集長期預測需較短歷史，反駁傳統認知。跨序列共享策略因資料異質性而異，部分資料集適合完全共享，另一些則需獨立調整。資料增強在多數情況下被選用，且時間域與頻率域噪聲均有貢獻。

**Conclusion:** 本文證明線性模型在時間序列長期預測中，透過細緻的前處理超參數調整，能達到甚至超越多數先進非線性模型的準確度，且訓練成本低廉。優化過的超參數不僅提升預測性能，也揭示資料的內在結構特性，如非平穩性與序列異質性，為後續模型設計提供指引。作者釋出SearchCast管線，方便業界與研究者在新資料集上快速診斷與調整前處理。此研究強調在時間序列預測中，前處理策略的重要性不亞於模型架構創新。

**Research Gap:** 本文雖在多資料集與多預測時段驗證方法有效性，但仍有幾點不足。首先，搜尋策略雖考慮序列與時段分組，但未明確探討搜尋成本與效益的平衡，實際部署時可能面臨計算資源限制。其次，資料增強方法僅限於高斯噪聲，未評估其他增強技術對性能的影響。第三，模型僅針對單變量序列獨立建模，未充分利用多變量間的交互資訊，可能限制在高度相關序列上的表現。第四，全文未明確說明在極端非平穩或異常事件下模型的穩健性與泛化能力。最後，雖提出超參數作為資料診斷工具，但未展示如何將此診斷結果具體應用於更複雜模型的設計或調整。

### 2. Error-Conditioned Neural Solvers

- 分數：0
- arXiv: [2606.27354v1](http://arxiv.org/abs/2606.27354v1)
- 發表日期：2026-06-25
- 分類：cs.LG, cs.AI, cs.CV, math.NA
- 入選理由：LLM rank #2 / 9/10: Introduces a novel neural solver architecture that directly incorporates PDE residuals as inputs to iteratively correct predictions, demonstrating superior accuracy, robustness, and generalization across multiple PDE fam

**Intro:** 本論文探討神經網路在偏微分方程（PDE）求解上的應用，指出現有神經代理模型雖能快速預測，但多將求解視為純統計問題，無法有效修正自身違反物理約束的錯誤，且難以在訓練分布外泛化。近期混合方法透過優化PDE殘差以強化物理正確性，卻面臨計算成本高昂及數值不穩定等問題。作者理論與實驗證明，對病態系統而言，殘差最小化並非重建準確度的可靠指標，解釋了混合方法低殘差卻預測不準確的現象。論文提出一種新框架，稱為Error-Conditioned Neural Solvers（ENS），將PDE殘差作為網路輸入，讓模型學習讀取錯誤空間結構並迭代修正預測。此方法在多種PDE家族中展現優異準確度，且計算成本低於現有混合方法。

**Motivation:** 現有神經PDE求解器多缺乏自我錯誤評估與修正機制，導致在訓練分布外表現不佳，限制其工程與產品應用。混合方法雖引入物理約束優化，但因需執行昂貴的數值優化步驟，且在病態問題中優化目標與實際解的誤差不一致，造成準確度不足與不穩定。這些問題在高維大規模模擬中尤為嚴重，阻礙神經求解器在實際科學與工程場景的落地。論文動機在於設計一種既能利用物理約束資訊，又能避免高成本與不穩定性的神經求解器，提升泛化能力與推理效率。

**Method:** ENS框架包含一個預測器與一個迭代修正器。預測器產生初始解，修正器則在每次迭代中接收當前解與其PDE殘差場作為輸入，學習一個非線性更新策略以修正解。此設計使模型能直接讀取錯誤的空間結構，避免將殘差作為優化目標的缺陷。訓練時，模型在多個迭代步驟中均以重建誤差監督，且中間輸入梯度不反向傳播，確保修正器能適應推理時的誤差分布。推理時可執行超過訓練步數的迭代，直到殘差收斂。ENS在靜態PDE使用改良的FNO架構，在動態湍流問題使用Transformer架構。

**Result:** ENS在四種PDE家族（Helmholtz、Darcy、Poisson及Navier-Stokes）及多種挑戰性場景（分布內、超解析度、參數外推、跨方程轉移）中，普遍達成最低重建誤差且保持低殘差。特別在病態問題如湍流Kolmogorov流中，ENS相較於基線方法有高達10倍的誤差改善。與現有混合方法相比，ENS推理速度快數十至數百倍，且不受初始解品質影響，展現強健的收斂性與泛化能力。消融實驗證明殘差作為輸入是性能提升的關鍵，且ENS能在多次迭代中持續改善解的準確度。

**Conclusion:** 本論文提出的ENS框架透過將PDE殘差作為網路輸入，學習非線性修正策略，有效克服了現有混合方法中殘差最小化與解重建誤差不一致的問題。ENS在多種PDE及分布外場景中均展現優異準確度與計算效率，具備良好的初始化魯棒性與泛化能力。此方法為神經PDE求解器在工程與科學模擬中的實際應用提供了可行且高效的解決方案。限制在於實驗僅涵蓋較簡單的二維系統，且假設推理時已知完整方程，未探討三維問題及帶有噪聲或不完全方程的情況。

**Research Gap:** 論文未明確說明ENS在三維高維度PDE或更複雜物理場景中的表現與可擴展性，這對實際工程應用至關重要。對於殘差計算中可能存在的離散誤差或方程不完全情況，ENS的魯棒性尚未充分驗證。此外，雖有消融實驗，但對比其他可能的錯誤訊號或不同修正策略的效果較少，缺乏更全面的比較。推理時迭代步數與計算成本的權衡、失敗案例分析及在真實噪聲數據上的測試也未見詳述，限制了對其實際部署風險與泛化能力的評估。

### 3. Recovering Governing Equations from Solution Data: Identifiability Bounds for Linear and Nonlinear ODEs

- arXiv: [2606.27285v1](http://arxiv.org/abs/2606.27285v1)
- 發表日期：2026-06-25
- 分類：cs.LG, cs.IT, math.CA, math.DS
- 入選理由：LLM rank #3 / 8/10: Addresses a fundamental theoretical gap in scientific machine learning by rigorously establishing identifiability and sample complexity bounds for learning governing ODEs from solution data, providing a strong long-term 

**Intro:** 本論文探討從觀察到的解資料中恢復常微分方程（ODE）統治方程的理論基礎，特別是辨識性與樣本複雜度的定量分析。過去文獻多聚焦於演算法設計，對於何時能唯一且穩定地辨識真實ODE結構尚缺乏嚴謹理論。作者提出以Hausdorff距離衡量不同ODE解集合間的差異，作為比較微分方程的自然度量，並針對線性及非線性（Lipschitz與Hölder連續）ODE建立辨識界限。此研究填補了辨識性理論與樣本需求分析的空白，對科學機器學習中方程發現任務具基礎性意義。

**Motivation:** 辨識統治方程是科學機器學習中核心挑戰，尤其在物理建模與系統辨識中，能否從有限且帶噪的解資料中唯一且穩定地恢復方程結構，直接影響模型的可靠性與泛化能力。現有方法多依賴特定參數化或正則化策略，缺乏普適的理論保證。透過Hausdorff距離，作者將辨識問題形式化為解集合間的最壞情況分離，揭示辨識的本質限制與條件，並進一步分析學習所需的樣本數量，對工程實務中資料收集與模型訓練策略提供指引。

**Method:** 核心方法是定義ODE間的Hausdorff距離，衡量兩ODE在指定初始條件集合下解軌跡集合的最大最小距離。作者針對多類ODE結構（線性、Lipschitz、Hölder及多項式）推導Hausdorff距離與結構函數距離間的上下界，證明辨識性條件。透過狀態空間轉換，將高階ODE化為一階系統，延伸理論結果。進一步利用度量熵分析ODE類別的複雜度，並結合樣本生成、數值誤差與訓練誤差，建立辨識結構方程的樣本複雜度界限，提供理論依據設計基於Hausdorff距離的學習目標函數。

**Result:** 論文明確證明線性ODE在Hausdorff距離下的辨識性，且初始條件集需涵蓋狀態空間基底以保證唯一性。對Lipschitz ODE也給出類似上下界，但數值實驗顯示下界可能非最緊。對Hölder連續ODE，區分亞線性與超線性情況，並證明在適當時間範圍內仍可辨識。度量熵結果顯示線性ODE類別的複雜度與結構矩陣空間相當，Lipschitz ODE類別的複雜度與函數空間相當。樣本複雜度分析指出，為達到指定誤差，線性ODE需至少d個解軌跡樣本，Lipschitz ODE則需依維度與Lipschitz常數調整樣本數與時間解析度。數值實驗驗證理論界限的合理性。

**Conclusion:** 本研究建立了從解資料辨識線性及非線性ODE統治方程的理論框架，明確界定辨識性條件與穩定性，並首次量化了學習此類方程所需的樣本複雜度。透過Hausdorff距離作為解集合比較的度量，提供了辨識問題的最壞情況分析，對科學機器學習中方程發現的理論與實踐均有啟發。限制在於理論多依賴初始條件集的充分性與ODE結構假設，且對更複雜PDE或高維非線性系統的泛化尚未涵蓋。未來工作可針對辨識下界的緊致性、非自治系統及噪聲影響進行深入研究。

**Research Gap:** 本論文雖提供多類ODE的辨識界限與樣本複雜度分析，但仍存在幾項不足。首先，Lipschitz ODE的辨識下界非最緊，數值實驗顯示指數次方存在提升空間，理論證明尚未完全銳化。其次，辨識理論依賴初始條件集包含狀態空間基底，實務中此條件難以完全滿足，對有限且稀疏初始條件的辨識能力未充分探討。再者，對於非自治系統、帶噪聲資料及數值誤差的魯棒性分析不足，實際部署風險未明。最後，雖涵蓋部分非線性ODE類別，對高維複雜非線性系統及偏微分方程的理論延伸仍缺乏，限制了方法的廣泛應用。

### 4. Inherited Circuits, Learned Semantics: How Fine-Tuning Creates Evasion Vulnerabilities Invisible to Standard Evaluation

- 分數：0
- arXiv: [2606.27091v1](http://arxiv.org/abs/2606.27091v1)
- 發表日期：2026-06-25
- 分類：cs.CR, cs.AI
- 入選理由：LLM rank #4 / 8/10: Reveals subtle vulnerabilities introduced by fine-tuning in security-classification LLMs, supported by causal interventions and a novel evasion benchmark, with practical implications for robust AI security systems.

**Intro:** 本論文探討大型語言模型（LLMs）在進行安全分類任務時，透過微調所引入的隱藏漏洞。現有評估多以與訓練資料同分布的測試集為基準，無法揭露模型對行為不變轉換的脆弱性。研究聚焦於PowerShell腳本分類，指出微調後模型可能依賴特定指標詞彙，導致在別名替換、指令重構、字串建構、執行間接化及大小寫變異等行為不變轉換下失效。透過對比基礎模型Llama-3.1-8B-Instruct與其微調版本Foundation-Sec-8B-Instruct，揭示微調如何改變模型內部語義權重，並擴大攻擊面。

**Motivation:** 安全分類模型若僅依賴表面指標詞，容易被攻擊者利用行為不變的語法變換繞過，這在實務中會導致安全防護失效。微調雖提升準確率，但可能無意中強化了對特定指標的依賴，使模型在面對變異指令時表現不穩定。對工程與產品而言，這意味部署前的標準測試不足以保證模型的堅韌性，需有更細緻的評估與監控機制以防範潛在攻擊。

**Method:** 本研究採用因果干預技術定位分類迴路，發現模型在第12層注意力頭組成的晚期路徑承擔主要分類責任，該路徑為基礎模型繼承而非微調新建。微調集中並語義專化此路徑，形成對指標詞的強烈關聯。建立三層次逃避基準測試，涵蓋直接別名替換、指令重構、字串建構及大小寫變異等轉換，驗證微調模型在多種變異下的失誤。提出部署前監控方法，利用線性探針讀取分類邊界激活，結合指標詞消融的符號檢驗，辨識指標詞角色變化的指令家族，優先進行紅隊攻擊變異生成。

**Result:** 因果干預證實分類迴路為基礎模型繼承的第12層注意力頭組合，微調後該路徑權重集中且語義專化。逃避基準測試中，Foundation-Sec在Invoke-WebRequest別名替換（iwr）、Invoke-Expression格式字串重構及大小寫變異等多個技術上出現一致失誤，Llama則無此問題。線性探針在家族層級能有效監測微調後語義漂移，指標詞消融測試揭示Invoke-WebRequest指標詞角色由驅動變為抑制，預測該家族逃避風險。微調提升基線準確率但擴大了逃避攻擊面，且逃避失誤對提示詞敏感。

**Conclusion:** 本研究揭示安全微調雖提升模型在標準測試上的準確率，卻可能引入對指標詞過度依賴，導致行為不變轉換下的逃避漏洞。透過因果分析與跨模型比較，確認微調主要調整繼承迴路的語義權重，形成脆弱的指標規則。提出的監控方法可在部署前識別高風險指令家族，協助優先進行紅隊測試。此工作提醒從業者，微調非必然提升安全性，需結合全面轉換空間定義與語義漂移監控，才能建立更堅韌的AI安全系統。

**Research Gap:** 本研究雖以293對匹配腳本進行深入因果分析，但指標家族數量有限，且逃避基準中部分技術的樣本數較少，限制了結果的普遍性與穩定性。微調模型的逃避失誤對提示詞敏感，顯示語義層面變化複雜，尚未完全解析MLP層如何累積信號反轉。部署成本、實時監控效能及泛化至其他安全任務或模型尚未探討。未涵蓋更複雜的網路物件間接化與分階段載荷流程等理論逃避技術。未明確說明在更大規模或多樣化資料集上的表現，且缺乏對失敗案例的詳細分析。未提供針對微調過程中語義漂移的自動化調整或防禦策略。

### 5. Mostly Automatic Translation of Language Interpreters from C to Safe Rust

- 分數：0
- arXiv: [2606.27122v1](http://arxiv.org/abs/2606.27122v1)
- 發表日期：2026-06-25
- 分類：cs.PL, cs.MA, cs.SE
- 入選理由：LLM rank #5 / 8/10: Presents a mostly-automatic, scalable approach to translating C interpreters to safe Rust, validated on multiple real-world interpreters with strong test coverage and a security case study, balancing novelty and practica

**Intro:** 本論文針對將 C 語言編寫的語言直譯器轉譯為安全的 Rust 程式碼，提出一套名為 Reboot 的半自動化技術。由於 C 與 Rust 在型別約束、所有權及借用規則上差異甚大，直接轉譯困難且容易產生安全漏洞。語言直譯器因處理不受信任的輸入且常見記憶體漏洞，是重要的轉譯目標。Reboot 透過特徵簡化與多代理系統協調，成功將六個實際直譯器（6k 至 23k 行程式碼）轉譯為無 unsafe 區塊的 Rust，並通過完整測試。此技術在安全性、正確性及人力介入成本上展現良好表現。

**Motivation:** C 語言直譯器廣泛使用但缺乏記憶體安全保障，導致空間及時間記憶體錯誤頻繁，造成安全風險。Rust 提供強記憶體安全保證，轉譯 C 程式至 Rust 可提升安全性。現有轉譯工具多保留 C 程式結構，生成非慣用且仍含 unsafe 的 Rust 程式，或僅能處理函式層級的轉譯，無法有效處理跨函式、跨檔案的複雜特徵。因此需要一種能處理大型、複雜直譯器且能生成安全、慣用 Rust 程式的自動化方法，降低人力負擔並提升轉譯品質。

**Method:** Reboot 採用兩大核心設計：一是特徵簡化（feature reduction），將直譯器依語言特徵（如例外處理、正則表達式）逐步簡化，產生一系列可獨立測試的里程碑版本，從最簡單版本開始逐步恢復特徵並驗證，允許自由重構 Rust 程式以符合所有權規則。二是多代理系統架構，包含翻譯者、簡化者、驗證者、程式碼審查者及管理者等角色，透過自動化驗證與歷史回饋機制監控代理狀態，偵測並修正錯誤，僅在必要時請求使用者介入。整體流程分為規劃、特徵簡化及轉譯三階段，並以有限狀態機確保工作流程正確。

**Result:** Reboot 成功將六個開源直譯器（6k 至 23k 行）轉譯為安全 Rust，所有翻譯版本均通過 100% 提供測試，並在獨立驗證測試中達 62% 至 92% 通過率。轉譯時間介於 28 至 90 小時，成本約 460 至 1780 美元，且僅需 1 至 11 次使用者介入，每次約 5 分鐘。安全性案例研究（mujs）顯示原 C 程式中的記憶體漏洞（如堆疊緩衝區溢位、使用後釋放）在 Rust 版本中被消除或緩解。性能方面，Rust 程式相較 C 程式中位數測試案例執行時間約慢 1.28 至 1.51 倍。消融實驗證明特徵簡化提升了驗證測試通過率 6% 至 20%。與現有工具比較，Reboot 生成的 Rust 程式碼更接近原始大小且無 unsafe，且能處理更大規模程式。

**Conclusion:** 本研究提出 Reboot，結合特徵簡化與多代理系統，實現大型語言直譯器從 C 到安全 Rust 的半自動轉譯。實驗證明該方法能生成無 unsafe 區塊且通過完整測試的 Rust 程式，顯著提升安全性並保持合理性能。特徵簡化對提升轉譯正確性及處理複雜度關鍵，尤其在大型程式中不可或缺。限制包括轉譯過程仍需少量人力介入，且驗證測試未必涵蓋所有語義差異，泛化至其他類型程式仍待驗證。未來工作可聚焦於降低人力介入、擴展至非直譯器程式及提升未見測試的正確率。

**Research Gap:** 本論文雖成功處理多個大型直譯器，但仍存在幾項不足。首先，轉譯過程需 28 至 90 小時及高額成本，對實際工程部署仍有挑戰。其次，驗證依賴測試套件，無法保證語義完全等價，且未充分說明失敗案例或錯誤類型分布。第三，特徵簡化策略雖有效，但對非直譯器或依賴複雜外部函式庫的程式適用性未明。第四，代理系統依賴 LLM，存在非決定性與錯誤回復限制，且人類介入仍不可避免。最後，安全案例研究集中於 mujs，其他直譯器安全提升未充分驗證。這些限制提示未來需加強成本效益、泛化能力、錯誤分析及自動化程度。全文未明確說明對多種 LLM 或代理架構的適應性及長期維護策略。
