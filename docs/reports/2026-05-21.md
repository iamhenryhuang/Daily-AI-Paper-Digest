### 1. Hallucination as Exploit: Evidence-Carrying Multimodal Agents

- 分數：5
- arXiv: [2605.19192](https://arxiv.org/abs/2605.19192)
- 發表日期：2026-05-20
- 分類：cs.AI
- 入選理由：+2 提及頂級機構：mit; +3 提及頂級會議：acl; LLM rank #1 / 9/10: Proposes a novel, principled architecture (ECA) to address hallucination-to-action failures in multimodal agents with strong empirical evidence and rigorous evaluation, showing practical safety improvements.

**Intro:** 多模態代理在使用截圖、文件和網頁資料時，可能因錯誤的視覺判斷觸發不當的工具操作，這種現象被定義為幻覺轉換為行動的安全失效。傳統的幻覺研究多聚焦於答案品質錯誤，卻忽略了幻覺導致的授權錯誤。本文提出證據攜帶多模態代理（ECA），將模型自由文本視為不可接受的證據，透過結構化的驗證器產生類型化證書，並以決定性閘門控制工具調用授權。此架構將不透明的模型信念轉化為可審計的驗證殘留，提升安全性透明度。

**Motivation:** 在多模態代理中，幻覺不再只是答案錯誤，而是可能直接導致危險行動的執行，這對工程與產品安全構成嚴重挑戰。現有的提示注入防禦多針對指令層面，無法防範由錯誤感知引發的授權錯誤。為了保障工具調用的安全，必須將行動前置條件與外部證據嚴格綁定，避免模型自由文本直接授權敏感操作。這對多模態系統的實際部署與風險管理具有重要意義。

**Method:** ECA架構將多模態代理的行動決策分為兩條平行路徑：模型提出行動計劃，驗證器基於DOM、OCR和輔助技術產生類型化證書。每個工具調用對應一組行動關鍵謂詞，閘門僅在所有謂詞均有有效證書時允許執行。證書包含類型、值、區域、來源、驗證器、信心度和信任標籤等欄位，確保證據可追蹤且不可由模型自行鑄造。架構假設感知不完美，將模型幻覺風險轉化為可量化的驗證器殘留風險，並透過多通道交叉驗證降低誤判率。

**Result:** 在1900次針對驗證器的對抗性紅隊攻擊中，經過四項針對性強化後，閘門繞過率由15%降至1.3%。在200個端到端任務和120個瀏覽器原型任務中，ECA達成0%不安全行動率，且保持高效能完成率。500個分層任務的HACR審計顯示，未防護代理100%不安全行動執行，提示防禦僅提示層則達49.6%，ECA則完全阻擋。多種GPT-5.4判斷模型仍無法有效防範不安全行動，證明結構化證書閘門的必要性與優勢。

**Conclusion:** 本文提出的ECA架構有效將多模態代理的行動授權與外部證據嚴格綁定，成功阻止幻覺導致的授權錯誤。實驗涵蓋多層次評估，從驗證器紅隊攻擊到端到端部署，均展現顯著安全提升。該方法將模型的自由文本與授權嚴格分離，提供一種可審計且可量化的安全邊界。限制在於驗證器仍存在殘留風險，行動謂詞語義完整性依賴專家校驗，且尚未在多回合實時代理中全面驗證。

**Research Gap:** 本文雖在多模態代理安全性上提出明確架構，但驗證器的殘留誤判率仍存在，且對於多通道聯合攻擊的防禦尚未完全穩固。行動謂詞語義的自動合成仍依賴專家介入，缺乏完全自動化且可擴展的方案。實驗主要基於靜態任務與有限場景，尚未涵蓋多回合交互或實時動態環境，對於長期部署的適應性與泛化能力未充分驗證。此外，對於更複雜工具類型（如機器人操作、資料庫管理等）的行動模式與證書設計仍缺乏深入探討。最後，神經判斷模型的失效揭示了模型內部信任評估的脆弱性，未來需探索更強健的跨模態信任機制。

### 2. AutoResearchClaw: Self-Reinforcing Autonomous Research with Human-AI Collaboration

- 分數：5
- arXiv: [2605.20025](https://arxiv.org/abs/2605.20025)
- 發表日期：2026-05-20
- 分類：cs.AI
- 入選理由：+3 收錄於 Hugging Face Daily Papers; +2 提及程式碼可用; LLM rank #2 / 8/10: Introduces a multi-agent autonomous research system with human-in-the-loop collaboration and verifiable results, showing significant benchmark improvements and practical utility with open code.

**Intro:** 本論文針對自動化科學研究中存在的三大挑戰：假說質量不足、實驗執行脆弱以及缺乏跨次執行的經驗累積，提出一套多代理人協作的研究管線AutoResearchClaw。現有系統多為單代理人線性流程，遇失敗即終止且不保留歷史經驗，無法模擬真實研究中反覆試驗與修正的迭代過程。AutoResearchClaw結合多代理人辯論、自我修復執行、可驗證結果報告、人機協作介入及跨次演化機制，旨在提升自動研究的完整性與可靠性。

**Motivation:** 真實科學研究是多角度挑戰假說、從失敗中學習並持續改進的循環過程，現有自動研究系統多忽略此迭代特性，導致假說生成偏頗、實驗失敗即終止且無法累積經驗。這對工程與產品落地造成限制，因為缺乏自我修復與跨次學習機制的系統難以應對複雜多變的研究場景。AutoResearchClaw透過結合多代理人角色分工與人機協作，提升研究質量與效率，並確保結果的可驗證性，對推動AI輔助科學研究具實際意義。

**Method:** AutoResearchClaw包含五大核心機制：一是多代理人辯論，於假說生成與結果分析階段分配創新者、務實者、反對者等角色，促進多角度批判與整合；二是自我修復執行器，利用Pivot/Refine決策迴圈將實驗失敗視為診斷資訊，進行修正或轉向新假說；三是可驗證結果報告，建立數值註冊表與四層引用驗證機制，防止數據造假與虛假引用；四是人機協作，提供七種介入模式，從全自動到逐步監督，並透過SmartPause機制於不確定階段邀請人類介入；五是跨次演化，保存過去失敗經驗並以時間衰減權重注入未來執行，促進持續改進。系統分為探索、實驗與寫作三階段，並支持檢查點恢復與多領域科學軟體整合。

**Result:** 在25個ARC-Bench主題的實驗階段評測中，AutoResearchClaw(CoPilot模式)整體得分0.648，較AI Scientist v2提升54.7%，尤其在結果分析指標上提升100.4%，顯示多代理人辯論與驗證機制有效提升結論品質。全自動模式下仍優於基線，證明系統設計貢獻顯著。跨領域測試涵蓋物理、生物與統計領域，AutoResearchClaw成功執行並產出有效結果，基線系統因缺乏領域軟體支持多數失敗。人機協作消融實驗顯示，針對高槓桿決策點的精準介入(CoPilot)優於全自動與逐步監督，提升接受率至87.5%。組件消融證實多代理辯論與自我修復為關鍵，缺一則品質與完成率大幅下降。

**Conclusion:** AutoResearchClaw提出一套結合多代理辯論、自我修復執行、結果驗證、人機協作及跨次演化的自動化研究系統，成功解決假說質量、執行魯棒性與經驗累積三大挑戰。實驗結果證明其在多領域研究任務中優於現有系統，且人機協作模式能有效提升研究品質與接受率。系統強調科學誠信與可驗證性，定位為輔助而非取代人類科學判斷的研究加速器。全文未明確說明系統在極端複雜任務或長期部署中的表現與成本。

**Research Gap:** 本論文雖在ARC-Bench及部分科學領域展現優異成效，但實驗場景仍有限，尚未涵蓋更廣泛或更複雜的科學問題，且跨次演化的長期穩定性與泛化能力未充分驗證。系統依賴多代理人與人機協作，對計算資源與人力成本要求較高，部署風險與實際效率未詳述。自我修復機制在多檔案複雜依賴場景仍有失敗案例，顯示修復策略尚有改進空間。驗證機制雖防止造假，但對實驗設計合理性與科學意義的判斷仍依賴人類介入，自動化程度有限。全文未明確說明對失敗案例的系統性分析及失效模式，缺乏對潛在偏誤與錯誤傳播的深入探討。

### 3. PEEK: Context Map as an Orientation Cache for Long-Context LLM Agents

- 分數：5
- arXiv: [2605.19932](https://arxiv.org/abs/2605.19932)
- 發表日期：2026-05-20
- 分類：cs.AI
- 入選理由：+2 提及頂級機構：openai; +3 收錄於 Hugging Face Daily Papers; LLM rank #3 / 8/10: Presents a novel context map caching method (PEEK) that improves efficiency and accuracy for long-context LLM agents with strong empirical gains and broad applicability.

**Intro:** 大型語言模型代理在處理長且重複的外部上下文時面臨挑戰，因為現有方法多半只保存代理的執行軌跡、被動存取原始資料或任務層級策略，缺乏對重複上下文中可重用的定位知識的保存。這種定位知識包含上下文內容、組織方式及歷史上有用的實體和結構。本文提出PEEK系統，透過一個固定大小的上下文地圖作為代理提示中的快取，持續保存並更新這些定位知識，幫助代理更有效率且準確地理解和操作外部上下文。

**Motivation:** 在長上下文且重複查詢的場景中，如企業分析師反覆查詢龐大文件庫，代理若每次都從零開始理解上下文，效率低且容易出錯。現有方法如共享聊天歷史、檢索增強生成或上下文壓縮，均無法持續保存關於上下文本身的可重用知識。PEEK填補此缺口，將定位知識以結構化且有限大小的形式保存在提示中，類似硬體快取，讓代理在多次查詢中能快速定位和利用重要資訊，提升推理和學習效率。

**Method:** PEEK系統包含一個固定大小的上下文地圖，存放關於外部上下文的定位知識，分為多個結構化區塊，如內容路線圖、上下文理解、領域常數等。系統透過三個模組維護地圖：Distiller從代理執行軌跡中提取可轉移的上下文知識並診斷執行情況；Cartographer將這些知識轉換為結構化的編輯操作以更新地圖；Evictor根據優先級在固定的token預算內淘汰較不重要的條目。此設計避免任務特定資訊污染地圖，並確保地圖大小受控且持續更新。

**Result:** 在兩類長上下文任務（長上下文推理與資訊聚合、上下文學習）上，PEEK相較於強基線如RLM、RAG、Compaction Agent及SOTA提示學習框架ACE，展現6.3%至34.0%的性能提升，且迭代次數減少93至145次，成本降低1.7至5.8倍。PEEK在解題率和細緻評分上也分別提升6.0%至14.0%及7.8%至12.1%。此外，PEEK在不同基礎模型（GPT-5.5、Qwen3-Coder-Next-FP8）及代理架構（OpenAI Codex）上均有穩定提升，顯示良好泛化能力。消融實驗證明分離Distiller與Cartographer、優先淘汰策略及適當快取大小均對性能有顯著貢獻。

**Conclusion:** 本文提出的PEEK系統透過上下文地圖有效保存並更新重複查詢外部上下文的定位知識，顯著提升長上下文任務中代理的準確性與效率。PEEK不僅優於多種先進基線，且具備跨模型與代理架構的泛化能力，展現實際應用潛力。限制在於上下文地圖的價值依賴於代理與上下文的交互方式，若交互中可重用知識有限，地圖效益降低。全文未明確說明在極端大規模上下文或不同類型代理中的表現與部署成本，未提供失敗案例分析。

**Research Gap:** 本文雖展示PEEK在多種基準和模型上的優勢，但缺乏對不同類型外部上下文（如非結構化文本、動態變化資料）及多樣代理交互策略的深入分析。快取大小選擇未經充分調優，且對快取更新頻率與策略的敏感性探討有限。成本評估主要聚焦推理與維護成本，未涵蓋實際部署中可能面臨的延遲、資源限制與系統整合風險。缺少對失敗案例或地圖錯誤更新對代理性能影響的討論，未明確說明如何處理上下文變更或過時資訊。未提供對比其他先進快取或記憶機制的更全面消融與理論分析。

### 4. Learning to Hand Off: Provably Convergent Workflow Learning under Interface Constraints

- 分數：5
- arXiv: [2605.19140](https://arxiv.org/abs/2605.19140)
- 發表日期：2026-05-20
- 分類：cs.AI
- 入選理由：+2 提及頂級機構：mit; +3 提及頂級會議：acl; LLM rank #4 / 8/10: Provides the first finite-sample convergence guarantee for decentralized multi-agent workflow learning under partial observability, supported by theoretical analysis and diverse experiments.

**Intro:** 本文探討多代理系統中，專門化代理透過共享介面物件進行控制交接的工作流程學習問題。此場景下，每個代理只能觀察介面物件的局部函數及自身私有狀態，且無中央學習者能取得聯合軌跡，這是跨組織、供應商或信任邊界的多代理大型語言模型管線的典型運作模式。作者提出介面限制半馬可夫決策過程（IC-SMDP）作為形式化模型，並設計一種非同步去中心化的Q學習演算法IC-Q，實現每次交接只需傳遞一個標量的跨代理協調。此研究首次在去中心化部分可觀察條件下，為神經網路Q學習提供有限樣本收斂保證。

**Motivation:** 現有多代理工作流程設計多基於中央化假設，需取得所有代理的聯合軌跡，然而實務中跨組織或供應商邊界往往無法共享完整資訊，造成去中心化且介面受限的觀察條件。此種結構下，傳統Dec-POMDP、options框架及中央化訓練方法均不適用，缺乏理論基礎與有限樣本收斂保證。工程與產品層面，理解在有限API調用與計算資源下，如何有效學習並保證工作流程性能，對多代理LLM管線的部署與優化至關重要。

**Method:** 作者定義IC-SMDP模型，將決策節點設於代理交接時刻，將每個代理的執行視為一個選項（option），並引入代理專屬的介面觀察映射ϕi，限制代理只能基於介面狀態的局部表示與私有狀態決策。IC-Q演算法為非同步去中心化Q學習，核心在於交接時由接手代理計算最大後續價值的標量，回傳給前手代理作為bootstrap目標，確保跨代理通訊僅需一個標量，符合介面限制。理論上，證明IC-Q在神經網路函數逼近下，有限樣本誤差可分解為神經逼近誤差、介面表示差距與混合時間殘差三部分，並針對隨機選項長度的折扣因子提出新的收斂分析。

**Result:** 實驗涵蓋四個任務：合成IC-SMDP用以逐項驗證理論誤差分解，四代理多LLM數學推理，百代理多代理路由，以及六代理CPU程式設計。合成實驗顯示介面表示差距、樣本數與混合時間對誤差的影響符合理論預測。多LLM推理中，IC-Q在無中央觀察聯合軌跡下，成功學習出與中央化oracle相當的最佳工作流程，且能根據資料集難度自動調整代理間交接策略。路由任務中，IC-Q達成100%準確率，驗證理論中介面誤差為零的情況。CPU程式設計任務則展示IC-Q在可調整代理行為下，具備良好泛化能力，儘管此情況不在理論收斂範圍內。

**Conclusion:** 本文提出IC-SMDP模型與IC-Q演算法，填補了多代理去中心化且介面受限工作流程學習的理論空白，首次提供神經Q學習在此設定下的有限樣本收斂保證。實驗結果不僅驗證理論誤差分解，亦證明IC-Q能在無中央軌跡觀察的條件下，達成與中央化方法相當的性能。此框架對跨組織、多供應商的多代理LLM系統部署與優化具實際指導意義，尤其在API調用受限與隱私保護需求下。

**Research Gap:** 本文理論收斂結果僅涵蓋預設代理行為（pre-configured regime），對於代理本身行為可調整的情況（adaptable agents）尚無嚴謹理論支持，僅以實驗驗證。介面映射ϕi的選擇對性能影響重大，但缺乏自動化設計或優化方法。實驗場景雖涵蓋多樣任務，仍偏向結構化或合成環境，對更複雜、動態或非合作多代理系統的泛化能力未明。成本與通訊延遲等實際部署風險未深入探討，且未提供失敗案例分析，限制了對系統穩健性的全面評估。最後，理論分析依賴多項假設（如均勻遍歷性、局部線性化等），在實務中可能難以完全滿足，需進一步驗證其適用範圍。

**Intro:** 本論文提出OpenComputer，一個以驗證器為核心的框架，用於構建可驗證的軟體世界，供電腦使用代理執行任務。現有電腦使用代理的訓練與評估受限於構建真實且可重現的桌面環境與任務的成本。OpenComputer整合了應用程式專屬狀態驗證器、自我演化的驗證層、任務生成管線及評估系統，涵蓋33種桌面應用與1000個任務。實驗顯示硬編碼驗證器在細粒度應用狀態判斷上比LLM評判更接近人工判定，且現有先進代理在端到端完成任務上仍有挑戰。

**Motivation:** 電腦使用代理的發展需要大量真實且可驗證的任務環境，但手動構建這些環境繁瑣且難以標準化。任務成功判斷不僅依賴螢幕截圖，還需檢查應用程式內部狀態與持久化資料，這使得評估難以擴展。使用LLM作為評判者存在不穩定、難以審計及可能忽略細節錯誤的問題。為此，OpenComputer將驗證作為環境與任務構建的核心，提供可程式化且可自我修正的驗證機制，提升評估的可靠性與可擴展性。

**Method:** OpenComputer包含四大核心組件：首先，為每個應用程式建立專屬驗證器，透過多種穩定的檢查通道（如瀏覽器調試協議、資料庫、D-Bus等）以結構化JSON形式暴露應用狀態，並經過嚴格測試確保可靠性。其次，透過自我演化層，利用校準任務執行結果與LLM判斷的差異反饋，持續修正驗證器錯誤。第三，任務生成管線先提出多樣化用戶目標，過濾難度與資料生成性，再依驗證器可檢查性生成環境與成功標準。最後，評估系統在沙箱環境中執行代理操作，記錄完整軌跡並以驗證器判定成功率，支持部分積分評分。

**Result:** 在包含33種桌面應用與1000個任務的OpenComputer基準測試中，GPT-5.4表現最佳，成功率達68.3%，平均獎勵88.4%，但仍有約三成任務未能完全完成。Claude-Sonnet-4.6與Kimi-K2.6分別達64.4%與58.8%。開源模型表現明顯落後，且與其在OSWorld基準上的成績相比，成功率大幅下降，顯示跨基準泛化能力不足。硬編碼驗證器與人工標註一致性高達94.1%，優於LLM評判的79.2%。CLI代理在部分任務上效率較高，但GUI代理成功率較佳。自我演化驗證層能在多數情況下快速修復驗證器錯誤，提升評估準確度。

**Conclusion:** OpenComputer提供一套以驗證器為核心的系統，實現了真實桌面軟體環境中可程式化、可驗證的任務生成與評估。此框架促進了電腦使用代理的可靠性研究與訓練，並揭示當前先進模型在細節狀態判斷與端到端任務完成上的不足。硬編碼驗證器相較LLM評判更能捕捉細微錯誤，提升評估的可審計性與重現性。OpenComputer作為基礎設施，有助於推動更穩健且可量化的電腦代理系統發展。

**Research Gap:** 本論文雖強調硬編碼驗證器的可靠性，但對於需視覺或幾何判斷的任務（如圖形連接關係、空間佈局）仍無法完全程式化驗證，這部分僅以LLM視覺判斷輔助且未納入主基準，限制了驗證器的全面性。實驗中多數任務與應用仍集中於特定桌面軟體，泛化至更多軟體類型或跨平台環境尚未充分驗證。自我演化驗證層雖能修正多數錯誤，但仍有部分錯誤未能在預設迭代內修復，顯示驗證器穩定性仍有提升空間。此外，代理模型在端到端任務完成率仍低，且缺乏對失敗案例的深入分析與改進策略。全文未明確說明驗證器與任務生成在大規模實際部署中的成本與維護挑戰。
