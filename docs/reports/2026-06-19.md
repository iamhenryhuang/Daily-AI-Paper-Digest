### 1. Multi-Task Bayesian In-Context Learning

- 分數：7
- arXiv: [2606.20538v1](http://arxiv.org/abs/2606.20538v1)
- 發表日期：2026-06-18
- 分類：cs.LG
- 入選理由：+2 提及頂級機構：meta, mit; +3 提及頂級會議：acl, icml; +2 提及程式碼可用; LLM rank #1 / 9/10: Introduces a novel multi-task in-context learning framework for hierarchical Bayesian inference with strong empirical results, practical relevance, and code release, balancing novelty, usefulness, and credible evidence.

**Intro:** 本論文探討貝葉斯預測推論在不確定性量化、資料效率與穩健泛化上的重要性，但指出傳統精確推論計算困難且近似方法計算成本高或需嚴格模型假設。近年來，Prior-Data Fitted Networks與in-context learning成為可攜式推論替代方案，但現有方法無法在測試時靈活調整先驗，導致對分布轉移的魯棒性不足。作者提出一種多任務in-context學習框架，將先驗資訊明確表示為上下文資料集的前綴，並利用Transformer學習跨先驗族的適應能力。

**Motivation:** 在實際工程與產品中，先驗分布往往隨用戶、領域或環境變化，固定隱式先驗的模型缺乏測試時調整能力，限制了模型在分布轉移下的穩健性與泛化。傳統貝葉斯推論方法如MCMC雖精確但計算昂貴，變分推論則可能因近似偏差而表現不佳。因此，開發一種能快速推論且可控先驗調整的模型，對於實際部署與應用具有重要意義。

**Method:** 作者提出Multi-Task Bayesian In-Context Learning框架，將多個先驗相關的資料集作為上下文序列的前綴輸入Transformer，模型學習從這些先驗資料推斷先驗分布，並結合目標任務資料進行預測。訓練時從元分布抽取多個任務及其先驗資料，優化目標任務的預測對數似然。方法支持線性與邏輯回歸的似然函數，並以Transformer作為黑盒函數映射資料序列到預測分布參數，實現階層式貝葉斯推論且無需測試時參數更新。

**Result:** 在多種實驗中，模型在元分布內先驗下能量化匹配貝葉斯oracle，且在先驗分布外（OoMD）仍展現優越的泛化能力，尤其在低資料量時優於傳統方法。對於重尾分布的先驗，模型能透過包含多樣先驗的訓練提升泛化範圍，並在高維流形先驗下達到與MCMC相當的預測性能，推論速度快數個數量級。實際應用於ERA5氣候資料的時空溫度預測，展示在IID與OOD分布下先驗前綴均能提升預測準確度與穩健性。

**Conclusion:** 本論文提出一種結合in-context學習與階層貝葉斯推論的多任務框架，透過明確的先驗資料前綴實現測試時先驗可控調整，兼具靈活性與計算效率。實驗證明該方法在多種先驗族及分布轉移下均能匹配或超越傳統貝葉斯推論基準，且在真實環境資料上具實用價值。限制包括Transformer注意力機制的計算成本隨序列長度平方增長，以及模型未明確強制資料集內外的排列不變性，可能影響部分應用場景。

**Research Gap:** 論文未充分探討模型在極端分布轉移或更複雜多模態先驗下的泛化極限，缺乏對失敗案例或不適用場景的分析。雖然展示了部分排列敏感性較低，但未系統驗證模型對資料集內外順序變化的魯棒性，這對實務中資料順序不確定性可能造成影響。此外，模型在大規模高維資料集上的計算與記憶體成本仍是挑戰，未明確提出解決方案。最後，全文未明確說明在非回歸任務或非結構化資料上的適用性與效果，限制了方法的通用性評估。

### 2. Your Mouse and Eyes Secretly Leak Your Preference: LLM Alignment using Implicit Feedback from Users

- 分數：4
- arXiv: [2606.20482v1](http://arxiv.org/abs/2606.20482v1)
- 發表日期：2026-06-18
- 分類：cs.CL, cs.HC, cs.LG
- 入選理由：+2 提及頂級機構：mit; +2 提及程式碼可用; LLM rank #2 / 8/10: Presents a novel method leveraging implicit user feedback (mouse and eye tracking) to improve LLM alignment, with a new dataset and significant empirical gains, addressing a key limitation in current alignment approaches

**Intro:** 本論文探討如何利用使用者的隱性反饋訊號，如滑鼠軌跡與眼動追蹤，來提升大型語言模型（LLM）的對齊效果。現有方法多依賴使用者明確的偏好標註，但這類標註稀少且昂貴，且未充分利用隱性反饋。作者建立了IFLLM資料集，收錄多輪問答過程中59名Mechanical Turk工作者的滑鼠與眼動數據，並結合明確偏好標註，嘗試以此訓練獎勵模型。研究目標在於驗證隱性反饋對提升偏好預測與LLM對齊的價值。

**Motivation:** 使用者在與LLM互動時，明確提供反饋的比例極低，且頻繁要求反饋可能降低使用者滿意度。相較之下，隱性反饋如滑鼠移動與眼動追蹤在搜尋引擎與推薦系統中已證明具價值，但在LLM領域尚未被充分利用。由於LLM回應中缺少可直接利用的點擊數據，故本研究聚焦於更普遍可取得的滑鼠與眼動訊號，期望藉此降低標註成本並提升模型對齊的實用性與精準度。

**Method:** 作者開發一個基於網頁的系統，透過網路攝影機與WebGazer.js進行眼動追蹤，並同步記錄滑鼠軌跡。使用者在多輪問答中選擇主題並與多款LLM互動，完成明確偏好標註（點評或兩兩比較）。從收集的滑鼠與眼動數據中提取多種特徵，如閱讀時間、注視位置、滑鼠與眼動的相關性等，並以隨機森林訓練獎勵模型。接著利用該模型的偏好預測結果，透過rDPO與NLL方法對多款LLM進行微調，以提升回應品質。

**Result:** 實驗中，隨機森林結合隱性反饋特徵的獎勵模型在偏好預測準確率由純文字模型的約55%提升至64%。滑鼠軌跡對長回應的影響尤為顯著，眼動信號則在短回應中較有用。利用該獎勵模型進行rDPO微調後，八款LLM的回應品質相較於基線提升近三倍，且生成回應長度較短但品質更佳。明確反饋模型表現不如隱性反饋模型，顯示隱性反饋能有效補足明確標註的不足。

**Conclusion:** 本研究首次系統性地收集並分析使用者在多輪LLM問答中滑鼠與眼動隱性反饋，證明這些訊號能顯著提升偏好預測與LLM對齊效果。滑鼠軌跡因反映使用者滾動與閱讀行為，成為關鍵訊號，且易於在實際產品中收集。研究釋出資料集與收集平台，有助後續隱性反饋研究與應用。限制包括獎勵模型需依賴隱性反饋輸入，且尚未充分利用多輪上下文與問卷數據，未來可進一步探討跨輪訊號整合與更廣泛的使用場景。

**Research Gap:** 本論文未明確說明如何在無法取得眼動或滑鼠數據的場景下應用該方法，限制了泛化性。實驗主要基於Mechanical Turk使用者與特定主題，缺乏多元使用者族群與真實產品環境驗證。隱性反饋特徵的噪聲與個體差異對模型穩定性影響尚未深入探討。獎勵模型依賴滑鼠與眼動數據，增加部署複雜度與隱私風險，未充分評估實際成本與用戶接受度。此外，對於長回應中眼動信號過度擬合的問題也未有解決方案。最後，消融實驗主要集中於特徵重要性，缺乏與其他隱性反饋類型或更先進模型的比較。

### 3. HumanScale: Egocentric Human Video Can Outperform Real-Robot Data for Embodied Pretraining

- 分數：4
- arXiv: [2606.20521v1](http://arxiv.org/abs/2606.20521v1)
- 發表日期：2026-06-18
- 分類：cs.CV
- 入選理由：+2 提及頂級機構：mit; +2 提及程式碼可用; LLM rank #3 / 8/10: Systematic study showing egocentric human video can outperform costly real-robot data for embodied model pretraining, with strong empirical evidence and practical implications for scalable embodied AI.

**Intro:** 本論文探討了在具身基礎模型（embodied foundation models）預訓練階段，使用第一人稱視角的人類影片（egocentric human video）與遙控真實機器人數據（teleoperated real-robot data）作為資料來源的效能差異。具身模型因為需要結合視覺與動作，資料收集成本高且多樣性不足，限制了模型的泛化能力。作者提出以大規模、多樣且成本低廉的第一人稱人類影片作為替代資料，並系統性比較兩種資料來源在相同後訓練與驗證條件下的表現差異。

**Motivation:** 遙控真實機器人數據雖然提供精確的動作監督與機體對齊，但收集成本高昂、環境與行為多樣性有限，難以大規模擴充。相較之下，第一人稱人類影片具備規模龐大、成本低、場景與互動多樣的優勢，但缺乏精確的機器人動作標註與機體對齊。如何評估這種資料是否能有效替代甚至超越真實機器人數據，對於工程實務與研究落地具有重要意義，尤其在提升模型泛化能力與降低資料收集成本方面。

**Method:** 作者從HumanNet資料庫中精選5,000小時的第一人稱人類影片，透過手部姿態重定向產生偽動作標籤，使其動作空間與機器人資料一致。與此同時，收集等量的多機體真實機器人遙控軌跡資料。兩者分別用於預訓練相同架構的自回歸世界動作模型（World-Action Model, WAM），該模型結合Mixture-of-Transformers架構，預測未來影像與動作。預訓練後，兩種模型均在相同的真實機器人後訓練資料集（AgiBot World）上微調，並在包含已見與未見任務的測試集上評估性能，確保比較條件一致。

**Result:** 實驗結果顯示，第一人稱人類影片預訓練在後訓練階段的驗證動作損失比真實機器人預訓練低24%，在已見任務與未見任務的成功率分別高出52.5%與90%。此外，第一人稱影片資料的預訓練效果隨資料量增加呈現良好擴展性，未見任務的損失持續下降，而真實機器人資料則在未見任務上表現停滯。真實機器人實驗中，第一人稱預訓練模型在分布外物件上仍維持90%成功率，顯著優於無預訓練基線的0%。

**Conclusion:** 本研究證實，第一人稱人類影片作為具身模型預訓練資料，不僅可行且在泛化能力上優於傳統的真實機器人遙控資料。其低成本、高多樣性與大規模可擴展性，為具身基礎模型的資料收集與訓練提供了新的可行路徑。後訓練階段利用少量真實機器人資料進行動作空間對齊，有效彌補了機體差異。研究結果對工程實務中降低資料收集成本、提升模型泛化性具有指導意義，但仍屬初步驗證。

**Research Gap:** 本論文雖在5,000小時資料規模下完成嚴謹比較，但尚未驗證更大規模資料的擴展性與效益，且僅限於特定世界動作模型架構，未涵蓋其他具身模型如視覺語言動作模型（VLA）。此外，偽動作標籤的準確性與其對最終性能的影響未有深入分析，可能影響模型表現穩定性。實驗場景與任務仍有限，未充分涵蓋更複雜或動態環境，泛化能力在更廣泛真實世界條件下的表現尚待驗證。最後，成本與部署風險分析較為簡略，未明確評估實際應用中可能遇到的挑戰與限制。

### 4. Multi-LCB: Extending LiveCodeBench to Multiple Programming Languages

- 分數：5
- arXiv: [2606.20517v1](http://arxiv.org/abs/2606.20517v1)
- 發表日期：2026-06-18
- 分類：cs.AI, cs.PL
- 入選理由：+2 提及頂級機構：mit; +3 提及頂級會議：iclr; LLM rank #4 / 8/10: Extends a widely used code generation benchmark to multiple programming languages, revealing important insights about LLM multilingual code capabilities and contamination, with broad practical impact and rigorous evaluat

**Intro:** 本論文針對現有的 LiveCodeBench (LCB) 只支援 Python 的限制，提出 Multi-LCB，一個涵蓋十二種程式語言的多語言程式碼生成評測基準。LCB 以競賽程式題目為基礎，並透過發布日期過濾來避免訓練資料汙染，提供持續更新且具污染意識的評測環境。Multi-LCB 將 LCB 的 Python 題目轉換成其他語言的等價題目，保留原有的評測協議與污染控管，並可自動追蹤 LCB 的未來更新。此舉旨在檢驗大型語言模型（LLM）是否能跨語言泛化程式碼生成能力，回應現實軟體工程中多語言開發的需求。

**Motivation:** 現有程式碼生成評測多集中於 Python，忽略了軟體開發中多語言並存的現實情況。LLM 若僅在 Python 表現良好，可能無法適應其他語言的語法、語意及慣用法，導致泛化能力不足。LCB 雖具污染意識與持續更新優勢，但語言單一限制了其代表性與實用性。Multi-LCB 透過多語言擴展，讓工程師與研究者能更全面評估模型在多語言環境下的實際應用潛力，對產品開發與模型訓練策略有直接指導意義。

**Method:** Multi-LCB 從 Hugging Face 取得 LCB 的 Python 題庫，涵蓋 LeetCode、AtCoder、Codeforces 三大競賽平台，並保留原始題目描述、輸入輸出範例及發布日期。針對 LeetCode 的函式式題目，設計自動轉換流程將其改寫為統一的 STDIN/STDOUT 格式，避免為每種語言分別撰寫測試模板。評測時，模型接收包含目標語言指示的自然語言題目與範例，並以零次學習方式生成完整程式碼。生成程式碼在隔離沙箱中編譯執行，通過所有隱藏測試即視為正確。此流程確保跨語言評測的一致性與污染控管。

**Result:** 作者評測了 24 款涵蓋 7B 至 685B 參數的公開 LLM，涵蓋指令調校與推理強化模型。結果顯示，Python 表現普遍優於其他語言，且模型在 Python 上的強勢不一定能延伸至其他語言，揭示明顯的 Python 過擬合現象。不同語言間存在顯著性能差距，靜態型別語言及較少見語言表現較弱。透過發布日期過濾，發現較舊題目存在訓練資料汙染跡象，評測時段跨越模型訓練截止日期後，成績明顯下降，驗證污染控管有效。與原 LCB Python 子集比較，Multi-LCB 在 Python 上的成績重現性良好，證明多語言轉換未引入額外困難。

**Conclusion:** Multi-LCB 成功擴展了 LiveCodeBench 的多語言評測能力，提供一個持續更新且污染意識明確的跨語言程式碼生成基準。該基準揭露了現有 LLM 在多語言泛化上的不足，特別是對 Python 的過擬合與語言特定汙染問題。Multi-LCB 的設計具備高度模組化與擴展性，未來可納入更多語言與模型，為多語言程式碼生成研究與產品開發提供重要參考。限制包括語言選擇未涵蓋所有重要語言、題目仍以競賽程式為主且不涵蓋實務軟體工程場景，以及評測格式可能影響模型表現的公平性。

**Research Gap:** 本研究雖涵蓋十二種主流語言，但未包含如 Swift、Haskell、R 等專業或新興語言，限制了泛化評估的全面性。題目仍以競賽程式為主，缺乏對實際軟體開發中 API 整合、除錯、團隊協作等複雜任務的評測，影響對模型實務應用能力的判斷。STDIN/STDOUT 格式轉換可能導致部分語言因語法或輸入輸出格式不熟悉而表現下降，混淆了語言能力與格式遵循的區別。評測僅涵蓋公開模型，未包含商業閉源系統，無法全面反映當前領先技術。最後，雖有日期過濾控管，仍可能存在隱性資料汙染及模型訓練期間語言分布不均的偏差，未充分探討失敗案例與泛化失效情境。

**Motivation:** 深度神經網絡成功的關鍵在於其能學習反映自然數據中潛在的層次性和稀疏結構的特徵。現有合成數據集多未能捕捉這些結構，導致可解釋性方法的評估缺乏現實參考。自然數據的稀疏性、層次性和低內在維度常伴隨冪律分布，這些特性對於理解模型如何學習和表示數據至關重要。透過建立一個理論上可解析且結構豐富的合成數據模型，能促進對神經網絡內部機制的深入理解，並推動可解釋性方法的工程與產品落地。
