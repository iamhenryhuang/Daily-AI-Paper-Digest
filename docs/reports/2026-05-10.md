### 1. Federated Cross-Client Subgraph Pattern Detection

- 分數：0
- arXiv: [2605.06433v1](http://arxiv.org/abs/2605.06433v1)
- 發表日期：2026-05-07
- 分類：cs.LG
- 入選理由：未命中正向訊號

**Intro:** 本論文聚焦於分散式圖神經網絡（GNN）中跨客戶端子圖模式偵測的挑戰。現有的GNN方法多假設圖資料集中存取，然而實務中圖資料往往分散於多個客戶端，導致本地GNN計算與集中式模型結果不一致，形成所謂的表示等價差距。此問題被形式化為結構可觀測性問題，即跨分割邊界的子圖模式在本地無法識別。論文提出一種逐層、逐步的嵌入交換框架，讓客戶端在每層前向傳播後同步節點表示，避免直接暴露原始特徵與標籤。此方法在共享模型參數與擴展子圖假設下，能恢復與集中式GNN相同的節點表示。

**Motivation:** 在金融犯罪、資安及生物資訊等領域，子圖模式偵測是揭露複雜互動結構的關鍵。然而，實際應用中圖資料分散於多個機構，且受限於隱私與法規，無法集中存取完整圖。傳統聯邦學習透過參數聚合無法解決跨客戶端子圖結構缺失的問題，導致模型表現下降。為了提升分散式GNN的結構可觀測性並縮小與集中式模型的差距，提出的嵌入交換機制能在不違反資料隱私的前提下，實現跨客戶端的訊息同步，對工程與產品部署具實際意義。

**Method:** 核心方法為在每層GNN前向傳播結束後，客戶端間交換邊界節點的中間表示，確保每個客戶端在下一層計算時能使用來自遠端節點的最新嵌入。此交換過程不傳送原始特徵或標籤，只同步學習到的節點表示。理論證明在擴展子圖假設及共享參數條件下，此機制能使分散式計算結果與集中式GNN完全一致。實驗中使用Multi-PNA模型，在合成有向多重圖上評估，並比較不同參數聚合與嵌入交換策略的效果。

**Result:** 實驗在兩種社群分割策略（Louvain與METIS）下，使用含有多種結構模式的合成圖進行。結果顯示，完全本地訓練與僅參數聚合均無法有效恢復集中式模型表現，存在明顯的表示差距。加入逐層嵌入交換後，表現顯著提升，且與參數聚合結合效果最佳。特別是每層即時交換嵌入優於每輪交換（stale embedding），在跨客戶端連結密集時差異更明顯。此外，使用同步SGD進行參數同步可進一步縮小差距。整體而言，嵌入交換與參數聚合互補，缺一不可。

**Conclusion:** 本論文針對分散式子圖模式偵測中跨客戶端結構不可觀測問題，提出逐層嵌入交換框架，有效縮小分散式與集中式GNN的表示差距。理論與實驗均證明該方法在共享參數與擴展子圖假設下能恢復集中式節點表示，且與聯邦參數聚合結合效果最佳。此方法在保護原始資料隱私的前提下，提升分散式圖學習的結構表現，對實務部署具參考價值。論文亦指出未來可朝降低通訊成本、強化隱私保護及擴展至動態異質圖方向發展。

**Research Gap:** 論文理論保證基於全批次訓練與擴展子圖假設，然而實務中多採用小批次與鄰居採樣，導致理論與實際表現存在差距，全文未明確說明此近似對結果的影響範圍。實驗僅使用合成圖，缺乏真實異質動態圖的驗證，泛化性尚待評估。通訊成本與延遲問題雖有討論但未深入量化，對大規模部署仍具挑戰。隱私保護層面，交換的中間嵌入可能洩漏部分結構資訊，尚未提出具體防護措施。此外，對失敗案例、模型穩定性及異常模式偵測的分析不足，限制了方法在複雜實務場景的應用信心。

### 2. FedFrozen: Two-Stage Federated Optimization via Attention Kernel Freezing

- 分數：0
- arXiv: [2605.06446v1](http://arxiv.org/abs/2605.06446v1)
- 發表日期：2026-05-07
- 分類：cs.LG
- 入選理由：未命中正向訊號

**Intro:** 聯邦學習在面對客戶端資料異質性時，常因本地更新不一致導致客戶端漂移，影響模型穩定性與準確度。現有方法多從整體目標函數層面透過正則化或更新校正來緩解此問題，但未深入探討Transformer架構內部參數對聯邦優化的影響。本文提出FedFrozen，針對Transformer的注意力機制，將其拆解為決定注意力核的query/key區塊與執行語義轉換的value區塊，並設計兩階段優化策略以提升聯邦學習的穩定性與效能。

**Motivation:** Transformer模型在異質資料的聯邦學習中展現較傳統模型更好的魯棒性，但直接對整個注意力模組進行聯邦平均可能破壞其內部結構，尤其是query/key與value參數的不同角色未被充分利用。透過分別優化這兩部分，可望減少客戶端漂移對注意力核的影響，穩定表示空間，並降低通訊成本，這對於實際部署大型Transformer模型於分散式環境具有重要意義。

**Method:** FedFrozen採用兩階段優化：第一階段為全模型熱身訓練，讓query/key區塊學習共享的注意力核；第二階段凍結query/key區塊，只更新value區塊以穩定表示空間。理論上，熱身階段相當於對正則化核剖面目標的近似下降，而凍結階段則是對固定核下的value區塊進行強凸優化。方法中引入ℓ2正則化以確保value區塊優化良態，並支持部分客戶端參與。此策略兼顧優化穩定性與通訊效率。

**Result:** 模擬實驗中，FedFrozen在不同異質性程度下均能有效控制熱身階段的偏差與漂移，並在本地更新步數增加時相較FedAvg展現更低的最終損失。真實資料集（CIFAR-10、CIFAR-100、FEMNIST）上，FedFrozen在多種非IID分布下測試準確度穩定且與最佳基線相當，且在通訊成本上相較FedAvg節省至少10%。理論分析與實驗結果均支持該方法在異質聯邦學習中提升Transformer模型穩定性與效率。

**Conclusion:** 本文提出的FedFrozen透過分階段凍結Transformer注意力機制中的query/key區塊，有效減少客戶端漂移對模型訓練的負面影響，提升聯邦學習的穩定性與效能。理論分析揭示熱身長度的權衡，實驗驗證其在模擬與真實資料上的優越表現及通訊節省。限制在於理論分析基於線性注意力模型，對標準非線性注意力的理論支持尚未充分，未來可擴展至更複雜架構及大型語言模型的分散式微調。

**Research Gap:** 本文理論分析依賴線性注意力假設，對標準Transformer中非線性注意力的適用性未明確說明，可能限制理論保證的泛化性。實驗雖涵蓋多個資料集，但缺乏對更大規模或更複雜異質場景的測試，且未充分探討不同凍結策略或熱身長度自適應調整的效果。此外，對於模型凍結後可能導致的表徵能力下降或失敗案例未有深入分析，通訊成本與計算負擔的實際部署風險亦未詳細評估。未來研究應補足這些方面以提升方法的完整性與實用性。

**Intro:** 本文探討Transformer模型在in-context learning（ICL）中的表現，特別是針對線性分類任務的能力。Transformer已被證明在ICL上有優異表現，但其內部如何隱式執行演算法仍缺乏理論解析。作者聚焦於使用softmax attention的Transformer，試圖揭示其如何透過內部機制實現類似邏輯斯迴歸的學習過程。此研究有助於深化對Transformer在ICL中運作原理的理解，對AI從業者在設計與優化模型時具參考價值。

**Motivation:** Transformer在多種任務中展現強大的ICL能力，但目前多數解釋仍停留在直覺層面，缺乏嚴謹的數學證明與機制說明。理解Transformer如何在上下文中執行類似優化演算法，對於提升模型設計、訓練效率及泛化能力至關重要。尤其在工程與產品落地時，若能明確掌握模型內部的學習機制，將有助於調整架構與訓練策略以達到更穩定且可控的性能。

**Method:** 作者首先構造一類多層Transformer架構，使每一層能精確執行一次正規化梯度下降步驟，完成in-context logistic regression。接著，透過訓練單層自注意力機制，並以單步梯度下降作為監督信號，獲得該層權重。最後，將訓練好的單層模型反覆迴圈應用，形成一個可重複使用的迴圈模型。文中同時提供了該自注意力層訓練收斂的理論保證，以及迴圈模型在分布外資料上的泛化保證，強化方法的理論基礎。

**Result:** 實驗部分明確設定了線性分類任務作為測試場景，並以標準的logistic regression作為基準比較。結果顯示，所構造的Transformer模型能有效模擬正規化梯度下降過程，達到與傳統優化方法相當的性能。訓練過程中，單層自注意力機制成功收斂，且迴圈模型在分布外資料上展現良好的泛化能力。全文未明確說明其他複雜任務或非線性資料的實驗結果。

**Conclusion:** 本文提出一種理論與實驗結合的方法，展示softmax Transformer如何透過多層結構實現in-context logistic regression，並以單層自注意力迴圈應用達成類似優化演算法的效果。此研究為理解Transformer的ICL機制提供了嚴謹的數學框架與實證支持，對於後續模型設計與理論分析具有啟發意義。然而，方法主要聚焦於線性分類，對於更複雜任務的適用性仍需進一步驗證。

**Research Gap:** 本文雖提供了理論保證與線性分類實驗，但缺乏對非線性或更高維度任務的驗證，限制了方法的泛化性評估。訓練與迴圈模型的計算成本與實際部署風險未被充分討論，對工程實踐的影響尚不明確。此外，全文未提供對失敗案例或模型在不同資料分布下表現波動的分析，缺少消融實驗以驗證各組件的重要性。未來研究應補足這些不足，以提升方法的完整性與實用價值。

### 4. Recursive Agent Optimization

- 分數：0
- arXiv: [2605.06639v1](http://arxiv.org/abs/2605.06639v1)
- 發表日期：2026-05-07
- 分類：cs.LG, cs.AI, cs.CL, cs.MA
- 入選理由：未命中正向訊號

**Intro:** 本文提出Recursive Agent Optimization (RAO)，一種用於訓練可遞迴代理的強化學習方法。遞迴代理能在推理時動態生成子代理，將任務拆解成子任務並遞迴處理，從而擴展上下文長度並提升解決更複雜問題的能力。RAO不僅在推理階段利用遞迴結構，也將遞迴納入訓練過程，讓模型學習何時及如何有效地委派子任務與溝通。此方法針對多種長上下文與複雜任務場景進行評估，展示了遞迴代理在訓練效率、泛化能力及執行時間上的優勢。

**Motivation:** 隨著大型語言模型代理被應用於軟體工程、研究輔助等複雜任務，這些任務往往需要處理長期依賴、龐大上下文及多階段探索。傳統單一代理受限於上下文窗口大小，難以有效處理長任務。遞迴代理透過分而治之策略，將大任務拆解成較小子任務，並利用多個子代理並行處理，理論上可突破上下文限制並加速推理。然而現有方法多將遞迴視為推理時的外掛結構，缺乏針對遞迴策略本身的訓練。RAO旨在解決此問題，通過強化學習讓模型學會如何在遞迴結構中有效委派與協調，提升實際工程與產品中長任務的處理能力。

**Method:** RAO訓練一個共享的語言模型策略，該策略在遞迴生成的執行樹中每個節點被實例化。模型需同時學會解決當前任務及生成子任務委派給子代理。代理行動空間擴展為包含非同步啟動子代理的函數，支持串行或並行執行子任務。RAO設計局部節點獎勵，結合自身任務成功信號與子代理成功率的委派獎勵，促進有效分工。訓練目標為多層深度任務分佈上的加權期望回報，並採用leave-one-out基線減少方差。為避免深度過多節點數量失衡，採用深度層級反頻率加權。此方法允許動態生成任務樹，無需固定層級或手動設計調度策略。

**Result:** RAO在三個基準測試中均優於單一代理基線。TEXTCRAFT-SYNTH中，RAO在8K上下文限制下成功率達95%，遠超單代理24%，且在無限制上下文下仍保持優勢，尤其在難題上成功率達88%。在OOLONG-REAL長上下文任務中，RAO突破32K上下文限制，處理超過55K字元輸入，平均獎勵顯著高於單代理。DEEPDIVE多跳研究任務中，RAO成功率提升16個百分點。RAO還展示了更快的訓練收斂速度及在可並行子任務場景下最高2.5倍的推理加速。實驗中代理能根據任務難度自適應遞迴深度，且消融實驗證明局部獎勵與深度加權對性能提升關鍵。

**Conclusion:** 本文提出的RAO方法成功將遞迴推理結構納入強化學習訓練，讓語言模型代理學會動態委派與協調子任務，突破上下文限制並提升複雜任務表現。RAO在多種長上下文及多階段任務中展現優越的成功率、訓練效率及推理速度，證明遞迴結構不僅是推理時的輔助，更應成為模型訓練的核心部分。限制方面，RAO目前僅訓練單一共享模型，尚未探討異質子代理協作，且實驗多集中於子任務與父任務相似的場景。未來需研究跨域遞迴策略遷移、多樣子任務類型的委派，以及更高效的訓練代理抽樣方法。

**Research Gap:** 本文雖展示RAO在多個基準的優勢，但仍有幾點不足。首先，實驗中遞迴深度有限，且多數子任務與父任務性質相似，缺乏對異質子任務（如檢索、驗證、實作等）的深入探討。其次，RAO訓練單一共享模型，未評估多模型異質代理協同的潛力與挑戰。第三，推理加速效果依賴任務是否可並行，對於高度序列依賴任務推理時間仍較長。第四，訓練過程中遞迴樣本數量不均問題雖有加權處理，但可能影響最終策略穩定性與泛化。最後，全文未明確說明在更大規模或更複雜真實場景中部署的成本與風險，缺乏失敗案例分析與泛化邊界的系統性評估。

**Intro:** 本文針對連續時間與連續狀態/動作空間的強化學習，提出一種發現並利用價值保持結構的方法。現有方法多聚焦於已知的對稱性或精確等變性，缺乏對更一般非線性運算子所定義的結構的探索。作者將連續強化學習建模為受控擴散過程，並透過李群作用與對應的拉回算子定義價值保持映射，提出VPSD-RL框架。該框架能發現精確及近似的價值保持結構，並將其整合於強化學習中以提升穩定性與數據效率。

**Motivation:** 連續強化學習在複雜環境中通常需要大量數據且對環境變異敏感，這限制了其實際應用。若能發現並利用價值保持的結構，則能在不同狀態與動作間共享經驗，降低學習複雜度並提升穩定性。現有方法多依賴預先指定的對稱性，無法自動發現更廣泛的結構，尤其在連續空間中更具挑戰。本文透過運算子層級的分析，提供一套可從數據中學習並利用這些結構的系統化方法，對工程與產品落地具實用價值。

**Method:** VPSD-RL核心在於學習無窮小生成元（infinitesimal generators）以滿足確定方程的殘差最小化，該方程源自李群作用下生成元與獎勵函數的交換條件。學習後，透過ODE數值積分將無窮小生成元指數化，得到有限變換。這些變換用於強化學習中，包含轉移樣本增強與變換一致性正則化。此方法結合了可微分環境模型擬合、生成元學習、流積分與結構引導的強化學習更新，理論上保證學習收斂與價值函數穩定性。

**Result:** 實驗涵蓋可解釋的受控擴散系統與MuJoCo連續控制基準。結果顯示，VPSD-RL能有效學習接近真實生成元的變換方向，減少價值函數非不變性，並在SymNav-15及MuJoCo多個任務中提升樣本效率與最終表現。在Ant-v4任務中，VPSD-RL在最終回報與學習曲線下面積均優於多種基線方法，且計算開銷適中。實驗同時驗證了理論中對生成元殘差收斂與價值穩定性的預測。

**Conclusion:** 本文提出的VPSD-RL框架從運算子層級定義並發現連續強化學習中的價值保持結構，將李群對稱性視為特例，並提供理論保證與實證支持。該方法能提升連續控制任務的數據效率與魯棒性，尤其在存在可重用結構但未知的場景中。整體而言，VPSD-RL為連續強化學習中結構發現與利用提供了一條可行且有理論基礎的路徑，對工程實踐與後續研究具參考價值。

**Research Gap:** 本文假設環境模型可微且生成元學習可達成，然而在高維或非平滑環境中，模型擬合與生成元學習的難度未充分探討。近似價值保持結構的穩定性分析依賴於價值函數的二階可微性，對於僅為黏性解的情況未明確說明。此外，生成元指數化的數值誤差及其對強化學習性能的影響僅在理論層面有界定，實際部署中可能面臨挑戰。實驗中對於不同類型結構的泛化能力、失敗案例及成本效益分析較少，限制了方法在更廣泛場景的適用性評估。全文未明確說明在部分觀測或非馬可夫環境下的表現與調整策略。
