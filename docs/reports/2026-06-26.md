### 1. PhysRAG: Enhancing Physics-Awareness in Video Generation via Retrieval-Augmented Generation

- 分數：7
- arXiv: [2606.26916v1](http://arxiv.org/abs/2606.26916v1)
- 發表日期：2026-06-25
- 分類：cs.CV
- 入選理由：+2 提及頂級機構：mit; +3 提及頂級會議：eccv; +2 提及程式碼可用; LLM rank #1 / 9/10: Introduces a novel Retrieval-Augmented Generation pipeline for physics-aware video generation with strong empirical validation, curated dataset, and code release, showing clear novelty and practical impact.

**Intro:** 物理感知的影片生成模型開發面臨重大挑戰，因為捕捉多樣的物理現象如熱力學、力學和光學非常困難。現有的影片生成模型雖然在視覺真實感上有進展，但往往缺乏對物理規律的遵循。本文提出PhysRAG，一種結合檢索增強生成（RAG）的新管線，旨在提升影片生成的物理感知能力。為解決高品質物理影片數據稀缺的問題，作者設計了基於WISA-80K資料集的兩階段數據篩選流程，並建立物理影片資料庫以注入物理知識。實驗在多個基準測試中展現出優異的視覺品質與物理規則遵循能力。

**Motivation:** 影片生成模型缺乏物理感知限制了其在真實世界場景中的應用，如機器人操作、自主駕駛和互動世界建模。現有方法分為顯式物理模擬和隱式數據驅動兩類，前者雖保證物理規則但泛化能力差，後者則難以精確控制物理指導。人類通過檢索過去經驗來理解物理現象，啟發本文提出利用檢索相關物理影片並將其物理特徵注入生成模型，以提升物理一致性。此方法有助於克服數據不足和物理知識難以直接編碼的問題，對工程和產品落地具有實際意義。

**Method:** PhysRAG採用檢索增強生成架構，首先從手工構建的物理影片資料庫中檢索與輸入文本提示相關的物理影片。資料庫包含17類物理現象，每類約10個高品質影片。檢索後，利用預訓練的VideoMAE V2編碼器提取影片特徵，並通過設計的Query Inject模組將物理知識以可學習查詢的形式注入Video Diffusion Transformer（DiT）模型中。該模組透過跨注意力機制選擇性抽取與生成任務相關的物理動態，並融合於DiT的中間表示中，提升生成影片的物理合理性。數據方面，採用兩階段篩選策略，先用文本質量評分篩除無關樣本，再用多模態檢驗確保文本與影片內容一致，最終得到約7K高品質訓練影片。

**Result:** 在PhyGenBench和VBench兩大基準上，PhysRAG展現出優越的物理正確性與視覺品質。PhyGenBench中，PhysRAG在物理正確性平均分達0.58，超越包括Kling和DiT-Mem等閉源及開源模型，尤其在熱力學和材料物理類別提升顯著。VBench評估中，PhysRAG在低階（顏色、風格、物體分類）及高階（主體一致性、空間關係、多物體處理）指標均優於基線Wan 2.2，低階平均分提升至65.48%，高階平均分達82.88%。消融實驗證實數據篩選、檢索機制及物理信息注入模組均對性能提升有關鍵作用。質性比較顯示PhysRAG生成的影片在複雜物理互動場景中更為真實自然。

**Conclusion:** 本文提出的PhysRAG通過結合檢索增強生成策略，有效提升了影片生成模型的物理感知能力。兩階段數據篩選確保了訓練資料的高品質與物理相關性，手工構建的物理影片資料庫和可學習查詢注入機制使模型能夠利用顯式物理先驗，從而在多項基準測試中取得領先表現。該方法在工程實踐中有助於生成更符合物理規律的影片，對機器人、模擬和互動系統等應用場景具有潛在價值。然而，全文未明確說明在更大規模或更複雜物理現象下的泛化能力及計算成本，未提供失敗案例分析。

**Research Gap:** 本文雖然在物理感知影片生成上取得進展，但仍存在若干不足。首先，物理影片資料庫規模有限（約170影片），且手工分類可能限制多樣性與擴展性，對於更複雜或罕見物理現象的支持尚不充分。其次，兩階段篩選雖提升數據質量，但未明確評估篩選標準對模型泛化的影響，也缺乏對錯誤檢索或物理知識注入失效情況的分析。此外，模型在實際部署時的計算資源需求及延遲未被討論，可能影響工程落地。最後，全文未提供對比其他檢索策略或物理知識注入方法的消融實驗，限制了對方法優劣的全面理解。

### 2. XMSE-Aware Adaptive Empirical Bayes Estimation

- 分數：3
- arXiv: [2606.26975v1](http://arxiv.org/abs/2606.26975v1)
- 發表日期：2026-06-25
- 分類：stat.ML, cs.AI, cs.LG, eess.SY, stat.ME
- 入選理由：+3 提及頂級會議：acl; LLM rank #2 / 8/10: Proposes a theoretically principled XMSE-aware empirical Bayes estimator with strong finite-sample guarantees and extensive benchmark validation, balancing novelty and credible evidence.

**Intro:** 本文探討經驗貝葉斯（Empirical Bayes, EB）估計在第二階風險尺度下的行為，特別是當核函數與真實參數不匹配時，EB估計可能比最大似然（ML）更差。傳統一階漸近理論無法充分解釋正則化何時有益或有害，故本文利用過量均方誤差（XMSE）分析，提出一種基於XMSE的混合估計器，透過在ML與EB之間插值調整，達成風險保護。全文理論嚴謹，並以系統識別中的有限脈衝響應模型為背景，結合多種核函數與超參數估計方法進行實驗驗證。

**Motivation:** 在工程與產品實務中，核基正則化EB估計常用於線性迴歸與系統識別，但其性能受核函數選擇影響甚大，錯誤的核可能導致性能退化。現有方法多依賴一階漸近風險，無法預測或避免此類退化。本文將XMSE分析從診斷工具轉為設計原則，提出混合估計器以自適應調整正則化強度，對工程師與研究者而言，這提供了一種理論支持的風險保護機制，有助於提升模型穩健性與泛化能力。

**Method:** 核心方法為定義一個混合估計器，透過權重ω在ML估計與基礎EB估計間線性插值。利用XMSE的二次形式，推導出閉式的oracle混合權重，保證在XMSE尺度上不劣於ML與EB任一端點。進一步提出基於有限樣本XMSE近似的plug-in實現，證明其一致性與oracle後悔界限。方法涵蓋緊湊核族、多參數核族及有限與增長字典的核選擇，並設計閾值邊界規則以處理零偏差邊界情況。實驗中結合SURE調整、硬選擇及trace校正等基線，驗證混合估計器在核錯配時能有效退回ML，且在核匹配良好時保留正則化優勢。

**Result:** 實驗涵蓋合成數據與公開系統識別基準（Silverbox與Cascaded Tanks），在核錯配的尾部不匹配情境下，混合估計器相較於基礎EB估計顯著降低均方誤差，且多數系統中優於ML。樣本數與信噪比掃描顯示plug-in混合估計器隨樣本增加趨近oracle權重。有限候選核選擇實驗證明該方法能以高概率選擇接近oracle的核。公開基準中，混合估計器能避免EB估計的性能退化，展現良好保護性，但SURE調整的plug-in組件在非線性模型錯配下存在校準失效，trace校正雖改善部分情況，仍無法完全修正。整體結果支持該方法作為一種風險保護機制。

**Conclusion:** 本文提出的XMSE-aware混合估計器將第二階風險分析從診斷工具轉化為設計準則，透過閉式oracle權重與plug-in實現，提供理論保證的風險優化與核選擇策略。該方法在核錯配時能自動退回ML，避免性能惡化，且在核匹配良好時保留正則化優勢。實驗涵蓋多種核與超參數估計方法，並在公開基準中驗證其保護性與有限樣本行為。限制包括plug-in組件的有限樣本校準偏差、非線性模型錯配下的校準失效，以及未涵蓋高維度參數增長情況。未來工作可針對隨機權重的XMSE展開、擴展至高維設定及更廣泛的模型類型。

**Research Gap:** 本文理論基於固定維度與二階漸近展開，未涵蓋參數維度隨樣本數增長的高維情況，缺乏對此類場景的理論保證。plug-in權重的有限樣本校準偏差是主要實務限制，尤其在非線性模型錯配時，trace校正仍無法完全修正校準失效，顯示plug-in組件的穩健性不足。隨機權重與估計器共同依賴數據導致的交叉矩問題未被理論控制，全文未明確說明此部分的風險影響。此外，實驗中未包含與最新閉式偏差估計器的比較，且合成數據多為高斯噪聲與已知噪聲方差，缺少對未知噪聲或非高斯擾動的驗證。公開基準中模型錯配明顯，方法表現保護性強但未展現明顯提升，顯示泛化能力與實際部署風險仍需進一步評估。

### 3. Geometric Gradient Rectification for Safe Open-Set Semi-Supervised Learning

- 分數：5
- arXiv: [2606.26973v1](http://arxiv.org/abs/2606.26973v1)
- 發表日期：2026-06-25
- 分類：cs.CV, cs.LG
- 入選理由：+3 提及頂級會議：eccv; +2 提及程式碼可用; LLM rank #3 / 8/10: Presents a novel gradient-level control method for open-set semi-supervised learning with solid theoretical motivation, extensive experiments, and code availability.

**Intro:** 本論文聚焦於開放集半監督學習（Open-Set Semi-Supervised Learning, OSSL）中，如何利用含有異常分布（OOD）樣本的未標記資料，同時維持對已知類別的分類性能。現有方法多依賴樣本篩選或軟權重利用未標記資料，但這兩者在處理難以區分的異常樣本與困難內部分布樣本時，面臨過濾過嚴導致資訊流失或利用錯誤偽標籤引入對抗性梯度的問題。論文提出從樣本層面轉向梯度層面的控制，提出一種名為幾何梯度校正（Geometric Gradient Rectification, GGR）的框架。該方法以監督梯度作為錨點，將衝突的輔助梯度投影到允許區域，避免梯度方向相反，同時保留可能有用的正交成分。實驗涵蓋CIFAR與ImageNet資料集，展示GGR能提升多種OSSL基線的閉集及開集表現。

**Motivation:** 在實際應用中，未標記資料往往混雜未知類別，傳統半監督學習假設閉集設定不符實際。現有過濾方法為避免異常樣本干擾，可能錯誤剔除難以辨識的內部分布樣本，導致特徵匱乏；而利用方法則可能因偽標籤錯誤產生對抗性梯度，損害監督學習效果。這種樣本層面的二選一困境限制了OSSL的效能。作者觀察到，異常樣本的梯度多為與監督梯度正交的噪聲，而錯誤偽標籤的困難樣本則會產生直接反向的破壞性梯度。基於此，提出以梯度幾何關係為核心，透過梯度空間的投影來控制輔助梯度，避免破壞監督信號，提升學習穩定性與效能。

**Method:** GGR框架核心在於以監督梯度gs作為錨點，對輔助梯度gu進行投影，確保校正後的輔助梯度˜gu與gs的內積非負，避免方向相反。基礎方法為向量層級校正（Vector-Level Rectifier, VLR），當⟨gu, gs⟩<0時，移除gu中與gs反向的分量，保留正交成分。為提升穩定性，提出兩種子空間感知校正：正交子空間校正（Orthogonal Subspace Rectifier, OSR）完全移除輔助梯度在監督子空間的成分；錐形子空間校正（Conic Subspace Rectifier, CSR）則只移除與監督子空間方向相反的分量，保留正向成分。整體流程為：計算監督與輔助梯度，更新監督子空間基底，對輔助梯度進行校正，最後合併更新參數。此方法不需明確異常樣本檢測或修改基礎損失函數，具備即插即用特性。

**Result:** 實驗涵蓋CIFAR-10/100及ImageNet-30，採用多種開放集半監督學習基線方法（如OpenMatch、IOMatch、DAC），在不同標記數量及已知/未知類別比例設定下評估。結果顯示，GGR在大多數設定下均提升閉集準確率與開集平衡準確率，尤其在標記稀少及未知類別比例高的困難場景中效果更明顯。子空間校正方法在穩定性與性能間展現權衡，OSR較保守但穩定，CSR保留更多信號但較敏感。梯度衝突分析證實GGR有效降低輔助梯度與監督梯度的對立事件，減少優化損失。計算成本方面，GGR帶來約10%時間開銷，記憶體增加有限，具備實用性。校正範圍實驗顯示，根據基線方法不同，校正編碼器骨幹或整體模型均可獲得提升。

**Conclusion:** 本論文從優化幾何視角重新審視開放集半監督學習中的樣本識別瓶頸，提出以監督梯度為錨點的幾何梯度校正框架，有效避免輔助梯度對監督信號的破壞性干擾。理論分析證明校正後的更新方向保證監督目標的局部下降，實驗結果展示GGR在多種基線與資料集上均能提升閉集與開集性能，尤其在標記稀缺及異常樣本多的場景中效果顯著。GGR作為一種即插即用的優化層面控制方法，與現有異常檢測技術互補，為OSSL提供新的思路與工具。限制在於方法依賴局部梯度信息，無法保證全局最優，且在監督梯度高度噪聲時校正效果受限，仍需結合更強的異常檢測與偽標籤策略。

**Research Gap:** 論文主要驗證了局部梯度非對立性對提升OSSL的益處，但未充分探討該方法在更大規模、多樣化異常類型或更複雜任務（如語義分割、物件檢測）中的泛化能力。校正策略依賴監督梯度作為錨點，當監督梯度本身噪聲大或不穩定時，校正效果可能下降，子空間方法雖有緩解但仍有限。實驗中未明確分析GGR在不同偽標籤質量下的敏感度及失敗案例。成本分析僅限於時間與記憶體，未涵蓋實際部署中可能的延遲或能耗影響。最後，方法不涉及前向過程或損失函數設計，對於更複雜的異常樣本特徵提取與識別仍需依賴其他技術，整體系統整合與端到端效果尚待深入研究。

### 4. Semantic Early-Stopping for Iterative LLM Agent Loops

- 分數：5
- arXiv: [2606.27009v1](http://arxiv.org/abs/2606.27009v1)
- 發表日期：2026-06-25
- 分類：cs.AI, cs.LG, cs.MA
- 入選理由：+3 提及頂級會議：acl; +2 提及程式碼可用; LLM rank #4 / 7/10: Provides a theoretically grounded and empirically validated semantic early-stopping method for LLM agent loops, addressing a practical efficiency problem with reproducible code.

**Intro:** 本論文探討多代理大型語言模型（LLM）迭代循環中的停止策略，特別是Writer與Critic的反覆生成與修正過程。現有方法多以固定迭代次數作為停止條件，忽略了答案品質是否仍在提升，導致資源浪費或過早截斷。作者提出語義早停（semantic early-stopping）機制，透過衡量連續草稿的語義變化和答案品質變化來決定停止時機，並提供理論保證與實證驗證。論文同時提出一套高效的評估協議，方便公平比較不同停止策略。研究以HotpotQA多跳檢索增強問答為實驗場景，展示方法在節省運算資源的同時保持答案品質。

**Motivation:** 固定迭代上限作為停止條件在實務中常見，但此方法無法感知答案是否已收斂，容易在簡單問題上浪費計算資源，或在複雜問題上過早停止。這對工程與產品部署造成成本與效能的雙重挑戰。語義早停透過語義距離與品質指標的結合，能更精確判斷何時停止，降低不必要的計算與延遲，對提升多代理系統的效率與實用性有直接幫助。

**Method:** 核心方法是將每輪生成的草稿透過本地句子嵌入模型映射成向量，計算相鄰草稿間的餘弦距離作為語義變化指標。當連續k輪距離均低於閾值ε且答案品質（以多指標加權的Information Score衡量）不再提升時，觸發停止。停止決策依序檢查四個條件：Critic批准、語義收斂、品質無提升、以及硬性最大迭代限制。此停止邏輯以純函數實現，保證線上與離線分析結果一致。評估協議採用軌跡重播與快取判斷，分離運算代價與評估代價，確保公平且低成本的比較。

**Result:** 在HotpotQA測試集（60題）上，無需評分器的語義早停策略（entropy_only）相較於固定6輪迭代，節省約38%運算代價且答案品質無顯著下降（Information Score差異-0.004，p=0.81）。完整結合品質信號的策略反而因每輪評分成本過高，導致整體代價增加。固定3輪迭代策略節省53%代價且品質持平。最佳理論策略oracle_is能提升約0.115分，顯示現有停止信號尚無法精準定位最佳停止點。語義距離隨迭代趨勢下降但波動較大，需耐心窗口避免誤判。

**Conclusion:** 本研究提出一種基於語義變化與品質指標的早停策略，取代傳統固定迭代上限，實現了理論保證的終止性與公平的評估方法。實驗證明無需每輪評分即可大幅節省計算資源且保持品質，對多代理LLM系統的工程部署具實際價值。論文誠實面對理論假設限制，並指出現有停止信號無法有效捕捉最佳停止時機，將此問題重新定義為未來挑戰。

**Research Gap:** 本論文的實驗場景HotpotQA答案較短且多數問題可由單輪生成解答，導致迭代提升空間有限，限制了方法在長文本或內容累積任務的泛化性驗證。品質評分器為LLM代理，存在評分噪聲且未經人工驗證，影響品質差異的統計顯著性判斷。停止策略中品質信號的高成本問題未有有效解決方案，限制了其實務應用。論文未充分探討在不同模型規模、任務類型或多代理架構下的適用性與穩健性。缺乏對失敗案例或停止判斷錯誤的分析，無法評估系統在極端情況下的表現與風險。未提供學習式停止判斷器的實驗，未能驗證是否可縮小oracle與實際策略間的差距。

### 5. Ask, Don't Judge: Binary Questions for Interpretable LLM Evaluation and Self-Improvement

- 分數：5
- arXiv: [2606.27226v1](http://arxiv.org/abs/2606.27226v1)
- 發表日期：2026-06-25
- 分類：cs.AI, cs.CL
- 入選理由：+2 提及頂級機構：meta; +3 提及頂級會議：icml; LLM rank #5 / 7/10: Introduces an interpretable, training-free LLM evaluation framework decomposing criteria into binary questions, with strong empirical results and practical diagnostic value.

**Intro:** 大型語言模型（LLM）生成的內容評估一直是自然語言處理中的瓶頸。傳統的人類評估成本高且耗時，詞彙重疊指標與人類判斷相關性低，而整體的LLM評分往往不透明且難以診斷。本文提出BINEVAL框架，將評估標準拆解為原子級的二元問題，並將結果聚合成多維且可解釋的分數，旨在提升評估的透明度與可操作性。

**Motivation:** 在迭代開發過程中，準確且可操作的反饋對於比較不同提示、模型或解碼策略至關重要。單一的整體分數無法明確指出生成結果的具體問題，例如事實錯誤、相關性不足或流暢度差。透過將評估拆解成多個簡單的二元判斷問題，能夠提供更細緻且易於診斷的反饋，有助於提升評估質量並支持生成模型與評估器的優化。

**Method:** BINEVAL包含三個核心部分：首先，利用一個元提示(meta-prompt)將任務提示拆解成多個細粒度的二元問題，這些問題依據評估維度（如一致性、流暢度等）組織。其次，評估器LLM獨立回答每個二元問題並生成解釋，將答案聚合成每個維度及整體分數。最後，透過兩階段的迭代優化機制，利用問題級別的反饋來更新評估器和生成器的提示，支持自我更新及跨模型更新。

**Result:** 在SummEval、Topical-Chat及QAGS三個基準測試中，BINEVAL在多個維度上匹配或超越了包括UniEval和G-Eval等強基線，尤其在事實一致性評估（QAGS）表現突出。BINEVAL的分數分布更接近人類評分，避免了其他方法常見的天花板效應，能更好地區分邊緣與明顯錯誤的輸出。在迭代提示優化實驗中，BINEVAL在SummEval的評估器提示更新和IFBench的生成提示更新中均展現提升，尤其在格式及句子結構相關約束上效果明顯，但對於計數或比例等需要精確計算的約束提升有限。

**Conclusion:** BINEVAL提出了一種通用、無需訓練且可解釋的LLM輸出評估框架，通過將評估標準拆解為原子二元問題，實現了強勁的評估性能與實用的診斷及優化價值。該方法不僅提升了評估的透明度，也支持基於細粒度反饋的提示迭代改進。雖然在主觀性較強的評估維度如相關性上仍有挑戰，但BINEVAL為未來多輪及代理式評估提供了有價值的方向。

**Research Gap:** 本文方法依賴於生成的二元問題質量，若關鍵評估標準未被涵蓋，最終分數可能無法全面反映質量。對於主觀性較強的維度，如相關性，過度拆解可能導致評估過於嚴苛且與人類判斷不符。迭代提示更新在模型能力受限的情況下效果有限，尤其是需要精確計算能力的約束無法通過提示優化解決。此外，BINEVAL增加了模型調用次數與計算成本，對於大規模評估場景可能帶來效率挑戰。全文未明確說明在多輪對話或更複雜任務中的泛化能力及失敗案例，這些均為後續研究可深入探討的方向。
