### 1. Look Before You Leap: Autonomous Exploration for LLM Agents

- 分數：5
- arXiv: [2605.16143v1](http://arxiv.org/abs/2605.16143v1)
- 發表日期：2026-05-15
- 分類：cs.AI, cs.CL
- 入選理由：+2 提及頂級機構：mit; +3 收錄於 Hugging Face Daily Papers; LLM rank #1 / 9/10: Addresses a critical underexplored capability (autonomous exploration) for LLM agents with a novel metric and training paradigm, supported by systematic evaluation and clear practical impact on generalizable agents.

**Intro:** 本論文探討大型語言模型（LLM）代理在陌生環境中常見的過早利用問題，即代理過早依賴先驗知識而未充分獲取環境特定資訊，導致適應性不足。作者提出自主探索能力作為提升代理適應性的關鍵，並引入可驗證的探索指標——Exploration Checkpoint Coverage（ECC），用以量化代理對環境中關鍵狀態、物件及可用性發現的廣度。透過系統性評估，發現傳統以任務為導向的強化學習訓練往往導致代理行為狹隘且重複，限制下游任務表現。為此，論文提出交錯訓練策略與探索-執行（Explore-then-Act）推理範式，強調先行探索以獲取環境知識，再進行任務解決。

**Motivation:** 現有LLM代理多專注於任務完成獎勵，缺乏對自主探索的激勵，導致在新環境中容易陷入無效或重複行為，無法有效理解環境結構與限制。離線預先編譯環境知識的方法雖能提升特定場景表現，但無法應對多變且動態的真實世界環境。自主探索能力的培養能使代理在線上動態獲取環境知識，提升泛化能力與適應性，對工程實踐與產品部署具有重要意義。

**Method:** 作者將自主探索定義為無特定任務目標的資訊蒐集過程，並設計ECC指標衡量代理探索覆蓋率。訓練上採用Group Relative Policy Optimization（GRPO）框架，交錯進行任務執行與探索兩種rollout，分別以任務完成獎勵與ECC獎勵優化。推理階段提出Explore-then-Act範式，先以探索策略在固定互動預算內收集環境知識，並將該知識摘要注入後續任務執行策略中，實現探索與任務解決的解耦。

**Result:** 實驗涵蓋ALFWorld、ScienceWorld與TextCraft三種多樣化環境，評估多款開源及商用LLM。結果顯示，未經探索訓練的模型探索能力有限且常陷入重複行為，任務導向強化學習反而降低探索覆蓋率。引入ECC獎勵的探索訓練顯著提升探索覆蓋率與下游任務成功率，且Explore-then-Act範式在探索能力強的模型上帶來穩定性能提升。此外，探索訓練減少重複動作與循環失敗，提升錯誤恢復能力，並增強模型對環境變異的魯棒性。

**Conclusion:** 本論文明確指出自主探索為LLM代理適應陌生環境的關鍵能力，並提出ECC作為可驗證的探索指標。透過交錯GRPO訓練與Explore-then-Act推理範式，有效培養代理的探索能力並提升任務表現與環境適應性。實驗結果支持探索訓練不僅改善探索行為，也強化任務執行，為構建更通用且實用的智能代理提供了實證基礎與方法指引。

**Research Gap:** 論文雖提出明確的探索指標與訓練策略，但全文未明確說明ECC指標在不同環境中對探索質量的細節敏感度及其泛化性。實驗中探索與任務執行的交錯比例設定較為固定，缺乏對該超參數在多場景下的系統性調整與分析。此外，探索階段知識摘要的生成與利用過程細節有限，未充分探討其對任務執行策略的影響機制。成本與計算負擔方面，交錯訓練與雙階段推理可能增加部署複雜度，全文未評估實際應用中的效率與延遲問題。最後，缺乏對探索失敗案例及其對下游任務影響的深入分析，限制了對方法穩健性的全面理解。

### 2. RecMem: Recurrence-based Memory Consolidation for Efficient and Effective Long-Running LLM Agents

- 分數：5
- arXiv: [2605.16045v1](http://arxiv.org/abs/2605.16045v1)
- 發表日期：2026-05-15
- 分類：cs.CL, cs.AI, cs.LG
- 入選理由：+2 提及頂級機構：mila, mit; +3 提及頂級會議：acl; LLM rank #2 / 9/10: Proposes a novel recurrence-based memory consolidation method that significantly reduces token cost while improving accuracy, with strong empirical evidence and relevance for efficient long-running LLM agents.

**Intro:** 本論文針對長期運行的大型語言模型（LLM）代理中記憶系統的效率問題進行探討。現有記憶系統普遍採用積極的記憶整合策略，對每一次用戶互動都調用LLM進行資訊抽取，導致巨大的token消耗。作者提出RecMem，一種基於重複出現（recurrence）觸發的記憶整合機制，藉由引入一層輕量級的潛意識記憶層，僅在語義相似互動重複出現時才調用LLM進行記憶抽取與整合。此方法旨在降低記憶構建成本，同時保持或提升記憶準確度。

**Motivation:** 長期運行的LLM代理需要累積並管理大量用戶互動資訊，但LLM的上下文窗口有限，且對長文本中重要證據的利用率不足。現有記憶系統為避免遺漏資訊，對每次互動都進行LLM處理，造成高昂的計算與token成本，不利於實際部署。作者觀察到並非所有互動都值得立即整合，且認為重複出現的語義資訊更具價值，啟發其設計基於重複觸發的記憶整合策略，從而提升系統的效率與效果。

**Method:** RecMem採用三層記憶架構：潛意識記憶層、情節記憶層與語義記憶層。潛意識記憶層使用輕量級嵌入模型將每個用戶-助理互動單元向量化並存儲，支持高效檢索。當新互動與潛意識記憶中多個語義相似互動達到設定的相似度與重複次數閾值時，觸發LLM進行情節記憶的合併與摘要生成，形成事件級的連貫敘事。隨後，語義記憶通過語義精煉機制，從原始互動中補充細節，避免摘要遺漏關鍵事實。查詢時，系統從三層記憶中檢索相關內容，並將其與查詢一起輸入LLM生成回答。

**Result:** 在兩個長期記憶基準LoCoMo與LongMemEval-S上，RecMem與三個先進記憶系統Mem0、A-Mem及MemoryOS比較。結果顯示，RecMem在記憶構建階段的token消耗最多可降低87%，同時在整體問答準確率上超越所有基線方法。尤其在長上下文且需要時間推理的任務中，RecMem表現出明顯優勢。消融實驗證明潛意識記憶層對性能貢獻最大，語義精煉機制亦顯著提升了細節恢復能力。查詢階段的token消耗與其他方法相當，顯示節省主要來自構建階段。

**Conclusion:** 本文提出的RecMem通過引入基於重複觸發的記憶整合策略，有效降低了長期LLM代理中記憶構建的計算成本，同時保持甚至提升了任務性能。其三層記憶架構結合輕量級向量檢索與選擇性LLM抽取，模仿人類記憶的動態鞏固過程，為長期記憶系統設計提供了新的思路。限制方面，RecMem依賴手動設定的相似度與重複閾值，且基於重複出現判斷資訊重要性，可能忽略罕見但關鍵事件。未來工作可探索自適應觸發機制及更全面的資訊重要性評估。

**Research Gap:** 本研究尚未充分驗證RecMem在多樣化應用場景下的泛化能力，尤其是對於互動密度與語義結構差異較大的領域。手動設定的相似度與重複閾值雖有敏感度分析，但缺乏自動調整機制，可能影響部署靈活性。此外，基於重複出現的整合策略可能無法捕捉一次性但重要的資訊，且對於潛意識記憶層中未被整合的資訊，缺少更深入的跨層次推理支持。消融實驗雖展示各模組貢獻，但對比其他可能的記憶結構（如圖譜）及成本效益分析仍有限。全文未明確說明在極端長期運行或高頻交互場景下的系統穩定性與失敗案例。

### 3. Offline Semantic Guidance for Efficient Vision-Language-Action Policy Distillation

- 分數：5
- arXiv: [2605.16241v1](http://arxiv.org/abs/2605.16241v1)
- 發表日期：2026-05-15
- 分類：cs.CV, cs.AI
- 入選理由：+2 提及頂級機構：mit; +3 提及頂級會議：acl; LLM rank #3 / 8/10: Presents a practical distillation framework that substantially reduces model size and inference cost for vision-language-action policies, with strong empirical results and clear utility for real-time robotics.

**Intro:** 大型視覺-語言-動作（VLA）模型在機器人操作任務中展現出強大能力，但其龐大模型尺寸與推理延遲限制了實時閉環控制的應用。為了克服這些限制，本文提出一種名為VLA-AD的蒸餾框架，利用視覺-語言模型（VLM）作為離線語義監督，將大型VLA教師模型壓縮成輕量級學生模型。此方法在訓練階段引入高階語義指導，如任務階段錨點與多幀操作方向描述，提升學生模型的語義理解與穩定性。測試階段學生模型獨立運行，不依賴教師模型或VLM，實現低延遲部署。

**Motivation:** 現有大型VLA模型雖性能優異，但在邊緣設備上推理速度過慢，無法滿足機器人實時控制需求。傳統行為克隆蒸餾方法容易因資料分布偏移導致學生模型在閉環執行中累積錯誤，表現不佳。視覺-語言模型具備豐富的場景理解與語義推理能力，可作為輔助監督，幫助學生模型獲得更穩定且語義豐富的控制策略。本文旨在結合VLM的語義指導與教師模型的低階動作訊號，提升蒸餾後模型的效率與魯棒性，促進實際機器人系統部署。

**Method:** VLA-AD框架包含三階段：首先利用大型VLA教師模型在LIBERO任務中收集成功示範，取得原始視覺觀察與7自由度動作標籤。其次，利用VLM（如Qwen2.5-VL）生成語義豐富的階段錨點與多幀操作方向描述，作為輔助監督訊號。階段錨點將任務分為9個語義階段，提供穩定語義坐標，減少VLM描述的語言隨機性。多幀操作方向解決單幀視覺模糊的動作方向問題。最後，學生模型同時接受視覺輸入與VLM生成的語義描述，並以教師動作作為監督，採用雙路徑損失函數強化視覺與語義訊息融合。訓練完成後，學生模型可獨立推理，無需教師或VLM介入。

**Result:** 在LIBERO三個基準套件上，以OpenVLA-7B（7B參數）與π0.5-4B（4B參數）為教師，蒸餾出158M參數學生模型，分別達到44倍與25倍的模型壓縮。學生模型在RTX 4090上推理速度達12.5 Hz，較OpenVLA-7B快3.28倍，且在閉環成功率上與教師模型相差不到0.27%。跨教師實驗顯示相同蒸餾流程可泛化，學生模型在部分任務上甚至超越π0.5-4B教師。分析發現，階段錨點與多幀方向描述有效抑制教師模型中高頻率的抓取器指令噪聲，使學生模型輸出更平滑穩定的控制策略。

**Conclusion:** 本文提出的VLA-AD框架利用離線VLM語義監督，成功將大型VLA教師模型蒸餾成輕量且高效的學生模型，兼顧推理速度與閉環控制性能。階段錨點與多幀方向描述作為語義錨點，提升學生模型對操作階段的理解與魯棒性，並有效緩解教師示範中的噪聲問題。此方法不增加部署時的推理延遲，具備實際機器人系統應用潛力。限制在於目前僅在模擬LIBERO基準測試，尚未驗證於真實機器人或更複雜多任務場景，且語義監督依賴固定階段分類，可能限制對非結構化任務的適應性。

**Research Gap:** 本文蒸餾方法主要在模擬環境LIBERO基準上驗證，缺乏真實機器人部署的實驗，無法評估在真實感知噪聲與動態環境下的表現與穩定性。語義監督依賴固定的9階段分類與預設規則，對於更複雜或非結構化任務的泛化能力未明確說明。蒸餾過程中對不同階段描述的語言多樣性控制依賴啟發式規則，缺乏自適應或學習式方法，可能限制語義信息的豐富度。實驗中未充分展示不同超參數（如雙路徑損失權重α）對結果的敏感性分析，且未提供消融實驗以量化各語義監督組件的獨立貢獻。此外，蒸餾後模型在資源受限設備上的實際部署成本與能耗未討論，對工程落地的風險與限制仍需深入評估。

### 4. STABLE: Simulation-Ready Tabletop Layout Generation via a Semantics-Physics Dual System

- 分數：5
- arXiv: [2605.16137v1](http://arxiv.org/abs/2605.16137v1)
- 發表日期：2026-05-15
- 分類：cs.CV, cs.RO
- 入選理由：+2 提及頂級機構：mit; +3 提及頂級會議：icml; LLM rank #4 / 8/10: Introduces a novel semantics-physics dual system for simulation-ready tabletop scene generation, addressing LLM spatial reasoning limitations with a physics-aware correction module and demonstrating clear improvements.

**Intro:** 本論文聚焦於從任務指令生成可用於物理模擬的桌面場景佈局，這在具身人工智慧領域中具有重要應用價值。現有方法多依賴大型語言模型（LLMs）直接預測場景佈局，但因其三維空間推理能力有限，常導致物件碰撞或懸浮等物理不合理現象。為解決此問題，作者提出一套結合語義推理與物理校正的雙系統架構，旨在生成既符合語義需求又具物理合理性的桌面場景。該系統由語義推理器負責生成粗略佈局，物理校正器則利用幾何資訊進行姿態優化，兩者交替進行逐步擴展場景。實驗結果顯示本方法在物理合理性與任務對齊度上均優於現有技術。

**Motivation:** 在機器人操作等應用中，合成多樣且符合任務需求的桌面場景對訓練與評估具身智能系統至關重要。現有基於LLM的場景生成方法雖能理解語義，但缺乏對三維空間物理約束的考慮，導致生成的場景存在碰撞或懸浮問題，影響模擬的可用性。傳統後處理優化雖能改善物理合理性，卻計算成本高且可能破壞原始語義佈局，降低任務對齊度。因而需要一種能同時兼顧語義準確與物理合理的生成框架，以提升場景生成的實用性與穩定性。

**Method:** 提出的STABLE框架包含兩個核心模組：語義推理器與物理校正器。語義推理器基於微調過的LLM，將任務指令轉換為分階段的粗略場景佈局，先生成任務關鍵物件，再逐步加入背景物件，強化語義對齊與場景完整性。物理校正器則是一個基於流匹配的去噪模型，利用物件的三維點雲幾何特徵作為條件，對物件位置與旋轉進行連續空間的姿態修正。訓練時加入基於簽名距離函數(SDF)的碰撞損失與支撐接觸損失，確保物件間無穿透且穩定接觸。推理階段兩模組交替運作，逐步擴展並校正場景，避免錯誤累積並提升生成效率。

**Result:** 在MesaTask-10K資料集上，STABLE在多項指標上均優於包括MesaTask、I-Design-Table、Holodeck-Table、TabletopGen及Steerable等多種基線方法。具體表現為最低的碰撞率（0%）、最高的任務對齊度（AwT達99.4%）及較佳的視覺真實感（FID最低）。物理校正器在不同碰撞嚴重度的初始佈局中均能有效消除碰撞，優於傳統後處理優化方法。消融實驗證實物理損失函數與分階段生成策略對提升物理合理性與語義對齊均有顯著貢獻。此外，系統支持無需額外訓練的場景編輯，並在桌面場景重排任務中展現出較佳的物理一致性與功能關係保持能力。

**Conclusion:** 本論文提出的STABLE框架成功結合語義推理與物理校正，克服了純LLM生成方法在三維空間推理上的不足，並避免了後處理優化可能破壞語義結構的問題。其分階段生成與交替校正的設計有效提升了場景的物理合理性與任務對齊度，生成的桌面場景可直接用於物理模擬。實驗結果充分驗證了方法的優越性與穩定性，並展示了良好的編輯與重排能力。整體而言，STABLE為任務驅動的模擬場景生成提供了一種實用且高效的解決方案。

**Research Gap:** 全文未明確說明STABLE在更大規模或多樣化場景（如室內房間或戶外環境）中的泛化能力，僅聚焦於桌面場景。物理校正器雖能消除碰撞，但對於更複雜的動態交互或非剛體物件的處理未有探討。系統依賴高品質的3D資產庫與點雲編碼，對資源需求與部署成本未詳述。消融實驗中未展示不同語言模型架構或大小對語義推理器性能的影響，缺乏對模型選擇的敏感度分析。最後，對失敗案例或生成錯誤的具體分析較少，無法全面評估系統在極端或罕見任務指令下的魯棒性。這些均為後續改進與擴展的潛在方向。

### 5. From Flat Language Labels to Typological Priors: Structured Language Conditioning for Multilingual Speech-to-Speech Translation

- 分數：5
- arXiv: [2605.16026v1](http://arxiv.org/abs/2605.16026v1)
- 發表日期：2026-05-15
- 分類：cs.CL, cs.AI
- 入選理由：+2 提及頂級機構：mit; +3 提及頂級會議：acl; LLM rank #5 / 8/10: Proposes a structured typological prior approach for multilingual speech-to-speech translation, showing superior performance and data efficiency with thorough ablations and practical relevance.

**Intro:** 多語言語音到語音翻譯（S2ST）系統在跨語言溝通中扮演重要角色，尤其適用於醫療、教育及國際合作等場景。傳統S2ST多採用串接式架構，容易產生錯誤傳遞且無法整體優化。近年基於大型語言模型的語音模型（SpeechLLMs）推動了端到端及組合式S2ST的發展。現有系統多以平面語言標籤表示來源語言，忽略語言間的系統性結構，限制了在監督資料稀缺時的多語言適應能力。

**Motivation:** 平面語言標籤無法捕捉語言的形態、語序及語系關聯等結構性特徵，導致模型在多語言條件下難以有效共享參數與學習。這對於資料有限的多語言S2ST尤其不利。為提升資料效率及翻譯品質，需引入結構化的語言條件資訊，將語言視為具備系統性語言學特徵的實體，而非獨立標籤。

**Method:** 本文提出S2ST-Omni 2框架，將多語言語言條件從平面標籤轉換為結構化的語言類型學先驗。核心方法包含三層面：一是利用形態、語序、語系及語言特有殘差四個維度的分層語言編碼（TI-HLE）來結構化表示來源語言；二是設計動態門控的語言感知雙CTC模組，根據語言類型調節聲學特徵，並以雙重CTC損失輔助對齊；三是在解碼端透過語言類型感知的提示語，注入翻譯導向的語言學先驗。整體架構維持S2ST-Omni的組合式設計，語言類型模組僅於訓練時啟用，推理時不增加額外成本。

**Result:** 在CVSS-C資料集上，S2ST-Omni 2在BLEU、ASR-BLEU、COMET及BLASER 2.0等多項指標均優於多個代表性S2ST系統及原始S2ST-Omni，平均BLEU提升約5.8%，ASR-BLEU提升約4.6%。消融實驗顯示分層語言編碼、動態門控雙CTC及類型感知提示語均對性能有正向貢獻，且互補。資料量受限時（30小時以下）性能提升更明顯，且在日語→英語（約3小時監督資料）任務中也展現優勢，證明結構化語言條件有助於低資源及語言類型差異大的場景。不同TTS後端測試顯示性能提升不依賴特定語音合成器。

**Conclusion:** S2ST-Omni 2成功將多語言語音翻譯中的語言條件從平面標籤轉向結構化語言類型學先驗，透過分層語言編碼、動態聲學調節及解碼提示語三層策略，提升了多語言S2ST的資料效率與翻譯品質。實驗涵蓋多語言、多指標及低資源條件，證明該方法具備實用性與泛化潛力。此方法保持了組合式架構的模組化優勢，便於工程部署與後端替換。

**Research Gap:** 全文未明確說明該方法在多語言多目標（many-to-many）或低資源語言對的表現，僅聚焦many-to-one設定。語言類型學特徵的選擇較為粗糙，未探討更細粒度或自動化特徵學習的可能性。消融實驗雖有，但缺乏對不同語言類型間泛化能力的深入分析。成本與推理延遲方面的影響未詳述，尤其在實際部署中動態門控機制的計算負擔。最後，失敗案例與錯誤分析較少，無法充分評估模型在極端語言結構差異或口語化語料上的魯棒性。
