### 1. Code2LoRA: Hypernetwork-Generated Adapters for Code Language Models under Software Evolution

- 發表日期：2026-06-04
- 分類：cs.SE, cs.AI, cs.CL

**Intro:** 現有的程式碼語言模型在理解大型軟體專案時，需掌握倉庫層級的上下文資訊，如匯入、API及專案慣例。傳統方法多透過長輸入或逐倉庫微調LoRA來注入這些知識，但這在倉庫規模大且持續演進時，成本高昂且易失效。本文提出Code2LoRA，一種利用超網路生成倉庫專屬LoRA適配器的框架，能在推理時零額外token成本下注入倉庫知識。此方法針對穩定與演進中的程式碼庫，分別設計靜態與演進兩種使用場景。為評估效能，作者建立包含604個Python倉庫的RepoPeftBench基準，涵蓋靜態與演進兩種任務。實驗結果顯示Code2LoRA在多項指標上優於現有基線，並具良好泛化能力。

**Motivation:** 軟體專案通常包含大量檔案與複雜依賴，程式碼模型若無法有效整合倉庫層級知識，將難以完成如斷言補全等複雜任務。現有以長輸入方式注入上下文，會因上下文視窗限制與檢索成本而受限；而逐倉庫微調則在軟體頻繁變動時需反覆訓練，成本高且不實用。如何在不增加推理負擔的前提下，持續且高效地注入並更新倉庫知識，是工程與產品落地的關鍵挑戰。Code2LoRA透過超網路生成動態適配器，解決了這些瓶頸，對於持續演進的軟體專案尤為重要。

**Method:** Code2LoRA包含三大模組：共享的倉庫編碼器將整個倉庫或程式碼差異編碼成固定維度向量；超網路根據該向量生成LoRA適配器權重；凍結的基礎語言模型則注入該適配器進行推理。靜態版本Code2LoRA-Static直接將倉庫快照編碼映射為LoRA權重，適合穩定倉庫；演進版本Code2LoRA-Evo則利用GRU遞迴聚合連續的程式碼差異向量，動態更新適配器，適合活躍開發中倉庫。訓練時僅更新超網路參數，基礎模型與編碼器保持凍結，使用斷言補全任務的交叉熵損失優化。RepoPeftBench基準提供靜態與演進兩種評測軌道，涵蓋跨倉庫與倉庫內測試。

**Result:** 在靜態軌道上，Code2LoRA-Static達成63.8%跨倉庫與66.2%倉庫內Exact Match，超越所有上下文注入與微調基線，且匹配逐倉庫LoRA的上限。演進軌道中，Code2LoRA-Evo以GRU聚合差異，達成60.3%跨倉庫與64.5%倉庫內Exact Match，較單一共享LoRA提升5.2個百分點。上下文注入方法在演進軌道表現大幅下降。對於2025年後新倉庫的OOD測試，Code2LoRA-Evo仍保持最高表現，顯示良好泛化能力。多項指標（Edit Similarity、CodeBLEU）與執行測試均支持此結論。

**Conclusion:** 本文提出的Code2LoRA框架有效解決了程式碼語言模型在大型且演進中倉庫注入知識的挑戰，透過超網路生成動態LoRA適配器，實現零推理時上下文成本且持續更新。RepoPeftBench基準提供了豐富的評測場景，實驗結果證明Code2LoRA在靜態與演進任務上均優於現有方法，並具備跨倉庫與時間的泛化能力。此技術對於構建高效、可定制且成本可控的AI程式碼助理具備實際應用價值。限制包括僅評估Python語言、單一基礎模型及單一任務，未涵蓋更大模型規模與多語言、多任務場景，未進行全面語義執行評估。

**Research Gap:** 本研究限制於Python語言與Qwen2.5-Coder-1.5B模型，尚未驗證於其他語言或更大規模模型，泛化性仍待檢驗。演進軌道的GRU聚合是否在更大模型下仍必要或有效，全文未明確說明。評估任務僅限斷言補全，缺乏多樣化下游任務測試。Exact Match指標無法完全反映語義正確性，雖有輔助指標與部分執行測試，但全面語義評估缺失。超網路參數量龐大，實際部署成本與效率風險未充分討論。缺乏對失敗案例、模型錯誤類型及對私有倉庫安全性的深入分析。未探討模型在極端軟體演進速度或大規模倉庫下的穩定性與效能。

**Intro:** 本論文聚焦於影片語意定位中的一個重要但未充分探討的問題：一對多時間定位（One-to-Many Temporal Grounding, OMTG）。傳統研究多聚焦於單一片段對應單一查詢的場景，然而現實中同一語意事件常在影片中多次出現，需定位多個不連續片段。此問題對於影片理解、檢索及編輯等應用極具實用價值，卻因現有多模態大型語言模型（MLLMs）缺乏事件數量感知而表現不佳。論文提出系統性解決方案，包含新基準、資料集及訓練策略，旨在彌補此技術空白。

**Motivation:** 現有時間定位任務多以一對一為假設，無法處理多次重複事件的定位需求，導致在真實影片分析中無法完整捕捉事件全貌。傳統評估指標如tIoU無法有效反映事件數量錯誤，容易被高重疊率掩蓋錯誤結果。頂尖MLLMs在OMTG任務上表現接近零，顯示技術缺口明顯。為提升影片語意理解的完整性與精確度，迫切需要針對OMTG設計專門的資料、評估指標與訓練方法。

**Method:** 論文提出三大核心貢獻：首先，建立首個完整OMTG基準，設計Count Accuracy與Effective Temporal F1等新指標，嚴格評估事件數量與定位精度。其次，透過五階段自動化且嚴格驗證的資料建構流程，從多個公開影片資料集產出56k高品質一對多標註樣本。最後，採用兩階段訓練策略，先以監督式微調（SFT）建立基礎能力，再以強化學習（RL）優化，設計結合時間定位、事件數量與密集字幕質量的複合獎勵函數，利用Chain-of-Thought推理提升模型對事件結構的理解與定位準確度。

**Result:** 在OMTG基準上，作者模型OMTG-4B達到43.65%的Effective Temporal F1，較現有最佳專有模型Gemini 2.5 Pro與Seed-1.8分別提升超過15%。開源模型多數在Count Accuracy為零，顯示無法正確辨識事件數量。訓練過程中，SFT階段已大幅提升性能，RL階段進一步優化定位與計數能力。消融實驗證實時間獎勵、計數獎勵與字幕獎勵三者結合效果最佳。模型在傳統一對一時間定位任務中亦有穩定提升，顯示OMTG訓練有助於泛化。

**Conclusion:** 本研究首次系統性定義並解決一對多時間定位問題，填補現有多模態影片理解技術的關鍵缺口。透過新基準、資料集與訓練策略，成功提升模型對多重事件的感知與定位精度，並提出更嚴謹的評估指標。該方法不僅提升OMTG任務表現，也促進一對一時間定位能力。限制在於訓練成本高昂，對超長影片的擴展性仍有限，未來可探索記憶機制與更高效訓練策略。

**Research Gap:** 論文未充分說明模型在極長影片或事件極度稀疏情況下的表現與可擴展性，訓練成本與推理效率對實際部署仍具挑戰。資料集雖涵蓋多領域，但仍以中短影片為主，缺乏對更複雜場景如多重重疊事件的驗證。獎勵函數設計雖有效，但對不同權重配置的敏感度與泛化能力未深入探討。缺少對失敗案例的分析，無法全面理解模型在何種情況下仍易出錯。此外，全文未明確說明模型對於語意模糊或多義查詢的處理能力。

**Intro:** 本文探討在重複博弈中，面對能根據歷史行為調整策略的適應性對手時，如何進行遺憾最小化。傳統的外部遺憾指標無法有效反映對手的適應性，導致學習策略可能陷入次優均衡。為此，作者提出一種新的遺憾度量——重複策略遺憾（RP-Regret），用以衡量在所有玩家均可根據歷史調整策略時，實際累積效用與事後最佳策略累積效用之差異。此指標更貼合重複博弈的本質，允許更強的比較策略與更少限制的對手策略空間。

**Motivation:** 在工程與產品層面，重複博弈廣泛應用於多智能體系統與策略互動場景，然而傳統外部遺憾指標忽略對手的適應性，導致學習算法無法達成更高效或合作的均衡。研究如何在適應性對手存在下有效最小化遺憾，對於提升多智能體系統的穩定性與效用具有重要意義。此外，該研究有助於設計能在實際動態環境中穩健運作的算法，促進理論成果向實際應用的轉化。

**Method:** 核心方法為定義並分析RP-Regret，該指標允許比較策略隨時間動態變化，並考慮對手基於歷史的策略調整。作者證明要實現RP-Regret的次線性增長，必須滿足比較策略變化緩慢與有限記憶（不完美回憶）兩大條件。針對RP-Regret非凸優化問題，提出三種算法：(1) 基於非凸優化oracle的方法；(2) 利用局部線性化代理指標LRP-Regret，透過投影梯度下降實現；(3) 在對手策略變化緩慢時，將重複博弈轉化為馬可夫博弈，並以占用測度空間進行凸優化。這些方法兼顧理論保證與計算可行性。

**Result:** 實驗部分以Stag-Hunt等博弈展示，最小化RP-Regret及其代理指標能促成更合作且效用更高的策略結果。理論上，作者證明當所有玩家均能有效最小化RP-Regret或LRP-Regret時，可學習到近似子博弈完美均衡（SPNE）或子博弈完美粗相關均衡（SPCCE）。此外，提出的算法在滿足必要條件下，能保證RP-Regret及約束違反的次線性增長，支持其在實際多智能體學習中的應用潛力。

**Conclusion:** 本文提出RP-Regret作為適應性對手環境下的遺憾度量，並系統分析其可最小化的必要條件與算法設計。透過三種不同策略，兼顧理論嚴謹性與計算實用性，並建立RP-Regret與重複博弈子博弈完美均衡的關聯。該工作為多智能體學習中處理適應性對手提供了新的理論工具與算法框架，對推動博弈論與在線學習的交叉研究具有價值。限制在於部分算法依賴非凸優化oracle，計算成本與實際部署挑戰仍待進一步探討。

**Research Gap:** 本文雖提出RP-Regret及相關算法，但對於非凸優化oracle的依賴限制了實際可行性，且未充分探討算法在大規模或高維策略空間的擴展性。對手策略變化緩慢的假設較強，未明確說明在快速變化或非有限記憶對手下的表現與穩定性。此外，實驗場景較為有限，缺乏多樣化博弈類型及真實應用場景的驗證。消融實驗與不同比較策略的影響分析不足，對算法在非理想條件下的魯棒性與泛化能力尚未充分揭示。未來工作可聚焦於降低計算複雜度、放寬對手行為假設及擴展至更廣泛的多智能體環境。

**Intro:** 本論文探討離散擴散語言模型（Discrete Diffusion Language Models, DLMs）在檢索增強生成（Retrieval-Augmented Generation, RAG）中的應用。傳統自回歸生成方法因逐字生成導致延遲與檢索受限於已生成文字，難以有效處理多跳推理問題。擴散模型則透過平行迭代去噪整段回應，能在生成過程中提前揭示潛在關鍵詞，為檢索提供前瞻訊號。論文提出SARDI框架，利用擴散模型中低信心但有用的暫定詞彙引導動態檢索，提升多跳問答效能。

**Motivation:** 多跳問答需要依賴中介實體來逐步檢索證據，傳統靜態檢索無法在生成過程中動態調整查詢，導致檢索效果受限。自回歸模型的檢索只能基於已生成文字，無法利用未來生成的潛在資訊。擴散模型的非自回歸結構允許同時預測所有位置的暫定詞，這些詞彙即使信心不足，仍能揭示重要實體，為檢索提供更早的線索。這種結構優勢若能被利用，將有助於提升檢索質量與生成速度，對工程部署和產品響應時間有直接影響。

**Method:** SARDI框架在每次擴散去噪迭代中，根據模型對每個位置的信心分數，將信心高於查詢閾值τq的暫定詞彙組合成檢索查詢，並刷新檢索結果作為下一步生成的上下文。信心高於承諾閾值τc的詞彙則被正式承諾為生成結果。τq低於τc，允許尚未穩定的詞彙提前影響檢索，實現檢索與生成的動態交互。此方法不需額外訓練，且對檢索器無依賴，適用於任何具備推理能力的離散擴散語言模型。實驗中使用BM25及E5-base-v2兩種檢索器驗證方法泛用性。

**Result:** 在五個多跳問答基準（2WikiMultiHopQA、HotpotQA、MuSiQue、CofCA、SynthWorlds-RM）上，SARDI相較於靜態檢索的擴散模型，Exact Match提升顯著，例如2WikiMultiHopQA從44%提升至59%。與多種自回歸訓練免費檢索基線相比，SARDI在準確度相當或更優的同時，推理速度提升最多達8倍。實驗證明擴散模型中早期暫定詞彙能有效提前檢索關鍵證據，且檢索增強生成降低了相鄰詞彙間的依賴性，促進平行解碼。不同檢索器替換及檢索更新頻率調整的消融實驗顯示方法穩健且可調節準確度與吞吐量平衡。

**Conclusion:** 本論文提出的SARDI框架利用擴散語言模型的去噪軌跡作為檢索的前瞻信號，實現了檢索與生成的動態交互，顯著提升多跳問答的準確度與效率。此方法無需額外訓練，且對檢索器無依賴，具備良好工程落地潛力。實驗結果支持擴散模型在RAG場景下的結構優勢，並展示了平行解碼與動態檢索的協同效應。限制在於目前擴散模型需輕度微調以產生推理軌跡，且每步檢索帶來的計算成本需進一步優化。未來擴散模型成熟後，SARDI有望進一步提升性能與實用性。

**Research Gap:** 本研究依賴擴散模型產生推理軌跡，然而目前需透過輕度微調才能達成，尚未驗證純prompting下的效果，限制了方法的即時應用。每次去噪步驟都刷新檢索帶來較高計算成本，雖有初步分析顯示檢索結果穩定，但缺乏完整系統優化方案。實驗集中於多跳問答，尚未涵蓋其他生成任務或更大規模語料庫，泛化能力與失敗案例未充分探討。此外，對於不同擴散模型架構或潛在擴散模型的適用性全文未明確說明。最後，與強化學習訓練的檢索代理相比，SARDI雖速度快但在極端複雜查詢上的表現及結合可能性仍待深入研究。

**Intro:** 大型語言模型透過生成明確的 chain-of-thought (CoT) 來提升推理能力，顯示中間計算的重要性。然而，文字形式的 CoT 受限於離散、序列且面向通訊的 token 流，每一步推理必須先被文字化，造成效率低下。潛在推理以連續且緊湊的狀態進行中間計算，提供更高頻寬的替代方案，但現有方法往往犧牲了 CoT 在自回歸模型中的原生優勢，如左到右生成、機率採樣、KV-cache 解碼相容性及可計算的似然估計。本文提出 NF-CoT，利用 normalizing flows 在 LLM 主體內建模連續思考，保留上述優勢並提升推理效率。

**Motivation:** 文字 CoT 雖然直觀且可解釋，但因為推理過程必須逐步文字化，導致推理成本高且速度慢。潛在推理能在連續空間中進行中間計算，理論上能更有效率，但現有潛在方法缺乏對推理軌跡的明確機率建模，且多數無法直接支援自回歸生成與 KV-cache，限制了實際部署與優化的可能。NF-CoT 旨在解決這些問題，讓潛在推理既保有自回歸語言模型的優勢，又能高效且可優化地生成連續推理狀態，對工程與產品落地有直接幫助。

**Method:** NF-CoT 首先利用一個預訓練的 VAE 編碼器將明確 CoT 文字轉換為連續潛在向量，接著透過一組淺層可逆 normalizing flow 模塊將該潛在向量重新參數化為一組更適合自回歸建模的連續思考向量。這些連續思考向量與答案文字序列共享同一 LLM 主體，分別由 NF 頭與 LM 頭負責生成與評分。訓練時同時最大化連續思考的似然與答案的交叉熵，推理時則直接在連續思考空間以左到右方式採樣，並利用同一 KV-cache 解碼答案。此設計支持端到端的似然訓練與基於策略梯度的強化學習優化。

**Result:** 在多個 Python 程式碼生成基準（MBPP、HumanEval 及其擴充版本等）上，NF-CoT (Unified) 相較於基礎模型 Qwen3-8B-Base 提升平均 pass@1 約 13%，並超越先前最強的潛在推理基線 LaDiR 約 7%。NF-CoT 在推理效率上也優於 LaDiR，推理時間減少約一半，計算成本降低約兩倍。強化學習階段利用執行回饋進一步提升性能，且不犧牲多樣性，維持 pass@k 隨樣本數增加的擴展性。消融實驗顯示兩階段訓練與統一架構設計對性能有顯著貢獻。潛在空間的局部擾動實驗證明 NF-CoT 學得的連續思考空間平滑且功能穩定。

**Conclusion:** NF-CoT 提出一種在 LLM 主體內部以 normalizing flows 建模連續推理軌跡的方法，兼顧了自回歸語言模型的生成與評分優勢，並有效提升推理效率與準確率。實驗證明其在程式碼生成任務上優於多種基線，且支持基於似然的強化學習優化。此方法為潛在推理提供了實用且可擴展的框架，對工程部署與後續優化具有實際價值。限制在於目前僅驗證於程式碼生成，且依賴固定長度的 VAE 編碼，潛在空間不可解釋，且對非程式碼領域的泛化仍待驗證。

**Research Gap:** 全文主要驗證於程式碼生成任務，缺乏對其他推理任務如自然語言推理或多模態推理的實驗，限制泛化性評估。VAE 編碼固定長度且基於明確 CoT，可能限制模型對不同問題複雜度的適應性。潛在空間雖可平滑擾動，但對失敗案例、錯誤推理的分析不足，且連續思考不可解釋，對可解釋性需求的應用有限。強化學習階段訓練步數較少，長期穩定性與收斂性未充分探討。成本與部署風險方面，雖較 diffusion 方法更優，但仍需評估在大規模生產環境的實際效益與資源消耗。消融實驗集中於架構與訓練策略，對不同 VAE 編碼器或 flow 設計的比較較少。
