### 1. LychSim: A Controllable and Interactive Simulation Framework for Vision Research

- 分數：8
- arXiv: [2605.12449v1](http://arxiv.org/abs/2605.12449v1)
- 發表日期：2026-05-12
- 分類：cs.CV
- 入選理由：+3 收錄於 Hugging Face Daily Papers; +3 提及頂級會議：cvpr; +2 提及程式碼可用; LLM rank #1 / 9/10: LychSim offers a novel, highly controllable simulation framework with strong practical utility for vision research, including OOD evaluation and interactive LLM integration, supported by code release and multiple downstr

**Intro:** 本論文提出LychSim，一個基於Unreal Engine 5的高度可控且互動的視覺研究模擬框架。隨著自監督預訓練減少了對合成數據的依賴，模擬仍然是閉環優化與嚴格的異常分布評估不可或缺的工具。現有模擬平台通常技術門檻高，需要豐富的電腦圖形與遊戲開發經驗，LychSim旨在降低這些障礙，促進視覺研究者使用高品質3D環境進行實驗與開發。論文詳細介紹了系統設計、Python API與大型語言模型（LLM）集成，並展示多個下游應用案例。

**Motivation:** 儘管自監督學習大幅降低了對標註合成數據的需求，模擬環境仍在視覺系統分析與閉環訓練中扮演關鍵角色。模擬能提供完美對齊的2D與3D標註，並能創造多樣且可控的異常分布場景，這是現實數據難以達成的。此外，對於具身AI與機器人等需要環境互動的任務，高保真且可互動的模擬環境是安全且高效的訓練場域。現有平台複雜且不易使用，限制了視覺研究者的應用與擴展，故有必要開發一套易用且功能強大的模擬框架。

**Method:** LychSim核心包含三大設計：一是簡化的Python API，抽象底層Unreal Engine 5與C++複雜性，統一操作不同3D資產類型，降低使用門檻；二是程序化數據生成管線，利用高品質UE5資產與場景，結合場景層級規則與物件姿態標註，自動生成多樣且具挑戰性的異常分布視覺場景，並提供豐富的2D與3D標註，包括深度圖、分割、表面法線、點雲、物件遮擋與截斷比例等；三是原生整合Model Context Protocol (MCP)，使得大型語言模型能以閉環方式與模擬環境互動，實現動態場景規劃與物件操作。系統支持多種下游應用，如合成數據生成、強化學習對抗測試與語言驅動的場景佈局。

**Result:** 論文通過三個案例展示LychSim的實用性：首先，作為合成數據引擎，生成多樣且高保真的2D/3D標註數據，用於診斷空間視覺語言模型的弱點及後續訓練；其次，利用強化學習訓練對抗測試者，系統性探索3D視角以揭露Segment Anything模型的分割弱點，實驗中以IoU指標評估並成功捕捉失敗案例；第三，結合agentic LLM與MCP實現互動式場景規劃與生成，模型能根據自然語言指令多輪調整場景佈局，雖存在物理不合理佈局與物件碰撞等失敗案例，但展現了語言驅動3D設計的潛力。全文明確說明實驗設置、基線與評估指標，並提供程式碼與數據註釋公開計畫。

**Conclusion:** LychSim提供一個易用且功能豐富的視覺研究模擬平台，結合簡化Python接口、程序化數據生成與agentic LLM閉環互動，填補了現有高保真模擬系統使用門檻高與互動性不足的缺口。系統不僅支援多種視覺任務的標註需求，還能用於模型弱點分析與語言驅動的場景生成，對視覺與多模態研究具有實際工程與研究價值。作者計畫公開完整程式碼與數據標註，促進社群使用與擴展。全文未明確說明在大規模真實世界部署的效能與成本，且互動場景規劃仍受限於現有LLM的空間推理能力。

**Research Gap:** 本論文雖展示多樣應用，但缺乏對模擬生成數據與真實數據域差異的系統性分析，對模型泛化性的影響未充分驗證。強化學習對抗測試僅針對單一模型與有限物件類別，缺少跨模型與多樣場景的比較。互動式場景規劃案例中，物理不合理佈局與物件碰撞問題顯示現有LLM空間推理能力不足，未提供有效解決方案。系統的計算成本、運行效率及在大型場景下的擴展性未明確說明。最後，缺少對失敗案例的深入分析與用戶體驗評估，限制了對實際工程部署風險的判斷。
