### 1. LightSTAR: Efficient Visual Document Retrieval via Lightweight Selection with Vision-Adaptive Refinement

- 分數：5
- arXiv: [2606.23539v1](http://arxiv.org/abs/2606.23539v1)
- 發表日期：2026-06-22
- 分類：cs.CV
- 入選理由：+3 提及頂級會議：eccv; +2 提及程式碼可用; LLM rank #1 / 9/10: Proposes a novel, practical framework (LightSTAR) that significantly improves visual document retrieval efficiency and accuracy, with strong empirical validation and code release, addressing a real-world bottleneck.

**Intro:** 視覺文件檢索需要在大型多模態文件庫中快速定位與用戶查詢相關的頁面。現有基於多模態大型語言模型（MLLM）的方法雖然準確，但對每頁均進行高成本編碼，導致計算資源消耗過大。用戶查詢通常包含關鍵詞，這些詞語在相關頁面中可直接觀察到，提供了快速篩選候選頁面的有效線索。本文提出LightSTAR框架，將檢索過程分解為兩階段：一是無需LLM的輕量視覺選擇，用於快速高召回候選集生成；二是基於視覺自適應的語義細化，對候選集進行精細匹配。實驗證明LightSTAR在保持準確度的同時，大幅降低了檢索延遲。此方法對於大規模文件檢索系統具有實際應用價值。

**Motivation:** 現有MLLM方法需對所有頁面進行重度編碼，計算成本高昂且不易擴展，尤其在長文檔和頻繁更新場景中不實用。用戶查詢多為關鍵詞導向，且相關頁面通常包含這些詞的可視文本，這為快速篩選提供了契機。如何利用輕量且高效的視覺文本特徵，快速過濾掉明顯不相關頁面，並將昂貴的MLLM推理集中於少量候選頁面，是提升檢索效率的關鍵。LightSTAR正是基於此觀察，通過分層計算策略實現準確與效率的平衡，滿足實際工程和產品對大規模多模態文件檢索的需求。

**Method:** LightSTAR包含兩個核心模組：第一階段為LLM-free視覺選擇，利用輕量的視覺編碼器和內容導向的查詢嵌入，過濾掉函數詞並聚焦於語義豐富的詞彙，通過一種尺度自適應的後期交互機制，對文檔圖像的patch進行相似度計算，快速產生高召回的候選集。第二階段為視覺自適應語義細化，對候選集使用共享的視覺編碼器和多模態大型語言模型進行精細語義匹配。該階段引入了區域自適應特徵融合策略，將文本敏感特徵與布局敏感特徵結合，並採用難度感知對比學習目標，強化對難區分負樣本的辨識能力。整體架構通過共享編碼器層和LoRA微調策略，實現了高效且穩定的訓練與推理。

**Result:** 在ViDoRe大規模視覺文件檢索基準上，LightSTAR在NDCG@5指標上達到89.1，超越多數先前MLLM方法，且模型參數量僅2B，顯著減少推理成本。延遲測試顯示，隨著文檔頁數從500增至7000，LightSTAR的檢索延遲增長趨勢遠低於VisRAG-Ret、ColQwen2.5和ColPali等競品，最高可達10倍加速。LLM-free視覺選擇模組Recall@100達97.8%，有效保留相關候選。消融實驗證實內容導向嵌入、尺度自適應交互、區域自適應融合及難度感知目標均對性能有顯著貢獻，整體架構在保持準確率的同時大幅提升效率。

**Conclusion:** LightSTAR提出了一種基於查詢關鍵詞特性，分層分配計算資源的視覺文件檢索框架。通過輕量的無LLM視覺選擇快速縮小候選範圍，再利用多模態大型語言模型進行精細語義匹配，實現了準確度與效率的良好平衡。實驗結果表明該方法在大規模多模態文件檢索場景中具備實際部署價值，為視覺文件檢索領域提供了可行的計算分配新思路。限制方面，全文未明確說明在不同語言或極端複雜布局文檔上的泛化能力，以及在更大規模動態更新場景下的實時性能表現。

**Research Gap:** 本文雖在ViDoRe數據集上展現優異性能，但缺乏對多語言、多樣化文檔類型（如手寫、低質量掃描件）及極端複雜布局的全面驗證。對於候選集大小的選擇策略及其對效率與準確度的權衡，全文未提供深入分析。難度感知對比學習雖提升辨識力，但對訓練穩定性及超參數敏感性未充分探討。部署成本與資源需求方面，尤其在邊緣設備或低算力環境下的適用性未明確。此外，對失敗案例及錯誤類型分析不足，限制了對模型魯棒性和改進方向的理解。未來工作可聚焦於這些方面以提升方法的全面性和實用性。

### 2. TROPT: An Open Framework for Unifying and Advancing Discrete Text Optimization

- 分數：5
- arXiv: [2606.23496v1](http://arxiv.org/abs/2606.23496v1)
- 發表日期：2026-06-22
- 分類：cs.LG, cs.CR
- 入選理由：+3 收錄於 Hugging Face Daily Papers; +2 提及程式碼可用; LLM rank #2 / 8/10: Introduces TROPT, a unified open-source framework for discrete text optimization that lowers adoption barriers and enables systematic comparison and advancement, supported by extensive studies and code.

**Intro:** 本文聚焦於離散文本觸發優化（discrete text-trigger optimization），即尋找能引導模型達成特定目標的離散文字序列。此技術在大型語言模型（LLM）紅隊測試、模型審計與解釋性研究中扮演重要角色。然而，目前離散優化器分散於不同研究代碼庫，且多數與特定模型、目標或領域綁定，導致採用與比較困難。本文提出TROPT，一個統一且模組化的開源框架，旨在降低離散文本優化器的使用門檻並促進其進展。

**Motivation:** 現有離散優化器因實現分散且缺乏標準化，造成工程整合與跨領域應用的高成本，阻礙了技術普及與創新。不同優化器版本繁多，缺乏可直接比較的基準，難以評估其真實效能。這些問題限制了安全紅隊測試等高風險應用中強大攻擊手法的快速部署與發展。TROPT旨在解決這些痛點，提供一個統一介面，促進優化器的可重用性、可擴展性與可比較性，對工程與產品落地具有實質幫助。

**Method:** TROPT框架將離散文本優化問題抽象為四個核心組件：模型、損失函數、優化器與輸入模板。用戶可自由組合這些組件形成優化配方（recipe），支持多種模型後端（如HuggingFace、OpenAI）、多種損失函數與多種優化策略。框架設計強調模組化與前後端分離，使得新增優化器或損失函數只需實作標準化API，並能即刻與現有組件兼容。TROPT內建超過30個優化配方，涵蓋15種優化器與15種損失函數，支持白盒與黑盒訪問模式。

**Result:** 作者利用TROPT進行多項實驗驗證其效用。首先，在LLM jailbreak任務中，對14種優化器在四款開源模型上進行大規模對比，發現部分較少採用的優化器（如PAL、MAC）在效能上顯著優於常用基線GCG。其次，對八種jailbreak優化策略的增強方法進行隔離比較，揭示替換目標響應字串能大幅提升攻擊普適性。最後，展示TROPT跨領域適用性，包括將LLM jailbreak優化器成功移植至語料庫中毒攻擊、通用觸發器逃避分類器，以及文本到圖像模型的提示恢復任務，均取得良好效果。

**Conclusion:** 本文提出的TROPT框架有效統一並標準化離散文本優化器的開發與執行，顯著降低了工程整合與跨領域應用的門檻。透過模組化設計，TROPT促進了優化器的快速比較與擴展，並支持多種模型與任務。實驗結果證明其在安全紅隊測試及其他文本優化場景中的實用性與靈活性。該框架有助於推動離散文本優化技術的民主化與持續進步，並為未來自動化優化器發現等研究提供基礎。

**Research Gap:** 本文雖展示TROPT在多領域的適用性與優化器比較，但優化器效能評估主要集中於LLM jailbreak任務，缺乏更廣泛的跨任務、跨模型系統性基準測試。優化器參數調校與比較指標的多樣性未充分探討，可能影響結果泛化。框架目前不涵蓋基於強化學習或代理系統的優化方法，限制了部分先進技術的整合。實驗中對黑盒模型的攻擊成功率雖有展示，但對部署成本、計算資源需求及實際應用風險缺乏深入分析。此外，對失敗案例與優化器在不同語言或文本類型上的表現差異未明確說明，這些均為後續改進方向。

### 3. Self-Compacting Language Model Agents

- 分數：5
- arXiv: [2606.23525v1](http://arxiv.org/abs/2606.23525v1)
- 發表日期：2026-06-22
- 分類：cs.CL
- 入選理由：+2 提及頂級機構：meta, mit; +3 收錄於 Hugging Face Daily Papers; LLM rank #3 / 8/10: Presents SelfCompact, a novel adaptive compaction method for language model agents that outperforms fixed-interval baselines with strong empirical evidence across multiple benchmarks and models.

**Intro:** 隨著語言模型應用於更長時間跨度的任務，生成的推理軌跡越來越長，導致上下文中累積大量過時或無用資訊，形成所謂的上下文腐敗問題。這會嚴重影響模型後續生成的質量，尤其在競賽數學和代理式搜索等長程推理場景中。現有的上下文壓縮方法多採用固定間隔或固定令牌數觸發，忽略了推理軌跡的結構，可能在子任務未完成時就進行壓縮，導致重要信息丟失。本文提出一種新的上下文壓縮框架SelfCompact，讓模型自身判斷何時以及如何進行壓縮，旨在提升長程推理的效率與準確性。

**Motivation:** 長時間推理和多輪代理交互會使上下文迅速膨脹，若不及時且合理地壓縮，會導致模型基於過時或錯誤信息生成，降低性能。固定間隔壓縮策略無法區分推理狀態，容易在推理中途丟棄關鍵信息，造成性能下降。如何讓模型具備判斷自身上下文狀態的能力，並在合適時機進行壓縮，是提升長程任務表現的關鍵。SelfCompact通過結合模型自我調用的壓縮工具與輕量級的判斷規則(rubric)，在無需額外訓練的情況下實現動態、結構感知的上下文管理，對工程和產品部署具有實用價值。

**Method:** SelfCompact包含兩個核心組件：一是模型在推理過程中可自主調用的壓縮工具，用於將累積的上下文摘要為更精簡的表示；二是一套輕量級的判斷規則(rubric)，定期檢查推理軌跡，判斷是否應該觸發壓縮。判斷條件包括子任務是否已完成或軌跡是否趨於收斂，避免在推理中途或卡住時壓縮。整個流程在推理階段執行，無需模型微調或外部監督。壓縮工具和判斷規則均由同一模型生成，通過附加提示詞實現，並利用KV緩存重用減少計算成本。該方法在多個競賽數學和代理搜索基準上進行評估，與固定間隔壓縮和無壓縮基線比較。

**Result:** 在競賽數學任務中，SelfCompact在四個Qwen模型和三個基準上表現優於固定間隔壓縮和無壓縮基線，最高提升達18.1個百分點，且在11/12個實驗組合中取得最佳結果。代理搜索任務中，SelfCompact在三個大型模型和三個基準上均優於其他方法，提升5至9個百分點，且每題令牌成本降低30%至70%。消融實驗顯示，僅有壓縮工具或僅有判斷規則均無法達到最佳效果，兩者結合才能有效避免不當壓縮。分析發現固定間隔壓縮常在子任務未完成時觸發，導致信息丟失，而SelfCompact能提前且準確地判斷壓縮時機，尤其在難題上效果更明顯。

**Conclusion:** 本文提出的SelfCompact框架通過讓模型自身判斷何時壓縮上下文，並結合輕量級判斷規則，有效解決了固定間隔壓縮忽略推理結構的問題。該方法無需額外訓練，能在多種模型和任務中提升性能並降低計算成本，對長程推理和代理系統的實際部署具有重要意義。SelfCompact揭示了模型在上下文管理上的元認知能力缺口，並展示了如何通過推理時的輔助規則彌補這一缺陷。

**Research Gap:** 本研究僅在公開權重模型上驗證，未涵蓋更先進的商業或閉源模型，這些模型可能具備更強的元認知能力，能自行判斷上下文腐敗。全文未明確說明在更複雜或多模態場景下的泛化能力及失敗案例。判斷規則設計依賴人工制定，缺乏自動學習或優化機制，可能限制適應性和擴展性。成本分析主要基於令牌計數和KV緩存重用，未充分探討實際部署中延遲和系統資源消耗。未涉及強化學習等訓練方法對判斷規則的潛在提升效果。未提供對比更多動態壓縮策略或更細粒度結構感知方法的實驗。

### 4. Teaching LLMs String Matching, Backtracking, and Error Recovery to Deduce Bases and Truth Tables for the Combinatorially Exploding Bit Manipulation Puzzles

- 分數：5
- arXiv: [2606.23672v1](http://arxiv.org/abs/2606.23672v1)
- 發表日期：2026-06-22
- 分類：cs.AI
- 入選理由：+2 提及頂級機構：mila, mit, nvidia; +3 提及頂級會議：acl; LLM rank #4 / 7/10: Innovative approach to bit manipulation puzzles using string similarity and backtracking, achieving top contest performance with clear methodology and strong empirical results.

**Intro:** 本論文針對NVIDIA Nemotron Model Reasoning Challenge中的Bit Manipulation Puzzle提出解決方案。該任務要求模型從輸入的二進位字串中推導出隱藏的邏輯規則，並應用於未見過的輸入。傳統大型語言模型（LLMs）在此類問題中表現不佳，因為需要模擬複雜的布林邏輯和算術運算，導致嚴重的幻覺問題。且位元操作的組合空間呈現指數爆炸，造成計算不可行。論文提出一種全新方法，放棄算術邏輯，改以字串相似度、結構化搜尋與自主錯誤恢復來解決問題。

**Motivation:** Bit Manipulation Puzzle的挑戰在於其龐大的搜尋空間及複雜的邏輯推理，傳統方法無法有效處理，導致LLMs容易產生錯誤推論。此問題對工程與產品落地具有重要意義，因為精確的演算法推理是許多AI應用的基礎。若能有效解決此類組合爆炸問題，將提升LLMs在嚴謹邏輯推理任務中的可靠性與實用性，並降低部署時的錯誤率與計算成本。

**Method:** 核心方法包括三大創新：首先，將邏輯閘推導重新定義為基底選擇問題，利用字串相似度（最小位元翻轉）來隔離原始轉換基底並推導真值表，避免複雜算術。其次，設計回溯深度優先搜尋（DFS）結合錯誤恢復機制，透過全局碰撞檢測確保邏輯一致性，遇錯即回溯。最後，強制將二進位字串以單位位元作為token，並透過動態遮罩模擬外部oracle反饋，訓練模型在生成過程中能假設、評估並回溯，實現互動式推理。整體流程分為基底生成與字串匹配、回溯搜尋與碰撞驗證、真值表合成與目標預測三階段。

**Result:** 在1,602個驗證謎題上，確定性求解器達成98.63%準確率，未解決的問題多因資料不足或規則不唯一。經過互動式推理SFT訓練的LLM在合成資料上達到96.13%準確率，略低於確定性上限。引入原始競賽資料微調反而稍微降低表現，推測是訓練步數不足導致模型生成行為偏離。模型能有效執行多次回溯，證明其具備動態錯誤恢復能力。錯誤分析顯示約半數失敗因上下文截斷，另一半因模型推理錯誤。整體結果在嚴格計算資源限制下展現出高度穩健性與效率。

**Conclusion:** 本論文證明透過將位元操作問題轉化為基底選擇與字串匹配問題，並結合嚴格單位位元token化與互動式推理SFT，LLMs能有效克服組合爆炸與推理錯誤。此方法避免了傳統算術推理的幻覺問題，並在有限計算資源下達成高準確率。提出的動態遮罩訓練策略為無需昂貴強化學習即可培養模型錯誤恢復能力提供了實用範例。該框架對工程實務與研究落地均具參考價值，尤其在需要精確邏輯推理的AI應用中。

**Research Gap:** 論文未明確說明模型在更大規模或不同類型位元操作謎題上的泛化能力，且實驗主要限於8位元輸入，缺乏對更長序列的驗證。回溯搜尋雖有效，但其計算成本與推理時間在實際部署中可能成為瓶頸，全文未提供相關分析。動態遮罩訓練依賴外部oracle反饋，實際應用時oracle的可用性與效率未被討論。此外，模型在面對高度不確定或資料不足情況下的行為與失敗案例缺乏深入探討。最後，混合原始資料微調導致性能下降，顯示訓練策略仍需優化以避免任務間干擾。

### 5. MORL-A2C: Multi-Objective Reinforcement Learning Reranker for Optimizing Healthiness in MOPI-HFRS

- 分數：3
- arXiv: [2606.23603v1](http://arxiv.org/abs/2606.23603v1)
- 發表日期：2026-06-22
- 分類：cs.LG
- 入選理由：+3 提及頂級會議：kdd; LLM rank #5 / 7/10: Addresses a significant public health problem with a novel multi-objective RL reranker improving health alignment in food recommendation, validated on benchmarks with corrected baselines.

**Intro:** 本論文針對美國普遍存在的飲食健康問題，指出現有推薦系統多偏重用戶偏好，忽略營養健康，導致健康飲食行為難以改善。MOPI-HFRS系統透過Pareto多目標優化同時考量偏好、健康與多樣性，但其靜態逐步優化無法捕捉飲食決策的序列性與累積效應。作者提出MORL-A2C，將推薦問題視為序列決策，利用強化學習方法優化健康與偏好的權衡，並基於MOPI-HFRS的凍結GNN嵌入進行多步重排序。此方法旨在提升健康指標，同時維持合理的推薦品質。

**Motivation:** 飲食健康問題嚴重且持續惡化，現有推薦系統未能充分考慮長期健康影響，僅優化短期偏好。MOPI-HFRS雖引入多目標優化，但其靜態方法無法反映推薦列表中項目間的相互作用與序列依賴，限制了健康與偏好間的平衡能力。強化學習能處理延遲回饋與序列決策，適合此類問題。透過序列決策，系統可動態調整推薦策略，提升健康指標，對工程與產品落地有實際意義。

**Method:** MORL-A2C基於MOPI-HFRS訓練好的GNN嵌入，將推薦視為一個K步的馬可夫決策過程(MDP)，每步選擇一個食物項目加入推薦列表。狀態包含用戶嵌入、已選項目平均嵌入、累積健康標籤覆蓋與當前步驟比例。動作空間為剩餘候選食物。獎勵為偏好與健康兩部分的加權和，權重β控制兩者平衡。採用Advantage Actor-Critic (A2C)演算法訓練策略網路與價值網路。為避免初期隨機探索失敗，先以行為克隆行為模仿基於GNN嵌入的點積排序器，作為策略的warm-start。訓練過程中凍結GNN嵌入，只更新策略與價值網路參數。

**Result:** 實驗在宏營養素與全營養素兩個基準上進行，評估指標包括Recall@20、NDCG@20與H-Score@20。首先，作者發現並修正了MOPI-HFRS原始評估管線中的錯誤，修正後基線性能大幅提升。與修正後基線比較，MORL-A2C在Recall@20與NDCG@20上有約2-4個百分點的輕微下降，但在H-Score@20上有超過23-25個百分點的顯著提升，顯示其能有效提升健康對齊度。結果證明序列決策與強化學習能更好地平衡健康與偏好，雖有排名品質的輕微犧牲，但換來健康指標的大幅提升。

**Conclusion:** 本研究提出MORL-A2C，透過多目標強化學習將健康與偏好推薦問題序列化，突破了MOPI-HFRS靜態優化的限制。利用凍結GNN嵌入與A2C演算法，並以行為克隆warm-start，成功優化了健康指標，同時保持合理的推薦品質。修正了基線評估錯誤，確保比較公正。實驗結果支持序列化策略在健康飲食推薦中的有效性。限制包括未納入多樣性獎勵、固定獎勵權重及離線評估，未來可擴展至動態權重、多樣性優化及線上反饋。

**Research Gap:** 本論文未將多樣性納入獎勵函數，導致推薦結果集中於少數食物，缺乏多樣性評估與優化。離線評估無法反映真實用戶的序列反饋與行為變化，缺乏線上或混合評估驗證。獎勵權重β為固定值，未考慮用戶或推薦位置的動態調整，限制個性化能力。缺少與非強化學習的簡單啟發式重排序基線比較，難以量化強化學習帶來的具體增益。此外，全文未明確說明模型在不同用戶群體或長期使用下的泛化性與穩定性，這些均為後續改進重點。
