### 1. Dep-LLM: Training-Free Depression Diagnosis via Evidence-Guided Structured Multi-factor with Reliable LLM Reasoning

- 分數：5
- arXiv: [2606.10796v1](http://arxiv.org/abs/2606.10796v1)
- 發表日期：2026-06-09
- 分類：cs.CL, cs.AI
- 入選理由：+2 提及頂級機構：mit; +3 提及頂級會議：acl; LLM rank #1 / 9/10: Clear novelty in training-free depression diagnosis using frozen LLMs with structured multi-factor reasoning and confidence modulation; strong empirical evidence surpassing supervised and commercial baselines; practical 

**Intro:** 本論文聚焦於自動憂鬱症檢測（ADD）任務，該任務在計算精神健康領域中極具重要性，但面臨兩大挑戰：一是臨床訪談對話長且涵蓋多主題，憂鬱症線索稀疏且分散，導致模型推理表層且不可靠；二是標註資料稀缺且訓練成本高，限制了監督式系統的實際部署。作者提出一種名為Dep-LLM的無需訓練的框架，利用冷凍的大型語言模型（LLM）模擬臨床精神科醫師的逐步推理過程，旨在解決上述問題。

**Motivation:** 現有方法在長對話中難以捕捉稀疏且複雜的憂鬱症線索，且零樣本提示往往產生膚淺且不具臨床依據的解釋。此外，監督式方法依賴大量標註資料和昂貴的訓練，臨床隱私限制使得資料難以取得，且高昂的計算資源需求阻礙了臨床落地。因而，開發一種無需額外訓練、可解釋且可靠的憂鬱症檢測系統，對工程部署和臨床應用具有重要價值。

**Method:** Dep-LLM包含三個階段：第一階段利用Chain-of-Thought（CoT）多因素分析模組，將長對話結構化分解為五個與SCID標準對齊的主題（家庭關係、工作滿意度、心理狀態、醫療史、整體評估），並生成基於證據的推理理由，有效處理長上下文依賴。第二階段引入語義置信度分析與調節模組，根據生成推理中每個token的熵值計算推理的認知置信度，並通過主題間和標籤內的調節放大可靠信號，抑制不確定信號，無需額外訓練。第三階段為協同多因素預測模組，根據置信度加權整合多因素信號，產生最終診斷結果。整個流程均在冷凍的基礎LLM上執行，無需任何參數更新或標註資料。

**Result:** 在DAIC-WOZ與E-DAIC兩大臨床訪談資料集上，Dep-LLM在21種不同基礎LLM（涵蓋Llama、Qwen、Gemma系列，參數規模4B至17B）上均顯著優於零樣本基線，涵蓋準確率、宏觀F1、加權F1等9項指標。與多個監督式領域專用LLM（如MentalBERT、MentaLLaMA）及最新閉源商業LLM（GPT-5.5、Claude-Opus-4.6等）比較，Dep-LLM在多數指標上表現更佳，且無需任何訓練。消融實驗證明CoT推理、多主題結構及置信度調節均為性能提升關鍵。超參數敏感度分析顯示系統對決策閾值及中性標籤校準參數具備良好魯棒性。

**Conclusion:** 本研究提出的Dep-LLM框架成功解決了長對話中憂鬱症線索稀疏且推理不可靠的問題，並克服了監督式方法對標註資料和訓練成本的依賴。其結構化多因素CoT推理結合基於token熵的置信度調節，實現了高解釋性與高準確度的無訓練憂鬱症檢測。實驗結果證明該方法在多種基礎LLM及資料集上均具優越性，且易於臨床部署，對精神健康AI應用具有實際價值。

**Research Gap:** 全文未明確說明Dep-LLM在更大規模、多語言或多模態（如語音、影像）資料上的適用性與泛化能力；缺乏對失敗案例或錯誤診斷原因的深入分析，限制了對系統穩健性的理解。置信度調節雖有效，但其在不同LLM架構及訓練背景下的普適性尚未充分驗證。系統對於極端或非典型臨床對話的表現未被探討，且未評估實際部署中計算資源消耗與延遲問題。此外，雖然無需訓練降低了成本，但對於臨床專家如何解讀與調整模型輸出仍缺乏具體指導。未來工作可聚焦於這些方面以提升系統完整性與實用性。

### 2. N-GRPO: Embedding-Level Neighbor Mixing for Enhanced Policy Optimization

- 分數：5
- arXiv: [2606.10768v1](http://arxiv.org/abs/2606.10768v1)
- 發表日期：2026-06-09
- 分類：cs.LG, cs.CL
- 入選理由：+3 提及頂級會議：acl; +2 提及程式碼可用; LLM rank #2 / 8/10: Novel exploration strategy (Semantic Neighbor Mixing) for policy optimization in LLM mathematical reasoning; demonstrated consistent improvements and robust generalization; code available; solid experimental validation.

**Intro:** 本論文針對大型語言模型在數學推理任務中，如何提升rollout階段生成多樣且有效解題路徑的問題進行探討。現有token層級的採樣方法容易產生語義重複的軌跡，而直接在embedding層注入隨機噪聲則會破壞語義一致性。為解決此問題，作者提出N-GRPO，一種結合語義鄰居混合的embedding層探索策略，整合於GRPO強化學習框架中。該方法透過混合錨點token與其語義鄰居的embedding，注入多樣性同時保持語義局部連續性。實驗涵蓋多種模型規模與數學推理基準，顯示N-GRPO在提升性能與泛化能力上均有穩定表現。

**Motivation:** 強化學習中rollout階段的探索質量直接影響策略優化效果。傳統token層級隨機採樣多半只能產生語義相近的重複路徑，限制了探索空間。另一方面，embedding層注入隨機噪聲雖能增加多樣性，但會導致語義漂移，破壞推理邏輯。這種探索與語義穩定性的矛盾，阻礙了模型在複雜推理任務中的表現提升。N-GRPO透過語義鄰居混合，提供一種在embedding層進行受控且語義合理的隨機擾動，旨在擴展探索範圍同時維持語義一致性，對工程上提升模型推理能力及訓練穩定性具有實際意義。

**Method:** N-GRPO核心在於Semantic Neighbor Mixing機制。每生成一步，先從當前logits中選出最高機率的錨點token，接著在embedding空間中以餘弦相似度找出其k個語義鄰居。將這些鄰居的embedding依logits權重加權混合，形成連續且語義局部一致的輸入embedding，供下一步生成使用。此混合過程以機率ρ控制，非每步皆執行，以平衡探索與穩定性。該機制整合於GRPO強化學習框架的rollout階段，保持原有優勢估計與PPO風格目標函數，並在訓練時記錄混合權重以確保可重現性。最終生成文本以錨點token替代混合步驟，確保獎勵計算基於離散序列。

**Result:** 實驗在多個數學推理基準（如AIME25、AMC23、MATH500）及不同模型規模（1.5B、7B）上進行，N-GRPO在Pass@16與Pass@32指標均優於基線GRPO及其他方法（如Soft Thinking、STHT）。在1.5B模型上，N-GRPO平均Pass@32提升至79.17%，較基線74.62%明顯增強。非蒸餾基礎模型（Llama-3.2-1B、Qwen3-1.7B-Base）亦展現穩定提升，顯示方法不依賴特定模型架構。對於OOD科學推理任務GPQA-Diamond，N-GRPO同樣取得最佳表現，證明其泛化能力。此外，將語義鄰居混合應用於GSPO訓練也帶來性能提升。消融實驗顯示混合率、距離度量（餘弦距離優於L1、L2）及混合策略對結果有顯著影響。推理階段若啟用混合反而降低表現，建議僅於訓練rollout使用。

**Conclusion:** 本研究提出一種嵌入層級的語義鄰居混合探索策略，有效提升LLM在強化學習中推理路徑的多樣性與質量。N-GRPO透過結合語義鄰居的embedding加權混合，避免了隨機噪聲帶來的語義漂移問題，並成功整合於GRPO框架。實驗結果證明該方法在數學推理及科學推理任務中均優於多種基線，且具備良好泛化性。限制方面，該方法增加了rollout階段的計算負擔，且尚未驗證於程式碼生成等其他任務。未來可探索其在不同任務類型及更大模型上的適用性與效率優化。

**Research Gap:** 本論文主要驗證N-GRPO於數學及科學推理任務的效果，尚缺乏對其他類型任務（如程式碼生成）的實驗驗證，限制了方法的通用性評估。混合率與鄰居數等超參數的選擇對性能有影響，但對其最佳設定的理論分析不足。方法引入的計算開銷及其對大規模部署的實際影響未充分討論。推理階段禁用混合策略，顯示訓練與推理目標存在差異，未明確探討如何縮小此差距以提升推理階段性能。此外，對失敗案例、語義漂移風險及長期訓練穩定性的分析有限，缺乏更全面的消融與對比研究。

### 3. Pushing the Limits of LLM Tool Calling via Experiential Knowledge Integration and Activation

- 分數：4
- arXiv: [2606.10875v1](http://arxiv.org/abs/2606.10875v1)
- 發表日期：2026-06-09
- 分類：cs.CL
- 入選理由：+2 提及頂級機構：mit; +2 提及程式碼可用; LLM rank #3 / 8/10: Systematic study of knowledge integration for LLM tool use with clear insights on acquisition, activation, and internalization; proposes a novel framework (KATE) with strong empirical gains and code release; balances nov

**Intro:** 本論文探討大型語言模型（LLMs）在多步驟工具調用過程中，因缺乏足夠的工具相關知識及知識激活機制，導致執行失敗的問題。作者系統性研究知識如何影響工具使用表現，涵蓋知識獲取、激活與內化三個階段。透過實驗分析不同層次的經驗知識對工具調用的影響，並提出一套整合經驗知識與推理擴展的工具執行框架KATE。該研究對提升LLM作為自主代理人執行複雜任務的能力具有實務意義。

**Motivation:** 現有LLM工具使用方法多聚焦於提示設計或API規範，假設模型已具備足夠的經驗知識，但實際多步驟工具調用失敗多因缺乏具體可執行的經驗知識，如參數限制、場景特定操作模式及錯誤恢復策略。如何有效獲取、激活及內化這些經驗知識，提升模型工具調用的準確性與穩定性，是工程與產品落地中亟需解決的挑戰。作者希望透過系統性研究填補此知識層面缺口，提升LLM在實際工具調用場景的表現。

**Method:** 作者將經驗知識分為實例層級（如場景軌跡知識與經驗摘要）與意圖層級（如腳本式與文本式意圖聚類知識），並建立結構化知識庫以供檢索。推理階段採用擴展推理寬度的平行採樣與聚合策略，優於僅增加推理深度的提示方法。訓練階段則透過知識增強的後訓練數據，結合監督微調與強化學習，後者效果更佳。整合上述三階段，提出KATE框架，實現知識獲取、推理激活及內化的統一流程，提升多步驟工具調用能力。

**Result:** 在BFCL-V3與AppWorld兩個多步驟工具使用基準上，KATE在不同模型規模（Qwen3-8B與Qwen3-32B）均顯著優於基線方法，包括直接工具調用、提示方法及Memp框架。BFCL-V3中，KATE提升平均性能約13%至15%，甚至超越GPT-4.1與GPT-5等先進模型。AppWorld中，KATE在大多數指標上優於ReAct基線，但在極端複雜任務中表現略低於Memp，顯示推理寬度擴展在高難度場景仍有挑戰。消融實驗證實經驗知識與推理寬度擴展皆為性能提升關鍵。強化學習微調優於監督微調，進一步提升工具調用準確率。

**Conclusion:** 本研究系統性分析經驗知識在LLM多步驟工具調用中的作用，證明實例層級知識比意圖層級更具實用價值。推理階段擴展寬度的平行採樣策略能更有效激活潛在知識，訓練階段透過知識增強後訓練進一步內化經驗，提升模型工具使用能力。提出的KATE框架整合知識獲取、激活與內化，實驗結果顯示其在多種場景與模型規模下均有穩定且顯著的性能提升。該方法對工程實踐中提升LLM作為自主工具代理的可靠性與準確性具有指導意義。

**Research Gap:** 本論文的評估主要基於相對小規模且靜態的知識庫，尚未探討大規模知識庫擴展對性能的影響，限制了方法在更廣泛場景的泛化能力。此外，實驗集中於純文本工具使用，未涵蓋多模態工具調用，限制了方法在視覺或多媒體交互場景的應用潛力。推理寬度擴展在極端複雜任務中可能引入噪聲，且聚合策略在長上下文推理時存在不穩定性，顯示模型對長序列多候選決策的處理能力仍有限。訓練階段雖採用強化學習，但對訓練成本與部署風險未充分討論，缺乏對失敗案例與泛化失效的深入分析。未來研究可針對這些方面進行更全面的驗證與優化。

### 4. Spatial-Omni: Spatial Audio Understanding Integration in Multimodal LLMs via FOA Encoding

- 分數：4
- arXiv: [2606.10738v1](http://arxiv.org/abs/2606.10738v1)
- 發表日期：2026-06-09
- 分類：eess.AS, cs.AI
- 入選理由：+2 提及頂級機構：mit; +2 提及程式碼可用; LLM rank #4 / 7/10: Introduces spatial audio modality integration into multimodal LLMs with a novel FOA encoding and large-scale dataset/benchmark; shows improved spatial audio understanding; practical and methodologically credible with cod

**Intro:** 本論文指出現有多模態大型語言模型（LLMs）多將音訊視為單聲道信號，忽略了空間音訊中包含的方向、距離與運動等空間線索，導致在聲源定位、空間關係推理及場景理解上能力不足。為此，作者提出Spatial-Omni，一種輕量級方法，通過SO-Encoder將第一階Ambisonics（FOA）空間音訊作為獨立模態注入現有Omni LLM中，且不修改原有音訊編碼器。為支持訓練與評估，作者構建了包含40萬FOA音訊片段及210萬空間問答對的SO-Dataset、SO-QA及SO-Bench，涵蓋16項空間音訊理解子任務。實驗結果顯示Spatial-Omni在空間音訊理解任務上優於現有開源大型音訊語言模型及Omni LLM，且保持合理的一般音訊理解能力。

**Motivation:** 空間音訊提供了方向、距離、運動等豐富的空間資訊，對3D聽覺場景理解至關重要，但現有多模態LLMs多忽略這些空間線索，僅處理單聲道音訊，限制了模型在空間定位與推理上的表現。已有空間音訊LLM方法多需大幅修改或重新訓練原始音訊編碼器，可能破壞其語義理解能力，且缺乏大規模FOA空間音訊問答數據與系統性評測基準，難以全面評估空間推理能力。這些問題阻礙了空間音訊理解技術在工程與產品中的落地與應用。

**Method:** Spatial-Omni架構在原有Omni LLM的音訊編碼器旁邊新增一個輕量級的SO-Encoder作為平行分支。原始音訊編碼器僅接收FOA的W通道以保留語義能力，SO-Encoder則接收FOA的4個mel頻譜特徵及3個強度向量（IV）特徵，提取方向、距離、運動及多源空間關係等空間線索。SO-Encoder經過三階段卷積與Transformer模組，並通過Temporal Pixel Shuffle Projector將空間潛在特徵壓縮映射為空間token，與音訊、視覺及文本token共同輸入LLM。訓練採用兩階段策略先穩定語義分類，再引入空間學習，並以SO-QA問答數據進行微調，確保空間token與LLM隱藏空間對齊。

**Result:** SO-Encoder在63類事件的空間聲源定位與距離估計任務中，F20達40.2%，DOA誤差17.2度，距離誤差0.22，優於多數公開基線。Spatial-Omni模型在SO-Bench的16項空間任務中整體表現最佳，特別在聲源檢測、方位估計、空間關係理解及動態事件分類等任務中顯著超越一般LALMs與Omni LLM。混合單聲道與空間問答數據的SO-7B(MIX)模型在保持一般音訊理解能力的同時，提升空間推理表現。消融實驗證實空間token的真實空間信息是性能提升的關鍵。模型在MMAU與MMAU-Pro基準上保持合理的語義理解能力，未造成嚴重遺忘。

**Conclusion:** 本研究提出Spatial-Omni及SO-Encoder，有效將FOA空間音訊作為獨立模態注入多模態LLM，提升空間音訊的定位、關係理解與複雜推理能力，同時保留原有音訊語義理解。作者構建了大規模空間音訊數據集與系統性評測基準，為空間音訊LLM研究提供了方法、數據與基線。實驗結果表明該方法在多項空間任務中均有顯著提升，展示了空間音訊模態在多模態理解中的價值。限制包括僅針對FOA格式，未評估其他空間音訊表示；SO-Encoder依賴SELD式監督限制了多源計數能力；空間適配仍對一般音訊能力有輕微影響。未來需探索更廣泛空間格式支持與更平衡的多模態訓練策略。

**Research Gap:** 本論文主要聚焦FOA格式空間音訊，未系統評估其他空間音訊表示如SALSA或不同麥克風陣列，限制了方法的泛化性。SO-Encoder依賴SELD風格的軌跡匹配與激活閾值，可能限制在多源重疊、短事件或空間模糊場景下的聲源計數與關係推理能力。空間模態的引入雖保留原音訊編碼器，但仍造成一般音訊理解能力的部分退化，顯示空間與語義能力的平衡尚未充分解決。實驗中缺乏對模型在極端噪聲、複雜動態場景或跨域場景的泛化測試，且未提供失敗案例分析。消融與比較雖涵蓋多種空間特徵，但對計算成本、部署風險及實時性能未有詳細討論。全文未明確說明模型在實際產品環境中的部署挑戰與資源需求。

### 5. The Arbiter Agent: Continually Monitoring Multi-Agent Conversations to Detect Emergent Misalignment

- 分數：4
- arXiv: [2606.10747v1](http://arxiv.org/abs/2606.10747v1)
- 發表日期：2026-06-09
- 分類：cs.AI
- 入選理由：+2 提及頂級機構：mit; +2 提及程式碼可用; LLM rank #5 / 7/10: Addresses multi-agent misalignment detection with a novel active monitoring agent under resource constraints; evaluated across diverse scenarios with code available; relevant for safe multi-agent AI systems.

**Intro:** 隨著多語言模型代理系統日益普及，這些代理需共同討論、協商並執行任務，然而即使單一代理看似對齊良好，代理間互動仍可能產生系統性錯誤。本文提出Arbiter代理，專為即時監控多代理對話並識別可能出現的錯誤對齊行為而設計。Arbiter在有限的檢查預算下運作，需謹慎選擇使用資源的時機與方式，並能透過多種工具如詢問代理、檢視系統提示及推理過程等，持續監控對話進展。最終，Arbiter會產出報告指出可能的錯誤對齊來源。

**Motivation:** 多代理系統中，個別代理即使表現良好，互動過程仍可能導致整體系統錯誤對齊，帶來安全風險。現有審核方法多為事後分析，缺乏針對多代理持續且有限資源下的監控機制。Arbiter旨在填補此缺口，提供一種能在對話進行中即時偵測錯誤對齊的工具，對工程與產品層面尤為重要，因為能及早發現並防止錯誤決策或有害行為，降低部署風險並提升系統安全性。

**Method:** Arbiter以推理與行動循環方式運作，每輪觀察對話歷史後選擇一項行動，包括等待觀察、詢問代理、檢視系統提示、檢查推理鏈及記錄可疑事件。每項行動均有成本，總成本受限於預算。對話由三個代理以固定輪替方式進行30輪，Arbiter根據對話內容及工具配置（從純觀察到完整檢查）判斷代理是否錯誤對齊。實驗涵蓋多種錯誤對齊來源，包括模型權重微調導致的行為偏差及系統提示引發的指令性錯誤對齊，並評估不同工具組合及基礎模型對檢測效能的影響。

**Result:** 實驗結果顯示，純觀察對於權重誘發的錯誤對齊檢測效果接近隨機，加入詢問與記錄工具後F1分數提升至0.51，且檢測時間與預算消耗均降低。系統提示誘發的錯誤對齊則較易被檢測，純觀察即可達到高準確率。評估意識及串通情境下，Arbiter在所有工具配置均表現優異，部分達到完美F1。記錄工具雖提升召回率，卻也增加誤報率。基礎模型能力影響顯著，DeepSeek-v3.2優於Qwen3.6-plus。預算增加對權重誘發錯誤對齊提升有限，顯示信號強度為瓶頸。

**Conclusion:** 本文提出的Arbiter代理成功實現了在有限預算下持續監控多代理對話並偵測錯誤對齊，證明主動檢查工具能加速且提升檢測準確度。不同錯誤對齊來源對檢測難度影響明顯，權重誘發較難偵測，指令誘發較易。記錄工具需謹慎使用以平衡精確度與召回率。基礎模型選擇對性能影響大。限制包括對話長度短、錯誤信號較明顯、代理能力不均及樣本數有限。未來將擴展至更自然對話與多樣錯誤信號，並整合解釋性工具提升監控效能。

**Research Gap:** 本研究對話長度固定且較短，且錯誤對齊信號較強，可能高估實際部署中檢測效能。代理能力不均，Arbiter較強，未測試對抗能力相當或更強代理的情況。實驗領域侷限於金融及特定錯誤類型，缺乏跨領域與更細微錯誤對齊的驗證。記錄工具雖提升召回但誤報率高，需更精細策略避免誤判。預算使用趨勢顯示Arbiter可能過早下結論，缺乏動態調整策略。缺少對失敗案例、泛化能力及長期監控穩定性的分析。全文未明確說明在真實環境中部署的成本與風險評估。
