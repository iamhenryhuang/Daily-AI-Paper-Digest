### 1. Hierarchical Abstract Tree for Cross-Document Retrieval-Augmented Generation

- 分數：7
- arXiv: [2605.00529v1](http://arxiv.org/abs/2605.00529v1)
- 發表日期：2026-05-01
- 分類：cs.LG, cs.AI, cs.IR
- 入選理由：+2 提及頂級機構：mit; +3 提及頂級會議：icml; +2 提及程式碼可用

**Intro:** 本論文聚焦於提升 Retrieval-Augmented Generation (RAG) 在跨文件多跳問題上的檢索與生成能力。傳統的 Tree-RAG 方法多針對單一文件，面對跨文件多跳查詢時存在分布適應性差、結構孤立及抽象粗糙等問題。作者提出 Ψ-RAG 框架，透過層級抽象樹索引與多粒度代理檢索機制，旨在解決上述限制，並支持從詞元級問答到文件級摘要的多樣任務。該方法在多個跨文件多跳問答基準上，明顯優於現有先進方法。

**Motivation:** 跨文件多跳問答是現代知識密集型應用的核心挑戰，要求檢索系統能有效整合多篇文件的資訊。現有的 Tree-RAG 方法因為依賴 k-means 聚類，對資料分布假設過於嚴格，導致檢索噪聲與性能下降。此外，樹狀索引缺乏跨文件節點的明確連結，無法捕捉多跳推理中的因果關係。抽象節點的粗糙表示也限制了細粒度資訊的檢索。這些問題直接影響工程部署時的檢索準確度與效率，進而影響產品的用戶體驗與研究的落地效果。

**Method:** Ψ-RAG 採用兩大核心組件：首先，透過一種層級抽象樹索引建構方法，利用迭代的“合併與折疊”過程，根據文件塊間的相似度動態構建樹狀結構，無需事先分布假設，能自適應資料分布。其次，設計一個多粒度代理檢索系統，包含一個檢索與回答代理（R&A agent），該代理能根據查詢動態調整檢索策略，並結合樹索引與稀疏關鍵字索引的混合檢索，透過查詢重組提升檢索精度。整體流程支持多輪檢索與推理，適應不同粒度的任務需求。

**Result:** 在多個跨文件多跳問答基準（如 HotpotQA、2Wiki、MuSiQue、MultiHop-RAG）上，Ψ-RAG 相較於 RAPTOR 提升約 25.9% 的平均 F1 分數，並超越 HippoRAG 2 約 7.4%。在單跳問答中，傳統稀疏與密集檢索方法表現良好，但在多跳任務中，Ψ-RAG 的代理檢索與混合索引策略顯著提升了檢索召回率與回答準確度。對於敘事問答與文件摘要任務，Ψ-RAG 也優於多數基線，展現其多粒度任務的通用性。消融實驗證實代理檢索、稀疏檢索與查詢重組均為性能提升的關鍵因素。效率分析顯示，Ψ-RAG 在索引構建速度上優於 RAPTOR，檢索延遲合理，適合實際部署。

**Conclusion:** Ψ-RAG 成功克服了傳統 Tree-RAG 在跨文件多跳檢索中的分布適應性差、結構孤立及抽象粗糙等問題，通過層級抽象樹索引與多粒度代理檢索實現了高效且準確的檢索增強生成。其在多種任務與資料集上均展現出顯著性能提升，且架構靈活，支持多種開源及閉源模型作為骨幹，具備良好的工程應用潛力。該工作推動了結構化 RAG 在大規模語料庫與複雜推理場景的發展。

**Research Gap:** 本論文雖然在多跳跨文件檢索上取得優異成績，但全文未明確說明在超大規模工業級語料（10M+ 甚至 100M+ 以上 tokens）上的實際效能與可擴展性，尤其是相似度矩陣計算的時間與空間成本可能成為瓶頸。此外，代理檢索的多輪互動雖提升準確度，但增加了檢索延遲，對低延遲應用場景的適用性尚未充分探討。消融實驗中對不同抽象類型的比較有限，未深入分析在何種任務或資料特性下選擇何種抽象更優。結構孤立問題雖部分透過代理檢索緩解，但缺乏對失敗案例或錯誤推理的系統性分析。最後，部署成本、模型大小與計算資源需求未被充分量化，對實際產品落地的影響仍需進一步評估。

### 2. Themis: Training Robust Multilingual Code Reward Models for Flexible Multi-Criteria Scoring

- 分數：7
- arXiv: [2605.00754v1](http://arxiv.org/abs/2605.00754v1)
- 發表日期：2026-05-01
- 分類：cs.SE, cs.LG
- 入選理由：+2 提及頂級機構：mit; +3 收錄於 Hugging Face Daily Papers; +2 提及程式碼可用

**Intro:** 本論文聚焦於語言模型後訓練中獎勵模型（Reward Models, RMs）在程式碼生成領域的應用，特別是多語言、多準則的程式碼評分。現有研究多以執行反饋為主，限制了後訓練只能優化功能正確性，忽略了程式碼的非功能性質量指標。作者提出Themis專案，包含多語言多準則的評測基準Themis-CodeRewardBench，以及大規模程式碼偏好資料集Themis-CodePreference，並基於此訓練多種規模的Themis-RM獎勵模型。論文旨在提升程式碼獎勵模型在多維度評分的準確性與泛化能力。

**Motivation:** 現有程式碼生成後訓練多依賴執行測試回饋，限制了只能評估可執行且自包含的程式碼，無法涵蓋多數真實世界中依賴外部模組的程式碼，且測試案例生成困難且易錯誤。此外，現有獎勵模型多聚焦於功能正確性，忽略執行效率、記憶體使用、可讀性與安全性等重要非功能指標。缺乏多語言、多準則的評測基準與訓練資料，導致獎勵模型泛化能力不足，難以應用於多樣化的工程與產品場景。

**Method:** 作者首先蒐集並整合多個現有資料集，建立Themis-CodeRewardBench，涵蓋五個程式碼品質維度與八種程式語言，並設計一套從GitHub合併提交中挖掘多準則偏好的管線。接著建立Themis-CodePreference資料集，包含超過35萬對程式碼偏好對，涵蓋功能正確性、執行效率、記憶體效率、可讀性與安全性五大準則。基於Qwen3模型，採用兩階段訓練策略：先在一般偏好資料集進行預訓練，再在程式碼偏好資料集進行微調。訓練時引入條件語言模型正則化與獎勵幅度正則化，並利用Bradley-Terry模型學習偏好排序，支援推理時指定評分準則的系統提示以減少準則間干擾。

**Result:** 實驗評估涵蓋多種現有獎勵模型與Themis-RM不同規模版本，使用Themis-CodeRewardBench進行多準則、多語言偏好準確度測試。結果顯示現有模型在非功能性指標上表現不佳，常接近隨機，而Themis-RM即使是最小模型也優於多數大型模型，最大模型達到最佳表現。消融實驗證明預訓練階段與輔助損失有助提升準確度，且推理時明確指定評分準則能有效減少準則間干擾。跨語言實驗顯示多語言訓練帶來正向轉移，且不同語言間表現差異小。下游任務中，Themis-RM在列表重排序與對抗性測試中均展現強健性與優越性能。

**Conclusion:** 本論文提出Themis專案，成功建立多語言、多準則的程式碼獎勵模型評測基準與大規模偏好資料集，並訓練出一套具備良好擴展性與泛化能力的多維度程式碼獎勵模型。實驗結果證明多準則訓練能有效提升模型在功能與非功能性質量評分的準確度，且具備跨語言轉移能力與下游應用的魯棒性。此成果有助於推動程式碼生成後訓練從僅依賴執行回饋轉向更靈活、多元的獎勵信號，促進工程與產品層面的實際落地。

**Research Gap:** 論文雖涵蓋多語言與多準則，但訓練資料仍主要來自特定時間區間與特定開源庫，可能限制泛化至其他程式碼庫或新興語言。對於更低資源語言或極端非功能性指標的評估較少，且未明確探討模型在極端錯誤或惡意攻擊下的失效案例。消融實驗中模型合併策略表現不佳，顯示多準則間干擾問題仍未完全解決。全文未明確說明訓練與推理成本，以及模型部署於實際系統的效率與安全風險。未提供對比於執行回饋方法的直接效能比較，缺乏對實際後訓練提升效果的量化分析。

### 3. End-to-End Autoregressive Image Generation with 1D Semantic Tokenizer

- 分數：6
- arXiv: [2605.00503v1](http://arxiv.org/abs/2605.00503v1)
- 發表日期：2026-05-01
- 分類：cs.CV, cs.LG
- 入選理由：+3 收錄於 Hugging Face Daily Papers; +3 提及頂級會議：icml

**Intro:** 本論文聚焦於自回歸圖像生成中視覺tokenizer的設計與訓練，特別是將圖像壓縮為一維序列的語義token。傳統方法多採用兩階段訓練，先訓練tokenizer再訓練生成模型，導致tokenizer無法針對生成任務優化。作者提出一個端到端的訓練框架，能同時優化重建與生成，並引入視覺基礎模型的語義對齊以提升tokenizer表現。該方法在ImageNet 256×256生成任務中達到無指導下的最先進FID分數1.48。論文同時探討了1D tokenizer在自回歸建模中的優勢及其與2D結構tokenizer的差異。

**Motivation:** 自回歸圖像生成依賴tokenizer將高維圖像轉為離散token，但現有多數tokenizer保留2D空間結構，與自回歸模型的單向序列建模不匹配，限制生成效果。兩階段訓練使tokenizer無法從生成結果中獲得直接反饋，影響token質量和生成性能。此外，如何有效利用視覺基礎模型的語義資訊提升tokenizer的語義表達能力，仍未有明確方案。解決這些問題對提升自回歸生成模型的實用性和效果具有重要意義，尤其在工程部署和大規模訓練中。

**Method:** 作者設計一個基於1D Vision Transformer的tokenizer，將圖像patch與可學習查詢token串接後輸入因果編碼器，產生離散token序列。核心創新是端到端聯合訓練框架，同時優化重建損失和自回歸生成的下一token預測損失，並引入自回歸預測重建(APR)損失，將生成token解碼回像素空間以提供端到端生成反饋，防止token空間崩潰。此外，提出隱式對齊策略，將視覺基礎模型的語義特徵對齊到tokenizer的隱藏patch嵌入，而非直接對齊token，避免強制2D空間結構，提升生成質量。整體模型包括tokenizer編碼器、解碼器及自回歸生成模型，並採用IBQ量化與多種損失組合訓練。

**Result:** 在ImageNet-1K 256×256生成任務中，EOSTok模型在無指導條件下達成1.48的gFID，優於多數1D及2D tokenization基礎的生成模型。端到端訓練結合APR損失顯著提升生成質量，避免了純NTP損失導致的codebook崩潰問題。語義對齊實驗顯示隱式對齊策略在提升生成FID和AR模型預測準確率上效果最佳。模型在不同規模（93M至644M參數）下均展現良好擴展性，且較長token序列提升重建質量但對生成有一定折衷。與兩階段訓練相比，端到端訓練在生成質量上有明顯優勢。實驗中也分析了codebook大小、序列長度等超參數對性能的影響。

**Conclusion:** 本論文提出一種端到端聯合訓練的1D語義tokenizer與自回歸生成模型框架，有效解決了傳統兩階段訓練中tokenizer與生成模型脫節的問題。APR損失提供了從像素空間到tokenizer的直接生成反饋，提升了生成質量。隱式語義對齊策略成功將視覺基礎模型的語義信息注入1D tokenizer，避免了2D空間結構限制。整體方法在ImageNet 256×256生成任務中取得了最先進的無指導FID分數，並展現良好的模型擴展性。該工作對自回歸圖像生成的工程實踐和研究落地具有重要參考價值。

**Research Gap:** 論文未明確說明端到端訓練在更大規模數據集或更高解析度圖像上的泛化能力與穩定性，且對於不同視覺基礎模型的選擇及其對生成性能的影響缺乏深入比較。APR損失雖有效防止codebook崩潰，但其計算成本及對訓練效率的影響未充分討論。序列長度與codebook大小的折衷仍需更細緻的理論分析與實驗驗證。對於生成失敗案例、模型在多樣化場景下的魯棒性及部署風險也未有說明。此外，隱式語義對齊策略的具體機制及其在不同架構上的適用性仍有待進一步探究。

### 4. Aitchison Embeddings for Learning Compositional Graph Representations

- 分數：5
- arXiv: [2605.00716v1](http://arxiv.org/abs/2605.00716v1)
- 發表日期：2026-05-01
- 分類：cs.LG, cs.SI
- 入選理由：+2 提及頂級機構：mit; +3 提及頂級會議：icml

**Intro:** 圖結構資料在社交網絡、通訊網絡及化學資訊等領域廣泛存在，學習高品質的圖節點表示是圖機器學習的核心。傳統圖嵌入方法多著重於保留結構鄰近性，但通常缺乏內在的可解釋性，難以揭示學習特徵與圖結構的關聯。許多網絡中節點角色可視為多個潛在典型因素的混合，這種組合性結構促使作者提出基於Aitchison幾何的組合式圖嵌入框架，將節點表示為單純形上的組合，並透過等距對數比（ILR）座標嵌入，保留相對比例的幾何距離。

**Motivation:** 現有圖嵌入方法多為歐氏空間表示，缺乏對節點角色的連續且重疊混合結構的有效建模，且其幾何距離缺乏語義解釋。混合成員模型雖允許角色重疊，但仍受限於軸對齊的離散角色假設，無法捕捉連續且非可識別的角色結構。作者認為，節點角色的相似性應基於相對比例的比較而非絕對值，故引入Aitchison幾何作為比較組合數據的標準幾何，藉此提升表示的可解釋性與結構合理性。

**Method:** 本方法稱為Aitchison Compositional Graph embeddings (AICoG)，將每個節點表示為潛在典型因素的組合，位於單純形空間中。利用ILR轉換將組合數據映射至無約束的歐氏空間，保證距離保持Aitchison距離不變性。邊的生成機率以ILR空間中節點距離的負平方作為log-odds，結合節點偏置以捕捉度數異質性。ILR基底可固定（如Helmert基底）或學習，支持靈活且可解釋的對數比對比。模型透過最大化伯努利邏輯斯迴歸似然訓練，並利用子組合一致性進行成分限制與分析。

**Result:** 在多個公開資料集（如Cora、Citeseer、DBLP等）上，AICoG在連結預測與節點分類任務中與強基線方法（如NODE2VEC、ROLE2VEC、SLIM-RAA等）表現相當甚至更優。實驗顯示，採用Aitchison幾何的組合嵌入明顯優於僅使用歐氏幾何的單純形方法。子組合限制實驗證明在不重新訓練下，移除部分成分後仍能保持較高的下游任務效能。透過不同ILR基底的比較，展示了模型解釋性的一致性與穩定性，並以對數比軌跡示範了可解釋的角色權重調整。

**Conclusion:** 本文提出的AICoG框架成功將節點角色建模為潛在典型因素的組合，並利用Aitchison幾何賦予嵌入空間明確的語義解釋，兼具表現力與可解釋性。實驗結果證明該方法在無特徵、無監督設定下，能提供與現有強基線相當的預測性能，同時支持基於組合結構的成分限制與分析。限制方面，該方法最適用於節點角色本質為組合結構的場景，對於非組合性角色結構可能無法提升準確度。全文未明確說明在大規模圖或帶有節點特徵的場景下的表現與擴展性。

**Research Gap:** 本文未充分探討AICoG在含節點特徵或監督學習場景中的適用性與效能，亦缺乏對大規模圖的計算效率與擴展性分析。子組合限制雖展示了成分移除的穩定性，但未提供對失敗案例或極端壓縮情況下性能下降的詳細評估。ILR基底的學習雖提升解釋性，但對不同基底選擇對下游任務影響的系統性比較不足。此外，模型對非組合性角色結構的泛化能力及其在動態或異質圖上的適用性尚未驗證。最後，全文未明確說明模型在實際部署中可能面臨的計算成本與調參難度。

### 5. Jailbreaking Vision-Language Models Through the Visual Modality

- 分數：5
- arXiv: [2605.00583v1](http://arxiv.org/abs/2605.00583v1)
- 發表日期：2026-05-01
- 分類：cs.CV, cs.AI, cs.LG
- 入選理由：+2 提及頂級機構：mit; +3 提及頂級會議：icml

**Intro:** 本文探討視覺語言模型（VLMs）在安全對齊方面的漏洞，特別是視覺模態作為攻擊面尚未被充分研究。隨著VLM在多模態助理、圖像搜尋及文件理解等應用中廣泛部署，確保其安全性成為重要挑戰。現有安全訓練多聚焦於文本模態，忽略了視覺輸入可能引發的安全風險。作者提出四種利用視覺模態進行繞過安全機制的越獄攻擊，並在多款先進VLM上驗證其有效性。

**Motivation:** 現有VLM的安全對齊主要基於文本訓練，未能有效涵蓋視覺信息中隱含的有害意圖，導致跨模態安全差距。視覺輸入能顯著影響模型行為，且視覺攻擊不依賴梯度或明顯文本提示，增加了安全防護的難度。針對這一問題，本文旨在揭示視覺模態的安全漏洞，促使研究者和工程師重視視覺作為安全防護的第一級目標，提升VLM在實際產品和系統中的安全性。

**Method:** 作者設計四種視覺越獄攻擊：1) 視覺密碼攻擊，將有害指令編碼為抽象符號序列並附帶解碼圖例；2) 視覺物件替換，將有害物件替換為無害替代品，並利用場景上下文誘導模型恢復原意；3) 視覺文字替換，將圖片中有害文字替換為無害詞彙，但保留視覺和文化上下文；4) 視覺類比謎題，通過隱含推理將禁止概念以類比形式呈現。所有攻擊均配合文本中性化策略，去除明顯有害詞彙，強調視覺信息的解碼與推理。評估採用多模型、多次採樣及多語言模型評判員組合，確保結果穩健。

**Result:** 在六款先進VLM（包括GPT-5.2、Claude-Haiku-4.5、Gemini系列及Qwen3系列）上，視覺攻擊普遍能繞過安全對齊，且在部分模型上視覺攻擊成功率顯著高於文本對應攻擊。以Visual Cipher為例，Claude-Haiku-4.5視覺攻擊成功率達40.9%，遠高於文本攻擊的10.7%。不同模型對視覺與文本攻擊的敏感度差異明顯，部分模型更依賴視覺上下文。與先前VLM越獄方法相比，本文提出的攻擊更具語義意義且機制清晰。初步機制分析指出視覺攻擊可能導致語義整合與拒絕判斷的時間錯配，輸出端分類器則能有效攔截多數攻擊成功輸出。

**Conclusion:** 本文揭示了VLM視覺模態存在的安全對齊缺口，證明純文本安全訓練不足以防範視覺渠道的有害指令。提出的四種視覺越獄攻擊方法不僅展示了跨模態安全漏洞，也提供了可解釋的攻擊機制。實驗結果強調未來VLM安全設計需將視覺視為核心防護對象，並結合輸出端分類器作為防禦策略。研究限制包括依賴HarmBench數據集、缺乏部分模型內部資訊及生成圖像質量波動，未涵蓋多模態複合攻擊和模型間遷移性。整體而言，本文為VLM安全研究提供了重要實證與方法論參考。

**Research Gap:** 本文主要依賴HarmBench作為評估基準，可能未涵蓋所有潛在有害類別，限制了結果的全面性。對於專有模型缺乏內部結構訪問，限制了深入機制分析的深度。生成圖像質量不穩定，可能影響攻擊一致性與可重現性。攻擊評估未考慮多模態複合攻擊或跨模型遷移，缺乏對攻擊泛化能力的探討。安全防禦主要聚焦於輸出端分類，對於輸入端視覺安全訓練和實時監控策略尚未充分展開。最後，視覺類比謎題的抽象性導致較高誤解率，顯示視覺推理能力仍有限，未來需提升模型對視覺隱含語義的理解與拒絕能力。
