### 1. Selective Disclosure Watermarking for Large Language Models

- 分數：7
- arXiv: [2607.05353v1](http://arxiv.org/abs/2607.05353v1)
- 發表日期：2026-07-06
- 分類：cs.CR, cs.AI, cs.CL, cs.LG
- 入選理由：+2 提及頂級機構：meta; +3 提及頂級會議：icml; +2 提及程式碼可用; LLM rank #1 / 9/10: Proposes a novel selective disclosure watermarking method for LLMs addressing privacy concerns with strong experimental validation and code release; accepted at ICML 2026.

**Intro:** 大型語言模型（LLMs）生成的文本容易被濫用，導致學術不誠實、錯誤資訊等問題，因此需要有效的內容驗證機制。現有的水印方法多為零位元方案，僅能區分機器生成與人類文本，或多位元方案嵌入完整元資料，但缺乏選擇性揭露功能。這使得驗證時必須公開全部嵌入訊息，造成隱私風險與資訊過度暴露。本文提出一種名為Hierarchical Vocabulary Routing（HeRo）的水印框架，能根據授權層級選擇性揭露嵌入的元資料，解決現有方法的限制。

**Motivation:** 在實際應用中，平台需要同時滿足公開驗證與私密審計的需求，既要讓大眾確認內容來源，又要限制敏感資訊的存取權限。現有多位元水印方案無法做到部分驗證，導致只能全揭露或全隱藏，限制了部署彈性。HeRo透過詞彙層級的遞迴分割，將水印訊息分散於多層次，讓不同權限的驗證者只能解碼對應層級的訊息，兼顧驗證準確度與隱私保護，對工程與產品落地具有重要意義。

**Method:** HeRo方法將詞彙表遞迴分割成多個層級，每層根據對應的訊息段落選擇子集，最終在生成過程中引導token採樣。此過程保持統計無偏性，確保生成文本品質不受影響。解碼時，驗證者根據持有的密鑰只能還原授權層級的訊息段，未授權層級則無法超越隨機猜測。方法以Gumbel-Max採樣為基礎，並設計高效的批次化GPU實現，兼顧生成與解碼效率。

**Result:** 實驗使用Llama2-7B模型與C4及OpenGen資料集，評估選擇性揭露、解碼準確度、文本品質、效率與魯棒性。HeRo在24至48位元水印下，解碼準確度優於MPAC、StealthInk及BiMark等基線，且文本困惑度接近無水印基準。兩層配置支持公開與私密訊息分層驗證，未授權驗證者私密訊息解碼準確度約為隨機。生成與解碼延遲低於基線，且支持批次處理。對隨機替換、插入、刪除及翻譯回譯等攻擊展現較強魯棒性。

**Conclusion:** 本文提出的HeRo框架有效實現了大型語言模型文本的多位元水印選擇性揭露，兼顧生成品質、解碼準確度與計算效率。其分層詞彙路由設計支持細粒度存取控制，滿足實際應用中公開驗證與私密審計的需求。實驗結果證明方法在多項指標上優於現有多位元水印方案，並具備良好魯棒性與低延遲。限制在於全文未明確說明在更深層次分層或更大規模模型上的表現，以及對更複雜攻擊的防禦能力。

**Research Gap:** 本文雖驗證了兩層及部分更深層次配置，但未充分探討多層次水印在極端分層權限下的效能與安全性。實驗主要集中於Llama2-7B與特定資料集，泛化至其他模型與文本類型尚待驗證。對抗性攻擊的防禦僅限於常見文本擾動，缺乏對更複雜語義或結構變換的評估。此外，方法在實際大規模部署中的密鑰管理與安全風險未詳述，解碼過程中可能的失敗案例與誤判率也未充分分析。未來需加強這些方面以提升實用性與安全保障。

### 2. GeoFlow: Geo-Aware Modeling of Inter-Area Relationships in Origin-Destination Flow Prediction and Generation

- 分數：7
- arXiv: [2607.05257v1](http://arxiv.org/abs/2607.05257v1)
- 發表日期：2026-07-06
- 分類：cs.LG
- 入選理由：+2 提及頂級機構：mit; +3 提及頂級會議：icml; +2 提及程式碼可用; LLM rank #2 / 8/10: Introduces GeoFlow, a geo-aware framework improving origin-destination flow prediction with clear novelty, strong empirical results, and code; ICML 2026 acceptance.

**Intro:** 本論文聚焦於起訖區域流量（Origin-Destination flow）建模，該問題對城市規劃與交通分析具關鍵意義。現有圖結構方法多忽略地理屬性，導致難以捕捉長距離及多區域依賴關係。作者提出GeoFlow框架，結合地理空間屬性與專門設計的編碼器與解碼器，提升OD流量的預測與生成能力。實驗涵蓋通勤、貨運及航空三種流量類型，並在多項指標上超越現有方法。

**Motivation:** OD流量建模面臨的主要挑戰是如何有效表示區域間複雜的地理與拓撲關係。傳統圖神經網絡雖具不變性與計算優勢，但難以捕捉長距離及多區域互動。純座標基方法敏感於參考原點與尺度，且難以推斷如測地距離等重要屬性。缺乏這些幾何屬性限制了模型對真實流動行為的理解與預測準確度。故需結合多種地理特徵並設計合適的編碼解碼策略以提升模型表現。

**Method:** GeoFlow首先對區域對增強三種地理關係特徵：相對位置、k跳距離與測地距離，並將區域屬性分為幾何屬性與內在屬性兩類分別編碼。幾何屬性透過多層感知機編碼，內在屬性則利用圖注意力機制結合地理關係加權聚合。兩者融合後形成統一表示。解碼器採用軸向-全局注意力機制，交替對起點與終點維度進行注意力計算，有效捕捉競爭性依賴，並透過緊湊全局注意力補充長距離多區域互動。生成任務則結合流匹配模型，提升生成樣本的真實性與多樣性。

**Result:** 在通勤、貨運與航空三個數據集上，GeoFlow在預測任務中相較於強基線TransFlower，CPC指標提升約10%以上，NRMSE與JSD顯著下降，顯示預測準確度提升。生成任務中，GeoFlow在重建誤差、分布相似度及樣本多樣性指標均優於NetGAN、DiffODGen及WEDAN。消融實驗證實相對位置、k跳距離與測地距離等地理特徵均帶來性能增益，且專門設計的編碼器與解碼器架構對結果有顯著貢獻。分析實驗進一步證明高階地理屬性難以從鄰接矩陣與直線距離推斷，支持明確提供這些特徵的設計合理性。

**Conclusion:** GeoFlow提出一套系統性整合地理空間屬性與專門編碼解碼架構的OD流量建模方法，顯著提升預測與生成任務的性能。其地理屬性增強策略與軸向-全局注意力機制有效捕捉長距離及多區域依賴，並透過流匹配生成模型提升樣本品質與多樣性。實驗結果與消融分析充分驗證了方法設計的有效性，為OD流量建模提供了新的思路與工具。

**Research Gap:** 論文未明確說明GeoFlow在極端數據稀疏或地理信息缺失情況下的表現與魯棒性，這對實際部署尤為重要。生成模型的計算成本與推理效率雖有提及，但缺乏詳細分析，限制工程落地評估。模型對不同區域劃分尺度的泛化能力與跨區域遷移性能未充分探討。此外，對失敗案例或模型在異常流量模式下的行為缺乏說明，限制對模型穩健性的全面理解。最後，消融實驗主要集中於單一數據集，跨多場景的穩定性驗證仍不足。

### 3. CanniUplift: A Holistic Framework for Mitigating Seller and Incentive Cannibalization in E-commerce Uplift Modeling

- 分數：5
- arXiv: [2607.05242v1](http://arxiv.org/abs/2607.05242v1)
- 發表日期：2026-07-06
- 分類：cs.LG, cs.AI, cs.IR
- 入選理由：+2 提及頂級機構：mit; +3 提及頂級會議：kdd; LLM rank #3 / 8/10: Addresses practical challenges in e-commerce uplift modeling with a unified framework validated on large-scale real and synthetic data, showing significant business impact; KDD 2026.

**Intro:** 本論文聚焦於電商平台中個性化激勵分配的增量建模問題，特別是在多賣家環境下傳統模型因違反穩定單位處理值假設（SUTVA）而面臨挑戰。作者指出兩大主要問題：賣家層級的食肉效應，即激勵導致用戶消費從一個賣家轉移至另一個賣家，平台整體GMV無增長；以及激勵層級的食肉效應，即用戶可能在未兌換指定激勵的情況下完成購買，造成增量估計噪聲。為解決這些問題，提出一個統一框架CanniUplift，旨在減緩雙重食肉效應，提升平台整體增量GMV。

**Motivation:** 在多賣家電商生態中，傳統增量模型忽略了跨賣家間的消費替代效應，導致局部賣家GMV提升卻犧牲了平台整體收益。此外，用戶同時面臨多種激勵，未兌換激勵的購買行為混淆了真實的激勵效果，造成增量估計偏差。這些問題直接影響激勵分配的精準性和資源利用效率，對工程和產品落地造成挑戰。因而需要一套能同時考慮跨賣家替代和激勵兌換行為的增量建模方法。

**Method:** CanniUplift包含三大核心模組：平台層級全局對齊（PGA）通過約束用戶在所有賣家上的GMV總和，捕捉跨賣家替代效應，避免局部賣家增量被錯誤放大；基於兌換行為的分解去噪（RDD）將處理組結果分解為兌換與未兌換兩條路徑，減少因未兌換激勵導致的增量歸因噪聲；以及Treat-Attention機制，利用Transformer編碼用戶歷史行為序列與當前激勵候選集，捕捉細粒度的用戶-激勵交互。整體模型以Tweedie損失函數優化，並結合多任務學習同時預測賣家GMV、平台GMV及兌換概率。

**Result:** 在合成數據和大規模工業數據集上，CanniUplift相較於多種先進基線模型（如DragonNet、EUEN等）在加權AUUC和QINI指標上均有顯著提升。消融實驗證明PGA和RDD模組分別從平台整體一致性和激勵兌換噪聲兩方面提升模型性能，且兩者結合效果最佳。行為分析顯示模型能合理捕捉用戶跨賣家替代行為及兌換與非兌換購買的差異。線上A/B測試結果顯示，該框架使平台增量GMV提升4.08%，同時降低2.45%行銷成本，ROI提升6.69%，證明其在實際生產環境中的有效性。

**Conclusion:** 本文提出的CanniUplift框架針對電商多賣家環境中常見的賣家層級與激勵層級食肉效應問題，通過平台層級全局對齊和基於兌換的分解去噪，有效緩解了SUTVA違反帶來的增量估計偏差。實驗結果和線上部署均證明該方法在提升平台整體增量GMV和ROI方面具有實際價值。該工作強調了在複雜市場生態中考慮跨單位干擾的重要性，為個性化激勵分配提供了更精準的因果推斷工具。

**Research Gap:** 本論文未充分探討時間維度上的食肉效應，即促銷可能導致用戶提前購買，短期增量被高估但長期收益受損。此外，模型依賴歷史行為數據，對即時替代行為的捕捉存在延遲，可能影響動態場景下的準確性。實驗中對激勵層級食肉效應的合成數據驗證有限，且未明確展示模型在多種激勵類型及異構賣家結構下的泛化能力。消融分析雖有，但對模型計算成本、部署風險及失敗案例缺乏深入討論，未來工作可聚焦於時序建模、實時替代行為預測及更全面的場景覆蓋驗證。

### 4. LLM-as-a-Verifier: A General-Purpose Verification Framework

- 分數：5
- arXiv: [2607.05391v1](http://arxiv.org/abs/2607.05391v1)
- 發表日期：2026-07-06
- 分類：cs.AI, cs.CL, cs.LG, cs.MA, cs.RO
- 入選理由：+3 收錄於 Hugging Face Daily Papers; +2 提及程式碼可用; LLM rank #4 / 7/10: Presents a general-purpose LLM verification framework with probabilistic scoring, state-of-the-art results on multiple benchmarks, and practical RL applications; code available.

**Intro:** 本文提出LLM-as-a-Verifier，一個通用的驗證框架，旨在提升大型語言模型(LLM)在判斷解決方案正確性方面的能力。驗證能力是目前LLM性能提升中較少被探討的維度，傳統方法多使用離散分數評價，導致判斷不夠細緻且易產生平手。該框架透過計算評分標記機率分布的期望值，產生連續且細緻的分數，從而改善驗證的精度與可擴展性。實驗涵蓋程式碼、機器人與醫療等多領域，展示其廣泛適用性與優越表現。

**Motivation:** 現有LLM在生成能力上透過擴大訓練資料與計算資源取得顯著進展，但驗證能力仍受限於粗糙的離散評分機制，導致無法有效區分複雜解決方案的優劣。訓練獎勵模型雖能提升驗證，但受限於訓練數據且跨領域泛化能力不足。為了充分利用LLM生成多樣解決方案的潛力，需要一種無需額外訓練即可提供細緻且可靠驗證信號的框架，以提升選擇最佳解決方案的能力並促進實際應用落地。

**Method:** LLM-as-a-Verifier核心在於利用語言模型對評分標記的機率分布，計算其期望值作為連續分數，避免傳統離散分數的平手問題。該方法從三個維度擴展驗證能力：一是提高評分標記的粒度，讓模型能更細緻地表達信心；二是透過多次重複評估降低隨機性與偏差；三是將評分標準拆解為多個子標準，減少單一標準的偏誤。此外，提出了Probabilistic Pivot Tournament演算法，在有限驗證預算下高效選出最佳解決方案。整體流程不需額外訓練，且可跨多模態與多領域使用。

**Result:** 在四個跨領域基準測試中，LLM-as-a-Verifier均達到或超越現有最佳成績：Terminal-Bench V2達86.5%，SWE-Bench Verified達78.2%，RoboRewardBench達87.4%，MedAgentBench達73.3%。在機器人視覺語言評分任務中，優於多個專門訓練的獎勵模型，且在人類標註的偏好準確度與誤差指標上表現更佳。細緻的驗證分數與任務進度高度相關，能用於實時監控代理行為。作為強化學習的密集獎勵信號，提升了SAC與GRPO算法在機器人與數學推理任務上的樣本效率，分別達約1.8倍與1.1倍提升。

**Conclusion:** 本文提出的LLM-as-a-Verifier框架，通過連續分數與多維度擴展策略，有效提升了LLM在多領域長程任務中的驗證能力。該方法不需額外訓練，具備良好泛化性，並能作為任務進度估計與強化學習獎勵信號，促進自主代理系統的安全與效率。實驗結果證明其在程式碼生成、機器人操作及醫療決策等場景的實用價值，為驗證能力的擴展提供了新視角與技術路徑。

**Research Gap:** 全文未明確說明該框架在極端長序列或超大候選池下的計算成本與延遲表現，對於低資源環境的適用性尚不清楚。雖然提出了成本效益較高的排名演算法，但對於不同模型架構或非logprob可用模型的兼容性與效能影響缺乏深入分析。實驗主要集中於特定基準，對於更廣泛或更複雜的真實世界場景泛化能力仍需驗證。此外，對於失敗案例的分析有限，缺乏對驗證錯誤原因及其對下游決策影響的系統性探討。最後，強化學習中獎勵加權參數的敏感性與穩定性亦未充分評估。

### 5. TabPack: Efficient Hyperparameter Ensembles for Tabular Deep Learning

- 分數：5

- 發表日期：2026-07-06
- 分類：cs.LG

**Intro:** 本論文聚焦於表格數據的深度學習，特別是多層感知機（MLP）集成方法。傳統的高效MLP集成通常使用相同的超參數，需耗費大量時間進行調參以達到最佳效能。作者提出TabPack，一種能在單次訓練中並行採樣並訓練多個具有不同超參數的MLP，並在訓練過程中動態選擇集成成員的新方法。此方法降低了對精確超參數設定的依賴，提升了使用便利性與效率。

**Motivation:** 表格數據是實務中常見的機器學習任務，傳統上以梯度提升決策樹為主流，但深度學習模型近年來表現逐漸提升。集成模型通常能超越單一模型，但現有高效集成方法缺乏超參數多樣性，仍需耗費大量資源調參。如何在保持集成效能的同時，減少調參成本並加速訓練，是實務與研究落地的重要挑戰。TabPack旨在解決此問題，提升模型在不同任務上的即用性與效率。

**Method:** TabPack基於Packed Ensembles架構，將多個具有不同深度、寬度、丟棄率等超參數的MLP模型打包成一個單一模組，利用向量化操作並行訓練。優化器也以類似方式打包，支持不同學習率、權重衰減等超參數。訓練過程中，TabPack每個epoch根據驗證集表現動態更新集成成員，並對表現不佳的基模型進行早停。超參數由使用者指定範圍隨機採樣，無需精確調整。此方法兼顧了效率、性能與調參靈活性。

**Result:** 在包含17個中大型公開表格數據集的實驗中，TabPack以預設超參數設定達到與大量調參方法相當的性能，顯著降低了調參時間與計算資源消耗。特別地，TabPack在MacBook上訓練速度優於部分基線在NVIDIA A100 GPU上的調參時間。與離線集成方法相比，TabPack的在線集成策略在性能與效率上均有優勢。實驗還顯示超參數多樣性主要促進了無需調參的工作流程，而非直接提升性能。不同基模型寬度的多樣化對性能影響有限。

**Conclusion:** TabPack提出了一種高效且易用的MLP超參數集成方法，能在表格數據任務中實現強勁的即用性能，顯著降低調參需求與計算成本。此方法適合實務應用與快速研究迭代，為表格深度學習提供了有力基線。限制包括對基模型架構的依賴以及對驗證集的使用，未來可探索更優化的在線集成算法、減少驗證集依賴及擴展超參數多樣化範圍。

**Research Gap:** 本論文未充分說明TabPack在極端小樣本或高度異構數據集上的泛化能力，且對失敗案例與模型穩定性分析有限。實驗中基模型架構較為單一，未探討更複雜結構或其他模型族的適用性。超參數空間的選擇與分布對性能影響未深入分析，且在線集成策略的收斂性與計算成本權衡未明確量化。此外，部署成本與實際生產環境中可能遇到的瓶頸未被充分討論。未來研究可針對這些方面進行更細緻的驗證與優化。
